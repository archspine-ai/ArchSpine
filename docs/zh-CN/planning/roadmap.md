# ArchSpine 路线图（v1.0 及以后）

使命：
通过确定性提取、语义索引和治理能力，把代码仓库变成高保真的 Agent 可读工程资产。

## 阶段 1：v1.0 稳定化

目标：
发布一个对外可承诺、可验证的稳定契约。

核心优先级：

- 以确定性 AST 提取作为不可妥协的基础
- 强化 SQLite 状态周围的原子写安全
- 稳定本地 Agent 工作流的 MCP 接入
- 让摘要更紧密锚定实际符号

开发者体验：

- 保持低门槛接入
- 减少命令行为歧义
- 在现有契约稳定前，不扩大功能面

## 阶段 2：v1.x 性能与反馈闭环

目标：
降低大型仓库的长期维护成本，提升响应速度。

方向：

- 符号级增量冒泡
- 更清晰的语义漂移可视化
- 扩展到 TypeScript/JavaScript 之外的语言

## 阶段 3：v2.0 系统级优化

目标：
把扫描速度和治理体验推进到更接近基础设施工具的水平。

可能的方向：

- 用 Rust 重写遍历和哈希热点路径
- 基于守护进程的原生同步触发
- 语言无关的语义 IR
- 更深的 CI 与 IDE 治理链路

## 长期方向

1. 让 `.spine` 成为 AI 辅助工程中可识别的仓库标准。
2. 从违规检测逐步走向更安全、受治理的修复闭环。
