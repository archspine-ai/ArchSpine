# Code of Conduct

## Our Standard

ArchSpine is an engineering project. We expect discussion to stay constructive, technically grounded, and respectful.

Participants should:

- assume good intent while still being precise about problems
- focus critique on behavior, code, decisions, and evidence
- share context clearly when reporting bugs or proposing changes
- respect project boundaries, release scope, and maintainer time

Participants must not:

- harass, insult, or intimidate others
- publish private information without permission
- spam, derail discussions, or repeatedly ignore project guidance
- use issues or pull requests to disclose unpatched security problems publicly

## Enforcement

Project maintainers may edit, hide, lock, or remove comments, issues, pull requests, or other contributions that violate this code of conduct. Repeated or severe violations may lead to temporary or permanent participation restrictions.

## Reporting

If you experience harassment, observe abusive behavior, or need moderator attention, contact the maintainers through the support route documented in [SUPPORT.md](./SUPPORT.md). Security-sensitive concerns should follow [SECURITY.md](./SECURITY.md).
