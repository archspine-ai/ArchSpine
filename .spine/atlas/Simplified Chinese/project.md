ArchSpine 是一个语义镜像系统，帮助开发者创建并维护代码库的 AI 增强结构化表示，用于架构分析、文档生成和智能代码理解。该系统编排了一个多阶段流水线，涵盖从文件系统扫描到规则执行和视图生成的完整流程。

### 主要模块及其协作方式

1. **CLI 层（`src/cli`）** – 命令行界面负责处理用户交互、参数解析和命令路由。它提供 `build`、`sync`、`init`、`view`、`check` 等子命令，将请求委派给相应的核心服务。

2. **核心流水线编排（`src/core`）** – 定义执行契约、错误码和流水线编排器。管理同步任务运行的生命周期，协调扫描、解析、摘要生成、验证和状态提交。

3. **扫描引擎（`src/engines`）** – 扫描器使用 Git 感知的忽略规则发现文件。与上下文解析器和规则执行器配合，加载架构规则（如层隔离、命名约定）并检查违规情况。报告引擎生成健康、使用和违规报告。

4. **基础设施服务（`src/infra`）** – 提供基础支持：
   - **配置** – 加载、验证和存储配置设置。
   - **数据库** – 管理 SQLite 生命周期、模式以及用于漂移、文件状态、符号和违规的仓库。
   - **LLM** – 抽象多个 LLM 提供商（Gemini、OpenAI、模拟），具备重试逻辑、凭据处理和提示组装功能。
   - **凭据** – 跨平台的安全存储，用于 API 密钥。
   - **清单** – 处理文件哈希、完整性验证和状态持久化。
   - **MCP 服务器** – 通过模型上下文协议将 ArchSpine 的内部资源和工具暴露给外部 AI 代理。

5. **服务编排器（`src/services`）** – 协调高级工作流，如检查、修复、同步和视图生成。管理会话生命周期、检查点可恢复性和跨服务集成。

6. **任务执行器（`src/tasks`）** – 实现端到端流水线阶段：扫描、AST 提取、语义摘要生成（通过 LLM）、验证、聚合、反向索引、视图推导、文档回填和清理。

7. **类型系统（`src/types`）** – 定义共享契约，如 `SpineUnit`、`SpineConfig`、清单、规则和视图工件，确保所有模块间的一致性。

8. **测试套件（`tests/`）** – 包括单元测试、集成测试和端到端测试，用于验证 CLI 命令、配置、LLM 集成、同步工作流、错误处理和规则执行。

### 实际工作流

一个典型的 `sync` 命令触发以下流程：
- CLI 解析参数并通过核心编排器启动流水线。
- 扫描器读取忽略模式（来自 `.gitignore` 和 `.spine/rules`）并发现源文件。
- AST 解析器提取每个文件的符号、导入和导出。
- 对于每个文件，调用配置的 LLM 提供商生成语义摘要。
- 架构规则（层隔离、命名约定）根据解析的代码和摘要进行评估。
- 结果存储在 SQLite 数据库中，清单跟踪文件完整性和同步状态。
- 视图（架构图、风险热点、公共表面）可以从收集的数据中渲染。

这个流水线使开发者能够持续监控架构质量，自动生成文档，并通过 MCP 服务器向 AI 代理暴露结构化代码智能。
