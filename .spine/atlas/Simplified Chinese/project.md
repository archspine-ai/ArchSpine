# ArchSpine：语义镜像系统

ArchSpine 是一个语义镜像系统，用于创建和维护软件项目架构的结构化、AI 可读的映射。它通过编排多阶段流水线，扫描源代码、提取结构化元数据、利用大语言模型生成语义摘要、强制执行架构规则，并将所有产物持久化到镜像的 `.spine` 目录中，从而实现自动化分析、治理和文档生成。

## 核心模块及其协作方式

### 1. CLI 层 (`src/cli`)
用户交互界面提供所有操作的命令：`check`（检查）、`fix`（修复）、`sync`（同步）、`publish`（发布）和初始化。它包括命令适配器（`src/cli/commands/`）和引导子系统（`src/cli/init/`）。CLI 将请求委托给服务编排层。

### 2. 扫描与 AST 解析 (`src/ast`)
AST 层发现源文件并使用语言特定的语法定义（`src/ast/rules/`）将其解析为结构化符号。它支持多种语言，提取导入、导出和类型信息。这些原始结构数据输入到核心流水线。

### 3. 核心流水线编排 (`src/core`)
核心层包含 L2 聚合逻辑和流水线编排。它协调从扫描到语义生成再到规则执行的整个流程。它管理兼容性（`src/core/compat/`）并确保各阶段的数据一致性。

### 4. 大语言模型基础设施 (`src/infra/llm`)
该模块提供客户端抽象、配置管理和弹性执行，用于语义生成。它包括提供商实现（`src/infra/llm/providers/`）和重试机制。LLM 层将原始 AST 数据转换为人类可读和 AI 可读的摘要。

### 5. 数据库与持久化 (`src/infra/db`)
基于 SQLite 的持久化层管理索引、状态持久化和数据访问。它包括所有核心实体的仓库（`src/infra/db/repositories/`）。数据库存储元数据、清单和检查点记录。

### 6. 服务编排 (`src/services`)
服务层协调核心流水线：`check`（验证架构）、`fix`（应用修正）、`sync`（更新镜像）和 `publish`（生成文档）。它还管理运行时会话，并包含一个视图层（`src/services/view/`），用于生成架构可视化和分析报告。

### 7. 规则引擎 (`src/engines`)
规则引擎强制执行架构规则，包括层隔离、命名约定和编码标准。它使用存储在 `.spine/rules/` 目录中的规则定义。引擎与检查和修复流水线集成，以验证和纠正违规行为。

### 8. 任务流水线 (`src/tasks`)
流水线任务实现处理核心处理阶段：文件发现、AST 解析、语义生成、规则执行和产物持久化。每个任务都是多阶段流水线中的一个离散步骤。

### 9. 配置与凭据 (`src/infra/config`, `src/infra/credentials`)
配置管理处理系统设置，而凭据存储为 LLM API 密钥和其他敏感数据提供安全的秘密管理。

### 10. MCP 集成 (`src/infra/mcp`)
模型上下文协议层将 ArchSpine 的内部能力与外部 AI 代理桥接，实现自动化分析和治理工作流。

### 11. 模板与提示 (`src/assets/templates`)
地图文档、提示、规则和视图的模板定义为生成的产物提供结构。提示基础设施（`src/infra/prompt/`）构建和组装用于 LLM 交互的提示。

### 12. 镜像目录结构 (`.spine`)
`.spine` 目录镜像项目结构，包含：
- **地图**：多语言架构文档（英语、简体中文、美式英语）
- **索引**：源代码的元数据和索引记录
- **规则**：架构治理规范
- **运行时**：同步运行的执行检查点记录

### 13. 模式定义 (`schemas/`)
JSON 模式定义验证所有元数据单元、清单和规则文档，确保整个系统的一致性。

### 14. 测试套件 (`tests/`)
全面的测试套件涵盖所有核心组件的单元测试、集成测试和端到端测试，包括引擎测试、基础设施测试和辅助脚本。

## 实际工作流程

1. **初始化**：CLI 引导项目，创建带有默认规则和配置的 `.spine` 镜像目录。
2. **扫描**：AST 解析器发现源文件并提取结构化元数据。
3. **语义生成**：LLM 基础设施将原始 AST 数据转换为结构化摘要。
4. **规则执行**：规则引擎根据定义的规则（层隔离、命名约定）验证架构。
5. **同步**：同步流水线用新的产物和检查点记录更新镜像。
6. **发布**：发布流水线生成文档和可视化。
7. **持续集成**：CI/CD 工作流（`.github/workflows/`）在每次提交时自动化这些步骤。

该系统设计用于人类消费（Markdown 文档）和 AI 代理消费（结构化 JSON 元数据），使其成为架构治理和自动化分析的全面解决方案。
