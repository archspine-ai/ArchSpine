# ArchSpine 配置摘要：代码格式化（Prettier）

此配置使用 Prettier 定义项目的代码格式化规则，确保整个代码库风格一致。运维人员应检查这些设置，以配合团队约定和持续集成（CI）期望。

## 当前值

| 参数              | 值      |
|------------------|---------|
| `printWidth`     | 100     |
| `singleQuote`    | `true`  |
| `semi`           | `true`  |
| `trailingComma`  | `"all"` |

## 参数说明

- **`printWidth`**（100）— 指定换行前的最大行长度。较高的值减少换行但可能导致水平滚动。此处设为 100，比默认的 80 更宽。团队应确保编辑器和审查工具能处理较长的行。

- **`singleQuote`**（`true`）— 如果为 `true`，则优先使用单引号而非双引号表示字符串。采用此约定的代码库可减少视觉干扰。与许多 JavaScript/TypeScript 项目一致。

- **`semi`**（`true`）— 如果为 `true`，则在语句末尾添加分号。在混合 JS/TS 项目中促进一致性。关闭此选项可能导致自动分号插入（ASI）引发的错误。

- **`trailingComma`**（`"all"`）— 控制对象/数组字面量中的尾逗号。`"all"` 尽可能添加尾逗号，使版本控制中的差异更清晰（例如，添加新属性时仅改变一行）。值必须为 `"none"`、`"es5"` 或 `"all"` 之一。

## 稳定性与风险

- **风险等级**：低。该配置使用标准的 Prettier 选项和有效值。
- **故障模式**：任何无效值（例如 `trailingComma: "everything"`）将导致 Prettier 报错，阻塞格式化或 CI 流程。
- **运维注意**：
  - 较高的 `printWidth`（100）可能导致代码审查中出现较长行——请确保团队的审查工具和显示器宽度适宜。
  - 将 `semi` 改为 `false` 可能因 ASI 引入细微错误；当前设置是安全的。
  - `trailingComma: "all"` 在现代 JS/TS 项目中广泛采用，风险较低。

## 建议

- 在 CI 中使用 Prettier 的 `--check` 标志验证此文件，以便及早捕获错误配置。
- 如果参与更广泛的生态系统（例如开源项目），请确保与社区约定保持一致。
- 无运行时影响；这纯粹是开发工具设置。
