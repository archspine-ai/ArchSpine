# ArchSpine 代码格式化配置

该配置为整个 ArchSpine 项目强制执行一致且可读的代码风格。通过统一格式化，降低代码审查时的认知负担，确保所有提交遵循相同规范。

## 控制的参数

- **`printWidth`**：设置最大行宽为 100 字符。这平衡了可读性与代码紧凑性，避免过长的行导致阅读困难。
- **`singleQuote`**（启用）：强制使用单引号而非双引号表示字符串，减少视觉杂乱，符合常见 JavaScript 样式偏好。
- **`semi`**（启用）：要求所有语句末尾使用分号，防止因 JavaScript 自动分号插入机制意外行为导致的错误。
- **`trailingComma`**（设为 `"all"`）：在多行数组、对象、函数参数等处启用尾逗号。这简化了版本控制差异，使重新排序或添加元素时不易出错。

## 稳定性和风险

此配置仅为代码格式化标准，不影响运行时行为。然而，代码库内格式化不一致会导致不必要的合并冲突，并在协作开发中降低代码质量。建议通过自动化格式化工具严格强制执行，以避免风格偏离，维护所有贡献者的可读性。
