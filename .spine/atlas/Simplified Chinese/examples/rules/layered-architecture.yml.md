# ArchSpine 架构治理规则

## 目的
本文档为 ArchSpine 项目建立并强制执行严格的分层架构模型，确保高层领域模块保持纯净，不依赖于底层基础设施或表现层。

## 上下文与受众
本文档面向所有参与 ArchSpine 代码库开发的开发者，特别是从事领域逻辑、API 层或基础设施模块的开发者。它作为开发与代码审查过程中维护架构完整性的参考依据。

## 主要职责
- 为 ArchSpine 项目定义分层架构约束
- 指定领域层、基础设施层、API 层和 CLI 层之间的依赖规则
- 强制领域隔离，防止反向依赖

## 不涵盖范围
- 具体模块的实现细节
- 测试策略或测试代码组织
- 部署或基础设施配置

## 关键要点
- 领域模块绝不能导入基础设施或 CLI 层的代码
- 领域层只能依赖内部领域逻辑或通用工具
- API 层只能调用服务层或领域层，不得绕过业务逻辑
