# ArchSpine 分层架构约束

## 目的与范围

本文档旨在为 ArchSpine 镜像系统实施**严格的分层架构**。它将防止反向依赖、强制领域隔离以及确保 API 层仅调用服务层或领域层的规则进行编码化，从而维护经典分层模型的完整性。如果没有这一规范，系统将面临领域逻辑与基础设施或表示层耦合的风险，削弱可维护性和可测试性。

## 目标读者

- 在 ArchSpine 任何模块中编写代码的**开发者**。
- 审查拉取请求或规划系统演进的**架构师**。
- 需要在生成或验证代码前理解架构边界的 **AI 代理**。

## 锚定的关键决策与工作流

- **领域纯净性**：领域模块不得从基础设施层或 CLI 层导入（错误级别）。这一决策确保领域成为纯净、独立可测试的核心。
- **最小领域依赖**：领域层仅可依赖内部领域逻辑或通用工具（警告级别）。这使领域保持专注，不受基础设施纠缠。
- **API 层职责**：API 层必须仅调用服务层或领域层；禁止绕过业务逻辑（错误级别）。这一工作流确保业务逻辑位于领域层，且不会被绕过。

## 关键要点

- 领域模块**不**得从 `src/infra/**` 或 `src/cli/**` 导入。
- 领域模块**应**仅依赖其他领域逻辑或通用工具。
- API 层**必须**仅调用服务层或领域层——绝不能直接调用基础设施层或 CLI 层。

这些规则以错误（阻断）或警告（建议）的严重级别强制执行，构成了 ArchSpine 镜像系统架构治理的基石。
