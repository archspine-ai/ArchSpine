# ArchSpine 元数据索引条目：demo.gif

## 角色
此文件是 ArchSpine 镜像系统中二进制/文档资产（`demo.gif`）的**元数据索引条目**。它不包含可执行代码，而是作为文件身份、语义、结构、依赖图和来源的结构化记录。

## 主要职责
- **身份追踪**：记录文件的路径、内容哈希（SHA-256）和语义哈希，以检测更改或漂移。
- **分类**：声明文件种类（`document`）和语言（二进制文件标记为 `unsupported`）。
- **结构骨架**：捕获导入、导出和声明的符号（非代码文件为空）。
- **依赖图**：维护此文件依赖的其他文件列表以及依赖此文件的其他文件列表，并附带反向索引是否完整的标志。
- **来源记录**：存储索引时间戳、生成器版本以及处理此文件的流水线阶段。

## 重要不变性
- `contentHash` 必须始终与实际文件内容匹配；任何不匹配都表示漂移。
- `semanticHash` 必须与文件记录的职责和角色保持一致。
- `fileKind` 必须准确反映文件类型（例如，非代码资产为 `document`）。
- 如果发现 `contentHash` 或 `semanticHash` 不一致，`driftDetected` 必须设置为 `true`。

## 负面范围（不负责的范围）
此文件**不直接**控制系统行为。它纯粹是一条元数据记录，不影响运行时逻辑、配置或处理流水线。

## 最重要的导出/外部可见行为
- 该文件由 ArchSpine 索引器和其他工具使用，以验证文件完整性、跟踪更改和分析依赖关系。
- `driftDetected` 标志是警告未经授权或意外修改的主要信号。
- `reverseIndexComplete` 标志指示依赖图是否完全解析；不完整的索引可能导致重构期间遗漏影响分析。

## 稳定性与风险
此元数据条目对于变更跟踪和一致性验证至关重要。主要风险是**哈希漂移**：如果 `contentHash` 或 `semanticHash` 变得过时，系统可能错误地认为文件未更改，从而可能掩盖安全或完整性问题。不完整的反向索引（`reverseIndexComplete: false`）可能导致遗漏依赖影响评估。总体而言，该文件通过提供可验证、可审计的记录来促进系统稳定性，但其准确性取决于文件更改后及时重新索引。
