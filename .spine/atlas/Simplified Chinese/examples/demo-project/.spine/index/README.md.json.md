# ArchSpine 元数据索引条目（README.md）

本文档描述了 ArchSpine 镜像系统中文档文件（README.md）的元数据索引条目。该条目作为被动记录，用于追踪文件身份、内容完整性、语义分析结果、依赖关系和溯源信息。

## 角色

元数据索引条目充当单个文档文件的结构化描述符。它同时捕获文件的物理特征（路径、哈希、语言）及其在系统架构中的语义含义。

## 主要职责

- **身份追踪**：记录文件的相对路径、内容哈希（SHA-256）、语义哈希、检测到的语言、文件种类和作用域。
- **完整性验证**：内容哈希必须与索引时的实际文件内容匹配，确保文件未被篡改或损坏。
- **语义分析**：存储文件的角色、职责、不涵盖的关注点、不变约束、变更意图和公共接口。
- **依赖管理**：维护依赖关系图（`dependsOn`/`dependedBy`）和符号级依赖关系，以追踪文件之间的关联。
- **漂移检测**：标记已偏离预期状态的文件，并提供偏离原因的解释。
- **溯源记录**：捕获索引时间戳、生成器版本和处理阶段，确保可追溯性。

## 重要不变约束

- 内容哈希必须始终与索引时的实际文件内容一致。
- 语义哈希必须与文件提取的语义含义保持一致。
- 语言字段必须准确反映检测到的编程语言或文档语言。

## 不涵盖的范围

此元数据条目本身是被动的索引记录，不会直接影响系统稳定性。然而，内容哈希与实际文件内容不一致可能表明文件被篡改或损坏。语义哈希不匹配可能意味着文件含义已更改但未重新索引。

## 导出的行为

元数据条目对外暴露以下顶级字段：

- `schemaVersion`：元数据模式的版本。
- `identity`：文件身份信息（路径、哈希、语言、种类、作用域）。
- `semantic`：语义分析结果（角色、职责、不变约束、变更意图、公共接口、漂移）。
- `skeleton`：结构提示（导入、导出、声明的符号、是否为桶模块或仅类型文件）。
- `graph`：依赖关系图（依赖项、被依赖项、反向索引完整性、符号级依赖关系）。
- `provenance`：索引元数据（时间戳、生成器版本、处理阶段）。

## 稳定性与风险

虽然元数据条目本身是被动的，但它在系统完整性中扮演着关键角色。依赖关系图有助于识别文件修改时的级联影响。漂移检测功能会标记已偏离预期状态的文件，这对维护系统一致性至关重要。溯源信息确保索引生成过程完全可追溯。
