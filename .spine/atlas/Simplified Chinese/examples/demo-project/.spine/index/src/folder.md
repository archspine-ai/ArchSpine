`src` 目录是 ArchSpine 镜像系统中索引层的一部分，负责聚合项目源代码目录树的所有元数据和索引记录。它定义了如何为整个 `src` 树及其三个主要子模块（`api`、`domain`、`infra`）捕获和组织结构模式、溯源信息、依赖图、架构规则、内容完整性校验以及流水线阶段日志。子文件夹 `api`（存储 API 源文件的索引元数据）、`domain`（领域逻辑的元数据）和 `infra`（数据库基础设施的元数据）各自拥有独立的索引记录；配置文件 `folder.json` 则提供了整个源树的结构层级和元数据。重要的实现领域包括跟踪索引溯源（时间戳和生成器版本）以及记录生成元数据的流水线阶段（`ast`、`llm`）。这些具体的子模块共同支持对源代码进行高效的搜索、依赖分析和规则执行。
