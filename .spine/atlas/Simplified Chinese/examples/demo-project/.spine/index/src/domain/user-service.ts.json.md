# ArchSpine 文件元数据与语义索引条目

## 角色
此文件作为 ArchSpine 镜像系统中单个 TypeScript 源文件的**元数据和语义索引条目**。它是一个类似配置的记录，捕获所描述源文件的身份、结构、语义和依赖图。

## 主要职责
- **身份追踪**：维护文件路径、内容哈希、骨架哈希和语义哈希，以在多个粒度上检测变更。
- **语义声明**：记录文件的功能角色、职责、不变量和范围外边界，供人类和 AI 理解。
- **结构骨架**：存储导入、导出、声明的符号以及结构提示（例如是否为桶文件、是否仅包含类型定义）。
- **依赖图**：跟踪前向依赖（`dependsOn`）和反向依赖（`dependedBy`），包括符号级依赖边。
- **来源与管道**：记录索引时间戳、生成器版本和管道阶段，用于缓存失效和审计。

## 重要不变量
- `schemaVersion` 必须是有效的 semver 字符串。
- `identity.contentHash` 必须与实际文件内容匹配。
- `identity.skeletonHash` 必须与解析后的 AST 骨架匹配。
- `identity.semanticHash` 必须与语义分析输出匹配。
- `filePath` 必须相对于项目根目录。
- `driftDetected` 必须为 `false`，除非提供了 `driftReason`。

## 负面范围（范围外）
此文件**不**控制运行时行为、执行业务逻辑或直接影响应用程序状态。它纯粹是一个用于分析和追踪目的的元数据索引。

## 最重要的导出/外部可见行为
- **基于哈希的变更检测**：三个哈希（内容、骨架、语义）使系统无需完全重新索引即可检测变更。
- **依赖图遍历**：`dependsOn` 和 `dependedBy` 字段支持影响分析和架构漂移检测。
- **不变量强制执行**：不变量作为契约，必须成立索引才有效；违反将触发调查。
- **来源追踪**：`indexedAt` 和 `generatorVersion` 字段确保缓存一致性和工具兼容性。

## 稳定性与风险
此文件对索引管道的完整性至关重要。风险包括：
- 哈希不匹配导致错误的漂移检测或遗漏变更。
- 不完整的反向索引导致不准确的影响分析。
- 过时的来源时间戳导致缓存不一致。
- 缺失或不正确的不变量允许不安全的代码变更未被检测到。

该文件的稳定性直接影响整个项目的依赖分析、变更影响评估和架构漂移检测的可靠性。
