# ArchSpine 目录元数据配置

## 角色
定义 ArchSpine 镜像系统中源代码目录的结构元数据和索引来源信息。

## 主要职责
- 声明目录路径及其在项目中的功能角色
- 列出子源文件及其角色和文件类型
- 记录索引来源信息，包括时间戳、生成器版本和管道阶段

## 重要不变量
- `schemaVersion` 必须是有效的语义化版本号字符串
- `directory` 必须是非空字符串，表示有效的项目路径
- `provenance.indexedAt` 必须是有效的 ISO 8601 时间戳
- `provenance.generatorVersion` 必须遵循 `archspine/<semver>` 格式

## 负面范围
此配置文件是单个源代码目录的元数据描述符，不直接控制运行时行为或系统安全性。不正确或过时的来源数据可能导致镜像系统对代码库的理解与实际源代码树不一致，从而可能引发索引错误、遗漏依赖关系或角色分配错误。

## 导出的行为
该配置由 ArchSpine 镜像系统使用，以了解源代码目录的结构和用途。它通过来源元数据提供可追溯性，但不强制执行操作约束。系统应验证目录和文件路径在仓库中实际存在，以防止静默失败。

## 参数定义
- **schemaVersion**：指定此配置文件所使用的模式版本，确保与解析工具的兼容性。
- **directory**：所描述目录的相对路径，镜像系统使用该路径来定位和组织源代码。
- **role**：描述目录或文件在项目架构中功能用途的可读标签，便于系统理解和文档化。
- **responsibility**：简要描述分配给此目录的职责，有助于系统理解和文档编写。
- **children**：子文件条目数组，每个条目包含文件路径、角色和文件类型，支持对源代码树的递归遍历。
- **provenance**：记录此配置生成时间和方式的元数据块，对审计追踪和可重现性至关重要。
- **provenance.indexedAt**：目录上次被索引的时间戳，用于检测过期或过时的配置。
- **provenance.generatorVersion**：标识生成此文件的 ArchSpine 生成器版本，对调试和兼容性检查很重要。
- **provenance.pipelineStages**：列出生成此配置所经过的处理阶段（如 AST 解析、LLM 分析），提供索引管道的透明度。

## 稳定性与风险
此配置文件是单个源代码目录的元数据描述符，不直接控制运行时行为或系统安全性。然而，不正确或过时的来源数据可能导致镜像系统对代码库的理解与实际源代码树不一致，从而可能引发索引错误、遗漏依赖关系或角色分配错误。不变量确保了模式的一致性，但系统应验证目录和文件路径在仓库中实际存在，以防止静默失败。pipelineStages 字段提供了可追溯性，但不强制执行任何操作约束。
