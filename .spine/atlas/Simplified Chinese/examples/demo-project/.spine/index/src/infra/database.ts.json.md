# ArchSpine 元数据索引 – 源文件摘要

此文件是 ArchSpine 镜像系统中单个 TypeScript 源文件的**元数据和结构索引**。它不包含运行时逻辑，而是记录所描述文件的身份、内容完整性、语义注释、依赖关系图和来源信息。

## 主要职责

- **文件身份与完整性** – 跟踪文件的路径、内容哈希、骨架哈希和语义哈希，以检测漂移并确保一致性。
- **语义声明** – 存储文件的角色、职责、不变条件和排除范围，提供人类和机器可读的契约。
- **依赖关系图** – 记录此文件依赖的其他文件（`dependsOn`）以及依赖此文件的其他文件（`dependedBy`），并包含反向索引完整性标志。
- **来源信息** – 捕获索引时间戳、生成器版本和流水线阶段，便于审计。

## 重要不变条件

- 文件路径必须唯一，并与项目的目录结构匹配。
- 每次文件更改后必须重新计算内容哈希，以检测漂移。
- 语义角色必须准确反映文件的实际用途。

## 排除范围（负面范围）

此索引文件**不**控制运行时行为、执行业务逻辑或修改源代码。它纯粹是一个元数据快照。

## 导出 / 公共表面

此文件没有公共 API 或导出的符号。它由 ArchSpine 索引流水线内部使用。

## 稳定性与风险

- **过时的哈希值**可能导致漂移未被检测到。
- **不完整的反向索引**会导致依赖分析不准确。
- **不匹配的语义注释**会错误描述文件的角色。
- 系统稳定性依赖于每次文件修改后重新生成索引。
