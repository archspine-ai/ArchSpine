# ArchSpine 源代码目录配置

## 角色
定义 ArchSpine 项目中源代码目录树的结构层次和元数据。

## 主要职责
- 将源文件夹（src、src/api、src/domain、src/infra）映射到其角色和职责
- 提供用于索引和生成流水线跟踪的来源元数据

## 重要不变条件
- `directory` 字段必须指向一个存在的根源文件夹
- 每个子条目必须具有有效的 `filePath` 和 `fileKind`
- 来源 `indexedAt` 必须是有效的 ISO 8601 时间戳

## 负面范围 / 超出范围
此文件未定义任何超出范围的行为。

## 导出 / 外部可见行为
此文件不导出任何函数、类或公共接口。它是一个静态配置文件，由 ArchSpine 索引和生成流水线使用。

## 参数定义
- **schemaVersion**：用于验证的配置模式版本。
- **directory**：源代码树的根目录路径。
- **role**：文件夹功能的描述。
- **responsibility**：分配给该文件夹的具体职责。
- **children**：子目录或文件列表，包含各自的角色和类型。
- **filePath**：文件或文件夹的相对路径。
- **fileKind**：条目的类型（例如 'folder'）。
- **provenance**：关于此配置生成时间和方式的元数据。
- **indexedAt**：目录被索引的时间戳。
- **generatorVersion**：生成此配置的工具版本。
- **pipelineStages**：生成流水线的阶段（例如 'ast', 'llm'）。

## 稳定性和风险
此文件定义了源代码树的逻辑结构。配置错误（例如错误的 filePath 或缺少子项）可能导致下游工具导航中断或角色分配错误。来源元数据对于可审计性至关重要；过时的时间戳可能表示索引已过期。总体而言，此文件操作风险较低，但对结构一致性至关重要。
