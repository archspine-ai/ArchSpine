# ArchSpine 领域源元数据

## 角色
定义 ArchSpine 镜像系统中领域源文件夹的结构元数据和索引来源信息。

## 主要职责
- 声明领域文件夹的目录路径和功能角色
- 列出子源文件及其角色和类型
- 记录索引来源信息，包括时间戳、生成器版本和流水线阶段

## 重要不变约束
- `schemaVersion` 必须是有效的语义化版本号字符串
- `directory` 字段必须是非空相对路径
- 每个子条目必须包含 `filePath` 和 `fileKind`
- `provenance.indexedAt` 必须是有效的 ISO 8601 时间戳

## 排除范围 / 非职责范围
此文件不包含运行时逻辑、可执行代码或行为定义。它纯粹是一个元数据描述符。

## 导出 / 外部可见行为
此文件由 ArchSpine 镜像系统消费，用于理解领域文件夹的结构和来源信息。它不导出任何函数、类或运行时 API。其主要外部作用是影响系统如何索引、定位和组织领域内的源文件。

## 参数定义
- **schemaVersion**：配置模式的版本号；用于确保解析器兼容性。
- **directory**：所描述领域文件夹的相对路径；影响系统定位和组织源文件的方式。
- **role**：文件夹的功能标签；系统据此进行分类和策略应用。
- **responsibility**：文件夹用途的简要说明；有助于文档编写和审计。
- **children**：子文件条目数组；每个条目指定源文件路径、角色和文件类型（例如 'source'）。
- **provenance**：记录此配置生成时间和方式的元数据块，包括索引流水线阶段（如 'ast'、'llm'）。

## 稳定性与风险
此文件为元数据描述符，无直接运行时影响。但错误的目录路径或缺失的子条目可能导致镜像系统错误索引或无法定位领域文件，从而引发分析不完整或引用断裂。来源信息块对于审计追踪和调试索引问题至关重要。模式版本不匹配可能导致解析失败。
