# ArchSpine 源代码目录元数据

## 角色
定义 ArchSpine 镜像系统中源代码目录的结构元数据和索引来源信息。

## 主要职责
- 声明目录路径及其关联的角色和职责描述
- 列出子源文件及其各自的角色
- 记录索引来源信息，包括时间戳、生成器版本和流水线阶段

## 重要不变约束
- `schemaVersion` 必须是有效的语义化版本号字符串
- `directory` 必须是非空字符串，表示相对路径
- 每个子条目中的 `filePath` 在文件内必须唯一
- `provenance.indexedAt` 必须是有效的 ISO 8601 时间戳
- `provenance.generatorVersion` 必须遵循 `archspine/<version>` 格式

## 负面范围
此文件不直接控制运行时行为。它纯粹是用于索引和搜索目的的元数据描述符。

## 导出的行为
该文件暴露一个描述源代码目录的结构化配置对象。ArchSpine 镜像系统使用它来理解代码目录的布局、用途和索引历史。

## 参数定义
- **schemaVersion**：配置模式的版本号，用于确保向前/向后兼容。
- **directory**：被描述的源代码目录的相对路径。
- **role**：对该目录功能的高级描述。
- **responsibility**：该目录在系统中的职责概述。
- **children**：子文件条目数组，每个条目指定文件路径及其角色。
- **filePath**：目录内源代码文件的相对路径。
- **fileKind**：文件类型（例如 'source'、'test'、'config'）。
- **provenance**：关于此配置生成时间和方式的元数据。
- **indexedAt**：目录被索引时的 ISO 8601 时间戳。
- **generatorVersion**：生成此配置的工具版本标识符。
- **pipelineStages**：索引过程中使用的处理阶段的有序列表（例如 'ast'、'llm'）。

## 稳定性和风险
此文件是源代码目录的元数据描述符。它不直接控制运行时行为，但对于镜像系统的索引和搜索功能至关重要。过时或不准确的来源数据可能导致索引结果不匹配。只要遵守不变约束，修改此文件是安全的；但是，如果删除或更改子条目而未更新实际的源代码树，可能会导致镜像对代码库的表示出现不一致。
