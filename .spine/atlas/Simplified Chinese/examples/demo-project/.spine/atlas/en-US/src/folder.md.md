# ArchSpine `src` 目录概述

本文档提供 `src` 目录的高级概述，说明其作为 ArchSpine 项目核心源代码容器的角色，并总结其主要子目录的职责。

## 上下文与目标读者

适用于需要快速了解 `src` 目录组织结构和用途的开发者及项目维护者，无需深入查看具体文件。

## 关键要点

- `src` 目录是项目源代码的主要存放位置。
- 它分为三个主要子目录：`api`、`domain` 和 `infra`。
- 每个子目录在项目架构中都有明确的角色。

## 职责

- 记录 `src` 目录的角色和职责
- 概述子目录（`api`、`domain`、`infra`）的结构

## 不涉及范围

- 详细的实现细节
- 代码级文档或 API 参考
