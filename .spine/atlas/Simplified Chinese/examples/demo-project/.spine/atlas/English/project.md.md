# ArchSpine 演示项目

## 目的
本文档是 ArchSpine 协议的最小演示项目，旨在展示如何在本地仓库中管理架构规则和语义上下文。

## 背景与受众
适用于评估 ArchSpine 能力的开发者和架构师，特别是对领域驱动设计实施和架构边界验证感兴趣的人员。

## 关键要点
- 演示了使用 DDD 原则将领域逻辑与基础设施分离
- 包含故意设计的违规案例以测试 ArchSpine 的规则检查能力
- 展示了 .spine/ 目录如何在本地存储语义上下文

## 范围
本项目通过一个最小示例，演示和验证 ArchSpine 协议的架构治理能力。它**不**适用于生产环境，不代表完整的系统架构，也不包含部署或运维流程。

## 职责
- 展示领域逻辑与基础设施之间的领域驱动设计分离
- 通过故意设计的架构违规来演示 ArchSpine 的边界强制能力
- 展示 .spine/ 目录中的本地控制平面和语义上下文存储

## 排除范围
- 生产就绪的应用逻辑
- 完整的系统架构文档
- 部署或运维流程

## 不变约束
无定义。

## 公开接口
本演示项目未定义任何公开 API 或导出项。
