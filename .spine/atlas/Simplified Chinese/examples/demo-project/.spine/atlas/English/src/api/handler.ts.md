# API 请求处理器：CreateUserHandler

## 目的

本文档定义了作为 ArchSpine 镜像系统入口点的 API 请求处理器。它通过协调领域服务和管理传入的请求负载来编排用户创建流程。本文档旨在帮助读者清晰理解系统的 API 边界、请求处理流程以及已知的架构问题。

## 目标读者

本文档面向需要了解系统 API 边界、请求处理流程以及已知架构问题的开发者和系统架构师。特别适用于参与用户创建功能开发或维护处理器层的相关人员。

## 关键要点

- `CreateUserHandler` 类是用户创建请求的主要入口点
- 存在一个已知的架构违规问题：对 `infra/database` 的直接依赖
- 该处理器充当外部调用者与内部领域逻辑之间的边界

## 本文档锚定的决策与工作流程

- **API 边界定义**：本文档锚定了将处理器作为用户创建唯一入口点的决策，确保所有外部请求通过一致的接口传递。
- **架构违规追踪**：已知的对 `infra/database` 的依赖在此文档中记录，锚定了识别和解决架构问题的工作流程。
- **处理器生命周期管理**：本文档支持维护和演进处理器层的工作流程，包括对请求处理或领域服务协调的变更。
