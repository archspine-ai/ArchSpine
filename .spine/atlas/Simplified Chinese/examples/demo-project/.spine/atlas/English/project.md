# ArchSpine 演示：协议展示与验证

## 本文档的目的

本文档介绍 **archspine-demo** 项目——一个最小化的具体实现，旨在展示 ArchSpine 协议的核心功能。它的存在是为了弥合抽象协议概念与实际理解之间的差距，为理解 ArchSpine 如何在本地仓库中强制执行架构边界和管理语义上下文提供实践参考。

## 目标读者

- **开发者**：希望看到 ArchSpine 规则执行的实际示例
- **架构师**：评估 ArchSpine 如何支持领域驱动设计模式和边界强制
- **技术评估人员**：希望在全面采用之前，先看到一个小规模的具体实现

## 本文档锚定的关键决策与工作流程

### 架构决策
- **领域驱动设计分离**：项目明确将领域逻辑（`src/domain`）与基础设施（`src/infra`）分离，展示了清晰的架构分层
- **有意违规**：演示中包含故意的架构违规（例如 API 直接调用 Infra），以验证 ArchSpine 的规则检查能力
- **本地控制平面**：`.spine/` 目录在本地存储语义上下文，展示了 ArchSpine 如何在不依赖外部服务的情况下维护架构意识

### 演示的工作流程
1. **规则检查**：ArchSpine 如何检测并报告架构违规
2. **语义上下文管理**：本地控制平面如何维护和利用架构知识
3. **边界强制**：ArchSpine 如何确保领域逻辑与基础设施关注点保持隔离

## 关键要点

- 该项目使用领域驱动设计原则将领域逻辑与基础设施分离
- 项目中包含了有意的架构违规，以演示 ArchSpine 的规则检查能力
- `.spine/` 目录在本地存储语义上下文，展示了本地控制平面的概念
