# ArchSpine API 处理程序：入口点与请求协调器

## 目的
本文档定义了 ArchSpine 镜像系统的 API 请求处理程序与入口点，描述了如何通过领域服务协调传入的用户创建请求，并建立了外部调用者与内部逻辑之间的边界。

## 目标读者
本文档面向需要理解 API 层在请求处理中角色的开发者和系统架构师，包括其当前的架构依赖关系以及关于直接基础设施耦合的已标记违规。

## 关键决策与工作流程
- `CreateUserHandler` 类管理传入的请求负载，并通过领域服务协调用户创建。
- 存在对 `infra/database` 的直接依赖，并已在注释中被标记为架构违规。这是一个已知问题，应予以解决以保持清晰的架构边界。
- 该文件充当外部调用者与内部系统逻辑之间的主要边界，确保关注点分离。

## 不涉及范围
- 领域业务逻辑实现
- 数据库基础设施细节
- 身份验证或授权相关事项
