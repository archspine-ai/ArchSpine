# ArchSpine 演示项目：项目概览与架构展示

## 目的

本文档提供 **archspine-demo** 项目的高层概述，该项目是 ArchSpine 协议的一个极简演示。它解释了项目的架构理念和关键设计决策，展示了 ArchSpine 如何在真实代码库中强制执行架构边界。

## 目标读者

- **开发者和架构师**：希望了解 ArchSpine 如何在真实项目中强制执行架构边界。
- **正在评估 ArchSpine 协议的人**：通过具体示例了解其能力，判断是否适合自己的项目。

## 关键要点

- **领域驱动设计实践**：该项目通过将领域逻辑（`src/domain`）与基础设施（`src/infra`）分离，展示了领域驱动设计。
- **边界强化**：项目中包含了故意的架构违规（例如 API 直接调用 Infra），用以展示 ArchSpine 的规则检查能力。
- **本地控制面**：`.spine/` 目录在本地存储语义上下文，充当项目的本地控制面。

## 本文档锚定的决策与工作流

- **架构决策**：领域层与基础设施层的分离是核心设计选择，所有后续开发必须遵循。
- **规则检查工作流**：包含的违规案例作为 ArchSpine 验证规则的测试用例，锚定了测试和持续集成流程。
- **本地上下文管理**：`.spine/` 目录模式被确立为存储项目级语义元数据的标准方式。
