# ArchSpine – API 请求处理程序 (handler.ts)

## 目的
本文档描述了 API 请求处理程序模块（`handler.ts`），该模块作为处理传入用户创建请求的入口点。它定义了 `CreateUserHandler` 类，通过与领域服务协作来处理请求负载，同时指出了当前因直接依赖 `infra/database` 模块而存在的架构违规问题。

## 上下文与受众
本文档面向负责系统 API 层的开发者和维护者。它提供了处理程序作为外部调用者与内部逻辑之间边界的高层概述，并强调了需要关注的已知架构问题。

## 关键要点
- 处理程序通过委托给领域服务来协调用户创建流程
- 对 `infra/database` 的直接依赖被标记为架构违规
- 处理程序充当外部调用者与内部系统逻辑之间的主要边界

## 文件角色
该文件作为用户创建请求的 **API 请求处理程序和入口点**。它定义了 `CreateUserHandler` 类，负责管理传入的请求负载，并与领域服务协作进行处理。

## 主要职责
- 通过与领域服务交互来协调用户创建请求
- 定义 `CreateUserHandler` 类以管理传入的请求负载
- 充当外部调用者与内部系统逻辑之间的主要边界

## 重要不变项与负面范围
- **不在范围内：** 直接数据库操作或持久化逻辑；领域特定的业务规则或验证
- **不变项：** 当前未定义任何不变项

## 最重要的导出或外部可见行为
`CreateUserHandler` 类是主要的导出实体。它接收请求负载，将处理委托给领域服务，并将结果返回给调用者。存在一个已知的架构漂移：处理程序当前直接依赖 `infra/database` 模块，这违反了预期的关注点分离原则。
