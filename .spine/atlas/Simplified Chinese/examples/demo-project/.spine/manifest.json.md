# ArchSpine 镜像同步快照 — 配置摘要

## 用途
该文件是 ArchSpine 镜像系统的**健康检查与同步状态快照**。它提供了一个易读的视图，展示最近同步活动的摘要及系统健康指标。权威状态存储在 `.spine/cache.db` 中——此文件**并非**主要状态来源。

## 关键字段及其含义

| 字段 | 描述 | 运维说明 |
|------|------|----------|
| `lastSyncAt` | 最近一次同步的 ISO 时间戳 | 用于监控同步是否过时；核心状态另有存储。 |
| `lastSyncMode` | 标明“全量”或“增量” | 若长时间未执行全量同步，下次同步的成本和耗时可能增加。 |
| `reverseIndexComplete` | 布尔标志，确认反向索引是否构建成功 | 若为 `false`，则搜索和查询功能可能不完整。 |
| `indexedUnitCount` | 上次同步处理的单元数量（如文件、文档） | 数值剧烈变化可能暗示配置错误或数据丢失。 |
| `activeViolations` | 上次同步结束时检测到的活跃规则违规数 | **零值为健康**。持续非零会降低系统稳定性，需及时排查。 |
| `lastSyncDurationMs` | 上次同步消耗的毫秒数 | 异常飙升可能意味着性能瓶颈或资源争用。 |
| `llm.provider` | 同步时使用的 LLM 提供者（例如 "mock"） | 影响同步时调用的 AI 服务。 |
| `llm.model` | 使用的模型（例如 "deepseek-chat"） | 必须与配置中可用模型匹配；不匹配会导致静默失败。 |
| `llm.providerSource` / `llm.modelSource` | 提供者/模型设置的来源（"project-config" 或 "global-config"） | 若未相应更新全局或项目配置而更改提供者或模型，会导致静默失败。 |

## 稳定性与风险

- **快照本质**：此文件是一个缓存视图，**不要**将其作为需要权威状态的决策依据——请始终参考 `.spine/cache.db`。
- **过时风险**：若快照过时（例如长时间未同步），监控面板和告警规则可能依赖过时信息，导致错误的健康结论。
- **治理漂移**：`activeViolations` 持续高值表明治理漂移，可能引发数据完整性级联问题。
- **LLM 配置**：此处嵌入的 LLM 提供者和模型设置必须与实际的项目或全局配置一致。不一致会导致同步时 AI 调用静默失败。
- **健康系统模式**：持续低违规、耗时合理的同步表示系统健康。任何持续偏离都应调查。

## 需要强制的不变规则

- `activeViolations` 必须为非负整数。零值表示状态清洁。
- `lastSyncMode` 必须为 `"full"` 或 `"incremental"`——不接受其他值。
- 该文件是缓存，绝不可用作主要状态来源。
