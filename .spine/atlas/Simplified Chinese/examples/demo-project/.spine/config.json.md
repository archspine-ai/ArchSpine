# ArchSpine 配置摘要

本配置文件定义了 archspine-demo 项目在扫描、钩子以及大语言模型（LLM）提供商方面的行为。以下说明将帮助运维人员理解各参数的作用、关键点以及潜在风险。

## 扫描策略
- **文件来源**：仅扫描 Git 追踪的文件，避免临时文件或生成文件干扰分析。
- **忽略链**：继承 Git 的忽略规则，同时包含项目级 `.spineignore` 和本地 `.spineignore.local`。这意味着必须谨慎维护这些文件，以防止无意中排除敏感或不相关的文件。
- **协议排除与包含**：整个 `.spine/` 目录被排除在扫描之外，但 `.spine/rules/` 和 `.spine/config.json` 例外。这既保护了 ArchSpine 的内部状态，又允许分析规则文件和配置。
- **运维提示**：如果向 `.spineignore.local` 添加自定义模式，请注意它会与项目忽略设置合并。配置错误可能静默排除重要文件。

## 钩子
- **预提交（preCommit）**：已禁用（`false`）。提交时不会自动进行验证。这减少了开发阶段的摩擦，但可能导致不一致状态在提交时未被发现。如果后续启用钩子，请确保提交流程能处理验证开销。
- **同步模式（syncMode）**：设为 `"hook"`，控制系统的同步方式；由于钩子已关闭，该模式当前未生效。

## 大语言模型（LLM）提供商
- **provider**：`"mock"`。所有 LLM 调用均为模拟。这消除了 API 费用、延迟和外部服务依赖。但不会执行真正的 AI 分析。仅适用于测试或演示；生产环境请切换为真实提供商。

## 模型上下文协议（MCP）
- **contextMode**：`"off"`。外部上下文推断已禁用。这阻止了 ArchSpine 引入外部上下文，虽更安全但限制了功能。如需更丰富的分析，需启用 MCP 并配置合适的上下文来源。

## 稳定性与风险
- **低风险**：扫描策略具有防御性——`.spine/` 受到良好保护，Git 追踪提供了可靠的文件基础。
- **无外部依赖**：使用模拟 LLM 并禁用 MCP，意味着没有网络调用，环境稳定且可预测。
- **验证缺口**：钩子禁用后，提交不会经过验证。建议在项目成熟后启用钩子，或在 CI 流程中添加独立的检查步骤。
- **忽略文件维护**：`.spineignore` 和 `.spineignore.local` 至关重要。误配置可能导致扫描中断或泄露不应包含的文件。请定期审查。

总体而言，本配置优先考虑安全性和简单性，适合演示、早期开发或需要完全控制且无需外部服务的环境。
