# ArchSpine 文件扩展名与语言支持配置

该配置文件用于控制 ArchSpine 系统在扫描和索引过程中识别的文件扩展名，并定义可用于 AST 提取和规则执行的语言。这些设置直接影响文件处理的范围以及分析的准确性。

## 关键参数

- **`schemaVersion`**：必须设置为 `1` 才能与当前处理管道兼容。更改此值可能导致配置文件无法被正确读取。
- **`detectedExtensions`**：系统将扫描和分析的文件扩展名列表。当前包括 `.gif`、`.json`、`.md`、`.ts`、`.yml`。任何未包含在此列表中的扩展名将被忽略。添加或移除扩展名会改变处理范围；缺少预期的扩展名（例如 TypeScript React 文件的 `.tsx`）将导致索引不完整。
- **`languages.typescript.extensions`**：映射到 TypeScript 语言分析的文件扩展名。当前仅定义了 `.ts`。如果项目中使用了 `.tsx` 文件，必须在此处添加该扩展名，以确保它们能够按照 TypeScript 规则被处理。
- **`languages.typescript.status`**：必须设置为 `"available"` 才能使 TypeScript 分析生效。任何其他值（例如 `"disabled"`）将跳过 TypeScript 文件的 AST 提取和规则执行。

## 稳定性与操作风险

- **扫描不完整**：如果 `detectedExtensions` 未包含仓库中存在的所有文件扩展名，这些文件将被静默忽略。这可能导致分析出现缺口，使架构违规未被发现。
- **性能影响**：添加过多的扩展名（尤其是大型二进制文件）会拖慢扫描和索引速度。建议仅添加常见的源代码扩展名。
- **语言分析被禁用**：意外将某语言的 `status` 更改为非 `"available"` 的值将完全禁用其分析，可能导致架构约束失效。
- **运行时崩溃**：添加不支持或格式错误的扩展名可能引发扫描器错误。务必确保扩展名与项目中有效的文件类型对应。

为确保系统稳定性，请使此配置与项目的实际文件类型和语言支持需求保持严格一致。在部署前仔细审查所有更改。
