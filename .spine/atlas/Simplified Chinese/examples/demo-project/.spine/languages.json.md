# ArchSpine 镜像配置摘要

## 概述
本配置声明了 ArchSpine 镜像系统识别的文件扩展名及支持的编程语言。它决定了文件的索引方式、处理逻辑以及哪些语言特定功能被启用。

## 参数定义

- **`schemaVersion`**（整数，当前为 `1`）：定义配置模式的版本。版本 1 是唯一有效的版本，任何其他值将导致解析失败。
- **`detectedExtensions`**（字符串数组）：系统主动监控和索引的文件扩展名列表。当前包括：`.gif`、`.json`、`.md`、`.ts`、`.yml`。未列入此列表的扩展名将被忽略。
- **`languages`**（对象）：编程语言的映射，包含关联的扩展名和可用状态。此参数控制哪些语言的工具链和分析功能被激活。
  - **`languages.typescript`**：
    - **`extensions`**（数组）：为 TypeScript 注册的文件扩展名 — 仅 `.ts`。该设置控制 TypeScript 源文件的检测。
    - **`status`**（字符串）：当前为 `"available"`，表示 TypeScript 支持已启用。若改为 `"unavailable"`，将禁用所有与 TypeScript 相关的功能。

## 稳定性与操作风险

- **扩展名准确性**：`detectedExtensions` 列表不完整或不正确可能导致文件在索引时被遗漏或分类错误，进而引发下游处理异常。
- **语言状态不匹配**：如果 `status` 设置错误（例如，实际未实现支持时却设为 `"available"`），系统可能尝试使用不存在的工具，导致运行时故障。相反，若所需支持被设为 `"unavailable"`，则会禁用必需功能。
- **模式版本锁定**：模式版本固定为 `1`，请勿修改此值。任何偏离都将导致配置无法被解析。
