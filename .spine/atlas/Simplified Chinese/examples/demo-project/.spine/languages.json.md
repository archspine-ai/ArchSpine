# ArchSpine 语言与扩展名注册表

## 角色
此文件作为 **ArchSpine 镜像系统的语言与扩展名注册表**。它声明系统可识别的文件扩展名，将编程语言映射到其关联的扩展名，并跟踪每个语言处理器的可用状态。

## 主要职责
- 声明系统将会处理的文件扩展名集合。
- 将每个编程语言标识符映射到其文件扩展名。
- 跟踪每个语言处理器是否已准备好使用（例如 `available`）。

## 重要不变性
- `schemaVersion` 必须为 `1`，这是当前配置格式的要求。
- 每个语言条目必须包含非空的 `extensions` 数组。
- 每个语言条目必须包含有效的 `status` 字段。

## 负面范围（不涉及）
- 此文件 **不** 定义任何语言的处理逻辑、转换规则或分析行为。
- 它 **不** 验证语言处理器的实际存在或正确性，仅记录其声明的状态。

## 导出的外部可见行为
- 系统读取此注册表以确定要发现和处理的文件类型。
- 如果某个语言标记为 `available` 但其处理器缺失或配置错误，系统可能无法分析或转换该类型的文件。
- `detectedExtensions` 中缺少扩展名可能导致对应文件被静默忽略。
- 若未同步更新模式而擅自修改 `schemaVersion`，将导致配置加载失败。

## 参数定义
- **schemaVersion**：定义配置模式的版本号，当前固定为 `1`。
- **detectedExtensions**：系统已发现并将会处理的所有文件扩展名列表。
- **languages**：编程语言标识符到其配置对象的映射。
  - **languages.typescript.extensions**：与 TypeScript 关联的文件扩展名，当前仅为 `.ts`。
  - **languages.typescript.status**：指示 TypeScript 语言处理器是否已准备好使用。值为 `available` 表示可正常运行。

## 稳定性与风险
此文件控制系统能够识别和处理哪些文件类型。如果某个语言标记为 `available` 但其处理器缺失或配置错误，系统可能无法分析或转换该类型的文件。`detectedExtensions` 中缺少扩展名可能导致对应文件被静默忽略。`schemaVersion` 的不变性确保向后兼容性，若未同步更新模式而擅自修改版本号，将导致配置加载失败。
