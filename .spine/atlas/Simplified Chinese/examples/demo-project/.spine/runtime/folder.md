`checkpoints` 目录位于项目 ArchSpine 运行时的 `.spine/runtime/checkpoints` 路径下，用于存放同步流水线的执行检查点记录。其主要职责是追踪同步命令运行从启动到完成的完整生命周期，记录每个阶段的完成时间戳以及单项级别的完成状态。这为中断运行的可靠恢复提供了基础，同时也形成了权威的审计追踪。

该目录包含一个或多个检查点文件，每个文件对应一次同步运行。重要的子项包括按运行 ID 或时间戳命名的检查点文件，由检查点管理器等内部模块负责读写。最关键的实现领域包括：检查点的创建（在每一阶段记录状态）、检查点的读取（加载状态以恢复中断的运行）以及检查点的清理（在运行成功完成或超过保留期后移除过期记录）。涉及的具体子模块有：检查点管理器（负责所有检查点的输入/输出操作）和同步运行器（通过检查点判断恢复点）。
