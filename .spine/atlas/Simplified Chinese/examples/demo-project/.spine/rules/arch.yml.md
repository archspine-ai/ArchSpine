# ArchSpine 演示项目：架构规则定义

本文档为 ArchSpine 镜像系统下的演示项目确立具体的架构边界与文档规范，确保人工与自动化工具（如 lint 工具、AI 代理）在修改代码时一致地遵守这些约束。

## 面向读者
- 在 `src/api/` 或 `src/domain/` 中修改代码的项目开发者。
- 执行代码变更、审查或生成架构上下文的 AI 代理。
- 负责验证项目架构约定合规性的任何团队成员或工具。

## 核心决策与工作流
- **层隔离规则**（严重级别：错误）—— API 处理程序（`src/api/**`）可以导入 `src/domain/**`，但**禁止**直接导入 `src/infra/**`。这使 API 层独立于基础设施细节，并强制使用领域层作为中间层。
- **文档规则**（严重级别：警告）—— `src/domain/**` 中的所有公开领域类和方法必须包含 TSDoc 注释。这确保生成的架构上下文中领域合约保持可读性。

## 要点总结
- API 处理程序不得直接导入 `src/infra/`；应通过领域层进行中转。
- 所有公开的领域类和方法必须包含 TSDoc 注释。
- 违反层隔离规则视为错误；缺少文档则作为警告处理。
