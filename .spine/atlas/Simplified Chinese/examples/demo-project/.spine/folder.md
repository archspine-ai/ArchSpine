## `.spine` — 根配置与状态管理目录

该目录是 ArchSpine 镜像系统的核心中枢，集中存放运行同步周期、执行架构规则以及诊断同步过程所需的全部设置与记录。

### 目录结构概览

| 路径 | 用途 |
|------|------|
| `config.json` | 系统级配置：扫描策略、预提交钩子、同步模式、LLM 提供商选择、MCP 上下文模式。 |
| `languages.json` | 文件扩展名与编程语言的映射表（例如 `.ts` → TypeScript），用于文件类型检测和语言感知处理。 |
| `manifest.json` | 人类可读的健康检查与同步状态快照。记录上次同步的时间戳、模式（全量/增量）、完成状态、索引单元总数、所使用的 LLM 提供商/模型、活跃违规数，以及同步耗时。权威状态存储在 `.spine/cache.db` 中。 |
| `rules/` | 架构规则定义：**层间隔离**（例如 API 处理器可以导入 `domain` 但不得导入 `infrastructure`）和**文档标准**（要求公开的领域类和方法必须有 TSDoc 注释）。 |
| `runtime/` | 每次同步运行的执行日志：流水线各阶段的状态与耗时、已处理文件的列表（用于追溯）以及用于恢复运行和钩子行为的元数据。 |

### 重点实现领域

- **扫描与钩子** – `config.json` 控制需要忽略的文件、运行的预提交钩子以及同步模式的行为。
- **语言支持** – `languages.json` 是文件类型与语言映射的唯一权威来源。
- **健康与状态** – `manifest.json` 提供上次同步的快速健康概览，每次同步后更新。
- **架构执行** – `rules/` 保存 DSL 规则，系统会在每次文件处理时检查这些规则，防止违反分层架构和文档要求。
- **执行追溯** – `runtime/` 提供同步流水线的详细分步记录，对调试和审计至关重要。
