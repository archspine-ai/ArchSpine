# ArchSpine 包配置摘要

## 角色
此文件是 ArchSpine npm 包的核心配置，决定了工具的安装、运行、构建、测试和发布方式。所有元数据、依赖关系和自动化流水线都在此声明，是项目生命周期管理的关键组成部分。

## 主要职责
- 声明包标识、版本和许可协议（Apache‑2.0）。
- 定义可执行入口点（`spine` 和 `archspine` 命令）。
- 配置构建、测试、文档和发布的自动化脚本。
- 设定 Node.js 运行时引擎要求（≥20）。
- 列出运行时依赖项和开发所需的工具。
- 指定分发文件，包括编译输出、模式文件和文档资源。

## 不变约束
- Node.js 引擎必须为 20 或更高版本才能运行脚本和已安装的依赖。
- `main` 字段必须指向存在的编译入口点 `dist/cli/index.js`。
- `type` 字段必须保持为 `module` 以确保 ES 模块解析。
- `files` 数组必须排除已发布包中的 `__mocks__` 目录和源映射文件。
- `scripts` 块中的所有脚本必须无需人工干预即可执行（例如通过 `npm run`）。

## 参数定义
| 参数 | 说明 |
|------|------|
| `version` | 指定当前发布版本（1.0.2）。用于语义化版本控制和依赖解析；版本不匹配可能导致安装冲突。 |
| `engines` | 要求 Node.js 版本 ≥20。此约束确保代码库中使用的现代 JavaScript 特性得到支持。违反此要求将在旧版 Node 上导致运行时错误。 |
| `scripts` | 定义了构建（`build`）、测试（`test`、`test:ci`）、文档（`docs:dev`）和发布验证（`release:gate`）等自动化命令。每个脚本都是一个潜在的安全面；若注入任意代码可能造成供应链攻击，应审计构建脚本（`scripts/build.mjs`）以确保安全。 |
| `dependencies` | 列出运行时包，如 `@ast-grep/lang-c` 和 `@ast-grep/lang-cpp`。版本范围可能引入意外更新，建议在生产部署中锁定精确版本以保持稳定性。 |

## 稳定性与风险
该配置通过控制构建输出和依赖树直接影响系统稳定性。`engines` 字段强制最低 Node 版本，防止因不支持的 API 导致失败。`scripts` 块，特别是 `build`、`release:gate` 和 `validate`，是质量把关者；配置错误可能导致发布有缺陷的包。`files` 字段通过排除测试和源映射文件来减小包体积，降低内部代码意外暴露的风险。然而，包含动态脚本路径（如 `node scripts/release-gate.mjs`）引入了对外部脚本完整性的依赖；这些脚本一旦被入侵，可能影响整个 CI/CD 管道。

## 导出 / 公开接口
此文件本身不导出任何运行时代码。其主要外部效果是定义包的入口点（`spine` 和 `archspine`）。所有其他行为通过 npm 对声明字段和脚本的解释来表达。
