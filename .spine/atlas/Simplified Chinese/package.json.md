# ArchSpine CLI – 包清单（`package.json`）

## 作用
此文件是 ArchSpine CLI 工具的**包清单**，定义了项目的元数据、入口点、依赖关系以及构建、测试和发布的脚本。

## 主要职责
- **CLI 命令注册** – 将 `spine` 和 `archspine` 命令映射到可执行脚本。
- **入口点解析** – 指定模块的主入口文件（`main`）和模块类型（`type: "module"`）。
- **构建、测试和发布流程** – 脚本定义了编译、代码检查、测试、文档生成和发布的自动化命令。
- **依赖管理** – 声明运行时（`dependencies`）和开发阶段（`devDependencies`）的包。
- **分发控制** – `files` 白名单决定 npm 包中包含哪些文件。
- **许可证** – Apache-2.0 许可证规定了使用、修改和分发权限。

## 重要不变项与排除范围
- **Node.js 引擎必须 >=20** – 防止在不支持的平台上运行时出错。
- **TypeScript 编译必须在 CLI 执行前成功** – `bin` 脚本依赖编译后的输出。
- **所有测试套件必须在发布前通过** – 脚本将测试和模式验证作为强制质量门禁。
- **排除范围** – 此文件不处理运行时逻辑、配置加载或除 CLI 注册外的用户功能。

## 导出的外部可见行为
- **`bin` 条目** – 安装后向用户暴露 `spine` 和 `archspine` 命令。
- **`main` 入口** – 允许以编程方式导入包（例如 `require('archspine')`）。
- **`scripts`** – 提供 `npm run build`、`npm test`、`npm run lint`、`npm run docs` 和 `npm run release` 作为开发和 CI 的标准接口。

## 稳定性与风险
此文件是项目构建和发布流程的基础。配置错误（例如 `main` 或 `bin` 路径不正确）可能导致 CLI 无法调用。引擎约束可防止在不兼容的 Node 版本上部署。`files` 白名单减少了分发体积，但可能意外排除必要资源。脚本定义了关键的质量门禁（测试、模式验证），跳过它们会增加发布有缺陷代码的风险。应锁定依赖版本以避免供应链问题。
