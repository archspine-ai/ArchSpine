# ArchSpine 缓存数据库模式迁移

本文档描述 ArchSpine 镜像系统的一个底层数据库迁移脚本。它确保缓存 SQLite 模式在部署和更新时保持一致。

## 目的

该脚本向缓存数据库（`cache.db`）的 `files` 表中添加两个必需列：`mtime`（最后修改时间）和 `size`（文件大小，单位为字节）。这两个列用于在镜像操作中跟踪文件元数据。脚本静默运行——如果列缺失则添加，如果列已存在则不执行任何操作。

## 适用读者

- ArchSpine 存储层的**维护者**。
- 修改缓存模式或调试迁移问题的**开发人员**。
- 部署或更新 ArchSpine 的**运维人员**，需要验证缓存数据库结构是否正确。

## 本文档锚定的关键决策与工作流

- 迁移使用 `ALTER TABLE` 语句，并通过 `try/catch` 块处理异常，避免列已存在时报错。这种幂等模式是未来所有模式变更的标准做法。
- 脚本操作路径为 `.spine/cache.db` 的 SQLite 数据库。该路径为硬编码，运行前必须存在。
- 此迁移在 ArchSpine 初始化时或模式更新后执行。它**不是**面向用户的功能，而是内部维护工具。

## 要点

- 在 `files` 表中添加两个整数列：`mtime` 和 `size`。
- 使用优雅的错误处理，允许重复运行而不失败。
- 仅涉及缓存数据库——核心镜像逻辑保持不变。
