# ArchSpine 开发构建监视器

## 概述
本文档介绍 ArchSpine 项目中的**自动开发构建监视器**脚本。它在本地开发环境中运行，持续监控源目录和配置文件，一旦检测到更改便自动触发重建。目的是确保开发者始终拥有最新的编译输出，无需手动运行构建命令。

## 文档目的
- 作为监视器行为、职责和配置的参考资料。
- 固定本地开发工作流：用自动重建替代手动执行 `npm run build`。
- 明确说明监视器**能做什么**和**不能做什么**，避免与 CI/CD 或生产工具混淆。

## 目标读者
- **ArchSpine 代码库的贡献者**，在搭建本地环境时需要了解此脚本。
- 需要理解或修改构建监视循环的开发者。
- 开发过程中遇到构建相关问题的调试人员。

## 工作原理
1. 脚本每秒钟轮询一组监视目标（例如 `src/`、`tsconfig.json`、`package.json`、`scripts/build.mjs`）。
2. 对每个目标计算签名（基于修改时间和文件大小）。
3. 当签名发生变化时，排队一次构建。如果当前已有构建正在运行，则请求被暂存，待当前构建结束后立即执行。
4. 构建完成后，记录结果（成功或失败），并继续监视。

## 关键特性与决策
- **自动重建** – 节省时间，确保输出始终反映最新源代码。
- **队列机制** – 一次最多只允许一个待处理的构建，防止并发冲突。
- **可配置的监视目标** – 可扩展 `WATCH_TARGETS` 数组以监控更多文件或目录。
- **跨平台支持** – 脚本根据操作系统自动选择正确的 `npm` 命令（Windows 下使用 `npm.cmd`）。
- **初始构建** – 启动时自动触发一次完整构建，保证输出是最新的。

## 不涵盖的范围
- 生产部署、CI/CD 流水线。
- 构建产物或分发的管理。
- 单元测试或代码检查 – 监视器仅负责重建。

详细实现请参见源代码文件 `scripts/dev-build-watcher.mjs`。
