# ArchSpine 开发构建监视器

## 目的
本文档实现了一个用于 ArchSpine 项目的文件监控构建循环。它会持续监视源文件、TypeScript 配置和包元数据，一旦检测到变化，就会自动重新运行构建脚本。

## 上下文与受众
面向参与 ArchSpine 构建管线的开发人员。该脚本在开发环境中运行，旨在无需手动重建的情况下，为代码变更提供快速反馈。

## 主要职责
- 轮询源目录和配置文件以检测变更
- 通过 `npm run build` 触发增量重建
- 管理构建队列以避免并发构建

## 不涉及范围
- 应用程序运行时逻辑
- 面向用户的文档
- 部署或发布工作流

## 关键要点
- 监控 `src/`、`build.mjs`、`tsconfig.json` 和 `package.json` 的修改
- 每 1000 毫秒轮询一次文件修改时间戳
- 如果构建正在进行中，则将重建请求加入队列
- 在控制台输出构建开始、完成和失败事件
