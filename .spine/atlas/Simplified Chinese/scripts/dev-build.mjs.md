# ArchSpine 开发构建监视器

## 用途
本文档描述了开发时期的自动重建机制，该机制会监控源文件的变化，并在检测到改动时自动触发项目重建，从而减少开发人员的手动编译操作。

## 目标读者
ArchSpine 核心工具的开发人员，他们在修改 TypeScript 源代码、构建脚本或配置文件时，需要即时获得重建反馈。

## 工作原理
- 每 1 秒轮询指定目录和文件以检测更改。
- 检测到变化后，运行 `npm run build`。
- 构建请求会排队，防止并发构建。
- 监控范围包括 `src/` 目录、构建脚本、`tsconfig.json` 和 `package.json`。

## 关键决策
- 采用轮询而非文件系统事件，以确保跨平台简单性。
- 使用队列机制避免同时进行构建。
- 在控制台输出构建状态日志。
