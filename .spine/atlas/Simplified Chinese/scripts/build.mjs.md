# ArchSpine 构建与分发脚本

## 文档存在的意义

本文档描述了 ArchSpine 项目将 TypeScript 源代码树转换为可供分发包的自动化构建脚本。它的存在是为了统一并固化构建流水线的各个步骤，确保发布过程的一致性和可重复性。该脚本锚定了每个维护者或贡献者在准备分发时必须遵循的工作流。

## 目标读者

- **项目维护者**，负责发布 ArchSpine 版本。
- **基础设施贡献者**，需要修改构建链（例如增加新的资产类型、更改编译标志或调整清理逻辑）。
- **调试分发问题的开发者**，希望准确了解源代码到最终 `dist/` 文件夹之间发生的一切。

## 关键决策与工作流

脚本采用 Node.js 模块编写，使用 ES 模块语法（`import/export`）。它可以直接运行，也可以通过编程方式调用。

### 编译

- TypeScript 编译通过标准 TypeScript 编译器 `tsc` 完成。该命令从仓库根目录执行并继承 `stdio`，因此编译错误会直接显示。

### 文件复制

- 非 TypeScript 资产（如 `src/ast/rules/` 中的规则文件以及整个 `src/assets/` 目录）通过 `fs.copyFileSync` 和递归目录复制函数（`copyDir`）手动复制到 `dist/` 中。
- 复制逻辑遵守一个排除函数（`shouldExcludeDistEntry`），它会过滤掉任何名称中包含 `__mocks__` 的路径，以防止测试存根泄漏到分发包中。

### 清理

- 脚本在完成初始复制后，会显式移除 `dist/` 下所有名为 `__mocks__` 的目录。这是一个安全网，以防某些模拟文件被意外复制。
- 在一切开始之前，整个 `dist/` 目录会被删除，以避免遗留旧产物。

### 可执行权限

- CLI 入口点（`dist/cli/index.js`）被赋予可执行权限（`chmod 755`），使用户无需通过 `node` 即可直接调用。

## 如何使用

将脚本作为独立命令运行（它会检查 `process.argv[1]` 是否等于自身的 URL）。通常通过 npm 脚本或持续集成任务来调用。

```bash
node build.js
```
