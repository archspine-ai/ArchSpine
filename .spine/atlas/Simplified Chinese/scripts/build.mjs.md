# ArchSpine 构建脚本文档概要

## 文档目的

本文档详细描述了 ArchSpine 项目中的自动化构建脚本（`build.mjs`）的工作原理与执行流程。该脚本负责将源代码编译、组装并准备成可分发的形式，是项目从开发状态到可交付产物之间的核心步骤。

## 目标读者

- 维护 ArchSpine 仓库的开发者
- 需要理解构建流程以便调试或扩展构建步骤的贡献者
- 参与 CI/CD 集成或发布流程的团队成员

## 构建工作流

1. 清理并重建 `dist` 输出目录。
2. 使用 `tsc` 编译 TypeScript 源代码。
3. 将 `src/ast/rules/` 中的 AST 规则文件（YAML 格式）复制到 `dist/ast/rules/`。
4. 将 `src/assets/` 中的资源文件（模板、提示等）递归复制到 `dist/assets/`，同时跳过所有 `__mocks__` 目录。
5. 从最终 `dist` 输出中递归删除所有 `__mocks__` 目录，确保测试模拟文件不会进入分发产物。
6. 为 CLI 入口文件 `dist/cli/index.js` 设置可执行权限（`0o755`）。

## 关键决策与设计意图

- **手动复制 AST 规则文件**：规则文件采用 YAML 格式，TypeScript 编译器不会自动处理，因此需要显式复制以保留其结构。
- **排除模拟目录**：`__mocks__` 目录仅用于测试，构建产物中不应包含它们，避免污染交付物。
- **可执行权限**：CLI 入口文件在构建后直接赋予执行权限，方便用户无需额外步骤即可运行。
- **清理后再重建**：确保每次构建从清洁状态开始，避免残留文件导致不可预期的问题。

## 不在本文档覆盖范围

- 构建产物的运行时执行
- 测试或验证步骤
- 部署或发布过程
- 源代码 linting 或格式化

## 使用方式

构建脚本可通过以下命令运行：

```bash
node build.mjs
```

该脚本作为独立的 `build.mjs` 模块存在，同时提供了 `main()` 函数供其他 Node.js 脚本调用。
