# ArchSpine CLI 演示录制脚本摘要

## 文档目的

本文档是一个终端录制脚本，用于生成演示 GIF，以可视化方式展示 ArchSpine CLI 工具在实际开发中的操作流程。它通过依次执行四个核心命令——构建项目（`npm run build`）、快速同步（`sync --fast`）、查看系统信息（`info`）以及执行预检扫描（`scan --dry-run`）——让观众直观地看到该工具在终端环境中的典型交互方式。这张演示 GIF 是项目对外展示的核心素材，帮助新用户或评估者快速建立对 CLI 工作流的感性认识。

## 目标读者

本摘要及演示内容面向：
- 希望快速了解 ArchSpine 主要 CLI 交互的**开发者**，无需阅读冗长的文档；
- 正在评估项目是否满足自身需求的**评估人员**；
- 需要将工具介绍给团队的**项目维护者**，可用于项目 README、演讲或新手入门材料中。

## 锚定的关键工作流与决策

该录制脚本锚定了以下关键工作流和设计决策：

- **构建流程**：脚本假定项目已经过一次完整构建（`npm run build`），且 `dist/` 目录已存在。这反映了 ArchSpine CLI 工具的使用前提：必须先构建项目，之后才能执行 CLI 命令。
- **终端环境配置**：录制指定使用 `zsh` shell、Catppuccin Mocha 主题、45ms 打字速度以及 1280×720 分辨率。这些设置决定了演示的视觉呈现风格，是团队选择在演示中保持一致性的决策。
- **核心命令序列**：`sync --fast`、`info`、`scan --dry-run` 这三个命令的组合展示了 CLI 工具的主要功能模块：同步、信息查看和扫描。录制中的顺序反映了常见的用户操作流：先快速同步，再检查信息，最后执行预检扫描。
- **模拟而非实时**：脚本是一个录制模拟过程，而非实时操作。这一决策意味着演示注重视觉流程的连贯性，而非提供命令背后的详细解释，适合快速展示而非深度教学。

## 关键要点

- 脚本在特定终端环境下运行（zsh、Catppuccin Mocha 主题、45ms 打字速度）。
- 假设项目已构建成功（`dist/` 目录存在），然后才执行 CLI 命令。
- 展示的命令包括：`build`、`sync --fast`、`info`、`scan --dry-run`。
- 这是一个录制的模拟过程，而非实时操作；重点关注视觉流程，而非详细解释。
- 该演示旨在为新用户提供立即、具体的工具使用印象，降低入门门槛。
