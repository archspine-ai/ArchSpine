# ArchSpine 发布门禁脚本

## 文档编写目的

本文档描述 **ArchSpine 的自动化发布门禁脚本** —— 在任何版本发布之前必须通过的最终质量检查点。它明确规定了一系列验证步骤的执行顺序：构建、单元测试、模式验证、协议验证以及打包检查。如果没有此脚本，发布流程将依赖人工操作，容易出错且不可重复。

## 目标读者

- **开发者**：需要在本地运行发布门禁或了解发布被阻止的原因时，可参考本文档。
- **发布管理员**：负责监督发布流水线，需要可靠且可重复的验证流程。
- **CI/CD 工程师**：维护或扩展持续集成工作流程时，需要理解门禁脚本的逻辑。

## 脚本定义的流程与决策

该脚本定义了一个严格按顺序执行的 **五阶段门禁流水线**：

1. **构建（build）** — 确保项目编译成功（`npm run build`）。
2. **单元测试（unit tests）** — 运行所有单元测试。
3. **模式验证（schema tests）** — 验证模式变更的正确性。
4. **协议验证（protocol validation）** — 检查协议一致性。
5. **打包检查（pack check）** — 确认软件包完整性。

任何阶段失败都会导致脚本立即停止并输出清晰错误信息。这种“快速失败”设计防止了不完整或有问题的构建进入发布环节。

脚本可通过两种方式运行：
- 作为 **CI/CD 流水线**的一部分，在每个发布候选版本上自动执行。
- 由维护者 **手动运行**，作为发布前的预检查。

## 不在此脚本范围内

- 手动发布步骤（如版本号提升、变更日志生成）。
- 部署、上线或回滚流程。
- 环境配置（脚本假设运行环境已预先设置好）。

---

**核心要点**：此脚本是发布前必须通过的所有检查的 **唯一权威来源**，它既是文档，也是可执行的发布门禁。
