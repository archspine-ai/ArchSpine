# ArchSpine 占位包发布脚本

## 本文档的目的

本文档记录了将 ArchSpine 初始占位包（版本 `0.0.1`）安全发布到官方 npm 仓库所需的脚本和逻辑。它的存在是为了确保首次发布过程的安全性：严格检查包名是否已被占用、验证元数据（名称和版本）是否准确，并在实际执行发布命令之前运行强制性的发布门禁流程。如果没有本文档，团队将缺乏一个可重复且有保障的流程来在 npm 上注册包名。

## 谁应该阅读

- 负责在 npm 上初始化 ArchSpine 项目的开发者。
- 任何执行 `archspine` 包首次发布操作的人员。
- 需要理解首次发布工作流约束的维护者。

## 关键工作流与决策锚点

1. **占位版本断言** – 脚本强制要求 `package.json` 中的包名必须为 `archspine`，版本必须为 `0.0.1`。如果任一值不符，脚本将立即失败，防止意外发布非占位版本。

2. **名称冲突预防** – 在尝试发布之前，脚本会查询 npm 仓库中是否已存在名为 `archspine` 的包。如果该名称已被占用，脚本将报错并停止，迫使团队选择其他包名。

3. **发布门禁** – 脚本在发布前执行 `npm run release:gate` 命令。该门禁必须通过（退出码为 0）才能继续发布，确保所有预发布检查（如代码检查、验证）均已满足。

4. **发布命令** – 所有检查通过后，脚本会针对官方仓库执行 `npm publish`。完成后，它会输出一条验证命令，供开发者确认包已成功发布。

这些步骤共同定义了一个安全、受控的发布流程，防止意外覆盖第三方包，并对 ArchSpine 的首次发布施加了必要的纪律。
