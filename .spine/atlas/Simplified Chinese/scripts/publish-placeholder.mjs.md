# ArchSpine 占位发布脚本

## 目的
本文档描述了一个 CI 脚本，用于自动化 ArchSpine npm 包的占位发布过程。其核心目标是在实际发布开始之前，在官方 npm 注册表上安全地保留包名 `archspine`，防止其他发布者占用该名称。

## 预期读者
ArchSpine 项目的维护者和 CI 流水线开发者。该脚本通常作为首次发布前的一次性设置步骤运行，或者当需要重新保留包名时使用。

## 脚本所锚定的关键决策与工作流
- 脚本严格执行占位版本（`0.0.1`），避免在保留步骤中意外暴露真实代码或引发版本变更。
- 它查询 npm 注册表以确认包名尚未被占用；若已被占用，脚本会提前失败，提示需要选择其他包名。
- 在发布之前，脚本会运行项目的发布门控（`npm run release:gate`），确保质量检查通过。
- 验证通过后，使用 `npm publish` 将包发布到官方注册表，完成保留操作。
- 如果 `package.json` 中的名称或版本与预期的占位值不匹配，脚本会明确拒绝发布，防止配置错误。

## 关键要点
- 占位版本**必须**为 `0.0.1`；任何其他版本都将导致脚本终止。
- 脚本实时查询 npm 注册表，不依赖本地缓存。
- 与发布门控的集成确保了占位步骤与项目整体质量标准一致。
- 该脚本是一次性过程，仅需运行一次（或当需要重新保留时再运行），不属于常规发布流水线的一部分。
