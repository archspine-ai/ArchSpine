# ArchSpine 占位包发布脚本

## 目的
本文档定义了用于将 ArchSpine npm 包的初始占位版本发布到 npm 注册表的自动化脚本。它确保包名称可用、版本设置为所需的占位符，并在发布前通过发布门禁检查。

## 上下文与受众
适用于负责在 npm 注册表上初始化 ArchSpine 包的开发者和 DevOps 工程师。该脚本是项目发布管道的一部分，仅在设置期间运行一次。

## 主要职责
- 描述将 ArchSpine 包的占位版本（0.0.1）自动发布到 npm 注册表的过程
- 涵盖验证步骤：检查包名称、版本和注册表可用性
- 记录发布门禁的执行和最终发布命令

## 不涉及范围
- ArchSpine 系统的详细架构设计
- 面向用户的功能文档或 API 参考
- 初始占位发布之外的维护流程

## 关键要点
- 脚本会检查包名称是否为 'archspine'，版本是否为 '0.0.1'，然后才继续执行。
- 它会验证该包尚未存在于 npm 注册表中，以避免冲突。
- 发布前必须通过发布门禁检查（npm run release:gate）。
- 脚本将占位包发布到官方 npm 注册表。
