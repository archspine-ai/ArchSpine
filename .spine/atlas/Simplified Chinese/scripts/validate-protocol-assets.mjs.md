# 协议模式验证器

## 用途

此脚本作为构建时的质量关卡，确保所有协议模式定义及其示例文件在语法和语义上保持一致。它能有效防止形式化规范（schema）与说明性示例之间出现意外偏差。

## 背景与受众

面向 ArchSpine 项目的开发者和发布管理者。该脚本在 CI 流程中以及本地修改模式或示例时自动运行。任何对 `schemas/` 或 `schemas/examples/` 目录进行更改的人员都应了解此验证步骤。

## 关键要点

- 使用 AJV 进行严格的模式验证，开启所有错误检查选项。
- 同时验证 JSON 示例文件和带有前置元数据的 Markdown 规则文档。
- 作为合入前的关卡确保协议资产的一致性。
- 验证失败时终止构建并输出详细的错误信息。
- 使模式与示例之间的不匹配立即可见，方便维护者修复。
