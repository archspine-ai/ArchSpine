# ArchSpine 协议验证脚本

## 目的

本文档描述了自动化验证脚本，该脚本是 ArchSpine 镜像系统的质量关口。它确保所有协议资产——包括 JSON 模式定义、示例文件和规则文档——符合其约定的契约。通过强制执行结构一致性，该脚本防止集成错误，并维护整个 ArchSpine 生态系统的可靠性。

## 受众

本文档面向以下人员：

- **开发者**：为 ArchSpine 协议贡献模式或示例文件的人员。
- **维护者**：负责审查更改并在 CI/CD 流水线中运行验证的人员。
- **DevOps 工程师**：将此脚本集成到自动化工作流程中的人员。

读者应熟悉 JSON Schema 及基本的验证概念。

## 锚定的工作流与决策

该脚本在开发或持续集成工作流中执行，任何模式或示例文件的更改在合并前都必须通过验证。其输出决定了一个更改是被接受还是被拒绝。该脚本锚定的关键决策包括：

- 新添加或修改的示例文件是否正确地实现了对应的模式。
- 带有 YAML 前置元数据的 Markdown 规则文档在结构上是否有效。
- 整个协议资产集合是否保持内部一致性。

## 关键结论

- 验证五种示例文件（`spine-unit.example.json`、`spine-folder-unit.example.json`、`spine-project-unit.example.json`、`spine-manifest.example.json`、`spine-rule-document.example.json`）分别与其对应的模式。
- 使用 **AJV** 采用 JSON Schema 2020-12 严格模式，并通过 `ajv-formats` 进行格式验证。
- 同时处理纯 JSON 示例文件和带有 YAML 前置元数据的 Markdown 文件（通过 `gray-matter` 解析）。
- 任何验证失败时会以非零退出码结束，适合自动化 CI 流水线。

## 行为概要

1. 从 `schemas/` 目录加载所有模式并注册到 AJV。
2. 根据文件名将示例文件映射到预期的模式。
3. 对每个 JSON 示例进行验证，报告成功或详细的错误信息。
4. 处理 Markdown 规则文档：解析前置元数据，将正文内容作为 markdown 字符串处理，然后将生成的对象按规则模式进行验证。
5. 所有验证通过时以退出码 0 结束，否则以退出码 1 并显示错误数量结束。
