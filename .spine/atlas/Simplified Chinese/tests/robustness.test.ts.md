# ArchSpine 锁工作器测试套件

## 角色
这是一个 Vitest 测试套件，用于验证跨进程锁获取和释放行为，使用隔离的工作器脚本。它确保锁工作器在进程边界上正确运行。

## 主要职责
- 生成子进程运行锁工作器脚本，支持可配置的操作模式：`acquire-release`（获取-释放）、`acquire-hold`（获取-保持）和 `acquire-release-rewrite`（获取-释放-重写）。
- 监听工作器的标准输出和标准错误，捕获并解析 JSON 结果以进行断言。
- 在每个测试后清理临时目录和进程，确保隔离性。
- 断言跨进程边界的锁负载序列化和解析一致性。
- 验证多进程锁操作中失败场景的错误码和消息。

## 重要不变性
- 使用 Vitest 测试框架，采用 `describe`/`it`/`beforeEach`/`afterEach` 结构。
- 依赖 Node.js 的 `child_process` 实现测试中的进程隔离。
- 依赖 `src/utils/lock.js` 中的锁负载序列化工具。
- 通过 `FileSystemManager` 清理测试工件。

## 负面范围（不在范围内）
- **不**实现核心锁获取逻辑（委托给工作器脚本）。
- **不**提供生产环境锁管理工具。
- **不**处理超出测试模拟范围的实时锁协调。

## 最重要的导出行为
该测试套件验证锁操作（获取、释放、保持、重写）在独立的 Node.js 进程间是否正确工作，确保序列化一致性和正确的错误处理。它是锁工作器多进程行为的主要验证层。
