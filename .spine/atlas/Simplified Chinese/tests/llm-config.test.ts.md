# ArchSpine 凭证后端集成测试套件

本 Vitest 集成测试套件用于验证 ArchSpine 镜像系统中凭证后端的可用性以及 LLM 配置的解析能力。

## 角色

该套件确保 LLM 配置解析系统能够正确处理凭证后端不可用的场景，从而保持凭证解析管道的健壮性。

## 主要职责

- **模拟不可用后端**：通过模拟不可用的 `CredentialBackend` 实现，模拟后端缺失的场景。
- **集成测试**：测试在不同凭证后端状态下，`Config`、`Secrets` 和 LLM 运行时解析之间的集成。
- **验证**：验证当凭证后端可用或不可用时，LLM 设置是否正确解析。
- **隔离管理**：管理临时测试目录以实现隔离的测试执行，并确保测试后的清理工作。

## 重要不变规则

- 所有测试文件必须使用 `.test.ts` 或 `.spec.ts` 后缀（由 `test-file-suffix` 规则强制要求）。

## 不涉及范围

- 对单个 `CredentialBackend` 实现的单元测试。
- 对 LLM 提供商特定 API 调用或网络行为的测试。
- 凭证系统的性能或负载测试。

## 导出的行为

本套件不导出任何公共 API 接口；它纯粹是一个用于验证集成行为的测试工具。
