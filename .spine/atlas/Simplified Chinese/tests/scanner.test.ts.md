# 扫描器测试套件

该 Vitest 测试套件在隔离的临时目录中验证扫描器引擎的文件扫描行为。每个测试都会创建一个包含 `.spine` 子目录和 `.gitignore` 文件的临时目录，然后实例化一个绑定到该目录的 `Config` 对象，以模拟真实项目配置。该套件针对受控的文件系统结构测试扫描器的文件发现逻辑，并在必要时使用 `vi.fn` 模拟和监视文件系统交互。

## 主要职责

- 为每次测试运行创建包含 `.spine` 子目录和 `.gitignore` 文件的临时目录
- 实例化绑定到临时目录的 `Config` 对象，模拟项目配置
- 针对受控的文件系统结构测试扫描器的文件发现逻辑
- 在需要时使用 `vi.fn` 模拟和监视文件系统交互

## 不涉及范围

- 单独测试非扫描器组件（如配置引擎）
- 与实时外部服务进行端到端集成测试
- 扫描操作的性能基准测试

## 不变约束

- 每个测试都在全新的临时目录中运行，确保隔离性
- 使用 `Config` 类为扫描器提供有效的配置上下文
- 依赖 Vitest 的模拟工具（`vi`）进行受控的文件系统交互

## 架构意图

为扫描器引擎的核心文件发现和扫描逻辑提供隔离、确定性的单元测试。最近的变更强化了扫描恢复验证并修复了回退路径。
