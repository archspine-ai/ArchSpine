# ArchSpine – 运行时会话助手单元测试

## 角色
针对运行时会话助手的错误处理、检查点管理和锁清理的 Vitest 单元测试。

## 主要职责
- 测试在加载检查点遇到问题时是否记录警告。
- 验证运行时发生错误时检查点是否被标记为失败。
- 确保即使在失败情况下，获取的锁在会话执行后也能正确释放。
- 模拟外部依赖（`writer-boundary`、`spine-gate`）以隔离运行时会话助手的行为。

## 重要不变项与范围外说明
- 使用 Vitest 测试框架。
- 依赖模拟来隔离被测单元。
- 遵循项目测试命名约定（`.test.ts` 后缀）。
- **范围外：** 生产运行时逻辑实现、实际的检查点持久化或锁管理、以及使用真实依赖的集成测试。

## 最重要的导出或外部可见行为
此测试文件不导出任何生产代码；它是一个纯粹的测试套件。其主要外部可见行为是验证运行时会话助手在失败条件下是否正确处理错误、管理检查点状态并清理锁。
