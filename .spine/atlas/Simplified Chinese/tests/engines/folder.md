# 扫描引擎测试套件

本目录包含 **ArchSpine 扫描引擎** 的集成测试套件，用于验证文件发现和排除逻辑的正确性。测试在隔离环境中运行，利用临时目录和模拟文件结构。

## 主要子模块

- **`scanner.test.ts`** — 基于 Vitest 的测试文件，负责：
  - 创建并管理临时目录，确保测试执行的隔离性。
  - 生成模拟文件系统结构（目录和文件），模拟实际扫描目标。
  - 调用已应用扫描策略的扫描引擎执行文件发现。
  - 验证扫描引擎正确包含预期文件并排除应忽略的模式。

## 重点测试领域

以下实现区域最为关键：
- 在各种目录结构下正确包含目标文件。
- 准确排除匹配忽略模式的文件（例如 `.gitignore` 规则、自定义排除项）。
- 空目录、深层嵌套结构及符号链接（如适用）的边界情况。

所有测试用例均基于 **Vitest**，并作为项目持续集成流程的一部分运行，确保对扫描引擎的修改不会破坏核心发现功能。
