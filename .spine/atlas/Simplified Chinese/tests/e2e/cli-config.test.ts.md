# ArchSpine CLI 端到端测试套件

本文件为 ArchSpine 的 CLI 命令提供自动化的端到端测试套件，使用 **vitest** 和 **child_process** 执行集成测试。其核心职责是在隔离环境中验证 CLI 输出的正确性、错误处理能力以及命令行为。

## 主要职责

- **隔离测试执行**：通过 `fs.mkdtempSync` 和 `os.tmpdir` 为每次测试创建独立的临时目录。
- **CLI 运行辅助函数**：定义 `runCli` 函数，用于启动构建后的 CLI 二进制文件并传入参数，同时捕获标准输出、标准错误和退出码。
- **提示模拟**：编写包装脚本，实现对交互式提示的非交互式测试。
- **全面覆盖**：在 7 个文件中执行 37 个测试用例，覆盖 CLI 输出、错误处理及命令行为。
- **清理**：使用 `afterEach` 在每个测试完成后删除临时目录。
- **环境设置**：使用 `beforeAll` 准备测试环境，包括构建包装脚本。

## 重要不变量与范围说明

- **不变量**：所有测试文件必须以 `.test.ts` 或 `.spec.ts` 结尾；测试进程同步生成以确保执行确定性。
- **超出范围**：本套件**不涉及**内部 CLI 逻辑的单元测试、图形用户界面测试，以及依赖外部服务的网络功能测试。
- **公开接口**：不存在任何导出的符号；整个套件由测试运行器直接执行。

## 变更意图

- **架构意图**：提供全面的自动化端到端覆盖，以在发布前捕获 CLI 命令的回归缺陷。
- **最近变更**：新增 CLI 命令的端到端测试套件，包含 7 个文件中的 37 个测试（提交 `171f7b4`）。
