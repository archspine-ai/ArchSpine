# ArchSpine CLI 端到端测试套件

**角色：** 针对 ArchSpine CLI 命令的 Vitest 端到端（E2E）测试套件。

**关键职责：**

- 通过在临时项目目录中启动构建好的 CLI 来测试命令执行。
- 验证 CLI 命令的标准输出、错误输出、退出码及文件创建是否正确。
- 使用 `beforeAll` 初始化临时 Git 仓库并搭建项目脚手架。
- 使用 `afterEach` 清理临时目录以保持测试隔离。
- 验证 CLI 清单和生成命令的 JSON 输出。
- 确保 CLI 正确处理项目初始化、验证和文件生成等参数。

**不涵盖的范围：**

- 内部 ArchSpine 模块或服务的单元测试。
- 与数据库或容器等外部依赖的集成测试。
- CLI 输出的性能、安全或快照回归测试。
- 需要用户交互或模拟提示的测试。

**重要不变量：**

- 运行测试前，CLI 必须构建到 `dist/cli/index.js`。
- 测试文件必须遵循命名约定 `.test.ts` 或 `.spec.ts`（架构规则 test-file-suffix）。
- 测试依赖 `vitest` 作为测试运行器。

**变更意图：**

- 架构意图：为 ArchSpine CLI 建立全面的端到端测试套件，以验证命令输出、错误处理和文件生成在真实项目条件下的正确性。
- 最近变更：新增了 37 个端到端测试，分布在 7 个文件中，覆盖 CLI 命令（初始化、清单、生成），并通过临时项目目录实现隔离。

**导出接口：** 无（测试仅供测试运行器内部使用，不导出公共 API）。
