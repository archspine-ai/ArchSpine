本文件是 ArchSpine CLI 命令的集成测试套件，基于 vitest 构建。它启动编译后的 CLI 二进制文件并模拟用户交互，以验证端到端行为。

**主要职责：**
- 提供 `runCli` 辅助函数，启动 CLI 并捕获标准输出、标准错误和退出码。
- 每个测试中创建临时工作目录，防止文件系统污染。
- 通过包装脚本模拟 `prompts` 模块，实现自动化输入注入。
- 使用 `describe`/`it` 块执行覆盖 `init`、`generate` 和配置命令的全面端到端测试。
- 通过 `execFileSync` 集成 git 命令，验证脚手架和提交工作流。
- 在 `afterEach` 钩子中清理临时目录。

**重要约束与排除范围：**
- 测试文件必须遵循 `<name>.test.ts` 或 `<name>.spec.ts` 命名规范。
- 临时目录必须确定性地创建和销毁。
- 所有测试针对 `dist/cli/index.js` 位置的已构建 CLI 进行。
- 使用 vitest 生命周期钩子确保隔离性。
- 本套件**不涵盖**内部模块的单元测试、UI/浏览器测试、性能/负载测试，或超出此测试环境的跨平台兼容性测试。

**主要外部接口：** `runCli` 辅助函数是最重要的导出行为，允许任何测试以编程方式执行 CLI 命令并断言输出和退出码。
