# ArchSpine CLI 端到端测试套件

**角色：** 该文件是 ArchSpine CLI 的端到端测试套件，用于在真实的临时 Git 仓库环境中验证 CLI 的正确行为。

**主要职责：**
- 为每个测试创建临时目录并初始化 Git 仓库。
- 通过 `child_process`（`spawnSync`、`execFileSync`）执行已构建的 CLI 二进制文件，传入不同参数。
- 断言预期的 CLI 输出、退出码以及副作用（如文件创建）。
- 使用 Vitest 生命周期钩子提供测试工具（例如 `afterEach` 清理）。
- 通过包装或拦截 `prompts` 模块来模拟用户输入，验证 CLI 的交互行为。

**不负责的范围（负面职责）：**
- 内部函数或模块的单元测试。
- 需要数据库或外部服务的集成测试。
- 对 `prompts` 模块本身的测试（该模块已被模拟或包装）。
- CLI 的性能或负载测试。

**重要不变量与假设：**
- 需要 Node.js 运行时以运行 Vitest 和执行 CLI。
- 假定 CLI 已预先构建（存在 `dist/cli/index.js`）。
- 依赖 `prompts` npm 模块已安装在 `node_modules/prompts/index.js` 路径下。
- 要求系统中已安装 Git 并可在 PATH 中访问。
- 使用 `os.tmpdir()` 下的临时目录，并在每个测试后清理。
- 测试文件必须以 `.test.ts` 或 `.spec.ts` 后缀命名，以便测试运行器发现。

**变更意图与范围：**
- 该套件是最新变更的一部分，新增了 7 个文件共 37 个测试用例，重点验证 CLI 命令的执行与输出。其架构意图是在发布前提供可靠、隔离的验证。该文件没有公开导出的接口，纯粹作为测试框架使用。
