## ArchSpine CLI 端到端测试套件（基于 Vitest）

本文件提供了用于 ArchSpine CLI 命令的完整端到端测试套件，采用 Vitest 实现。它通过在子进程中执行 `spawnSync` 来验证 CLI 行为：创建临时目录并配置执行环境，然后断言输出与预期结果一致。

### 主要职责

- 在隔离的子进程中启动 ArchSpine CLI 命令（例如 `init`、`build`）。
- 建立临时工作区并配置测试所需的执行环境变量。
- 执行特定子命令的集成测试，包括通过包装脚本模拟交互式输入的场景。
- 每个测试结束后，使用 `afterEach` 清理所有临时工件。
- 利用 `describe` 和 `it` 块组织测试用例，配合 Vitest 匹配器进行断言。
- 编写辅助脚本来模拟命令行交互输入。

### 不包含的内容

- 单个函数或模块的单元测试。
- ArchSpine 的生产代码。
- 非 CLI 功能（如图形界面、浏览器测试）。
- 跨平台或性能测试。

### 关键不变规则

- 测试文件名必须以 `.test.ts` 或 `.spec.ts` 结尾。

### 对外暴露的行为

- **无任何公开导出。** 本文件是自执行的测试套件，并非库模块。它直接由 Vitest 运行，不向外部暴露任何函数或类。
