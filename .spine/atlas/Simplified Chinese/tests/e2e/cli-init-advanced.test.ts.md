该文件是一个 Vitest 端到端测试套件，专门用于验证 ArchSpine CLI 的 `init` 命令的交互式提示与文件系统行为。主要职责包括：创建临时目录并初始化 Git 仓库以提供隔离的测试环境；通过 `child_process.spawnSync` 或 `execFileSync` 执行构建后的 CLI 二进制文件；通过封装 `prompts` 模块来模拟用户输入；断言生成的文件、目录结构与配置与预期一致；使用 `afterEach` 钩子清理每个测试后的临时目录。

重要不变性：文件名必须以 `.test.ts` 或 `.spec.ts` 结尾；所有测试必须是幂等的且不依赖全局系统状态；仅使用 Vitest 作为测试运行器。本套件不包含单元测试、不涉及完整 CLI 调用的集成测试，以及生产代码或非测试源文件。架构意图是通过模拟真实用户输入并检查文件系统结果，为 CLI 命令行为提供端到端验证。没有提供外部可见的导出；该文件纯粹是测试运行器。（注：来自提交 171f7b4 的变更意图添加了此 E2E 测试套件，包含 7 个文件中的 37 个测试。）
