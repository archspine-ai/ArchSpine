# ArchSpine 索引读取器模式强制测试套件

## 角色
这是一个 Vitest 测试套件，用于验证索引读取器强制模式版本控制并正确处理兼容性场景的能力。

## 主要职责
- 为每个测试用例创建隔离的临时目录，防止测试之间产生副作用。
- 在每个测试后自动清理临时目录，保持测试隔离性。
- 写入具有特定模式版本的测试 JSON 文件，模拟遗留或不支持的索引文档。
- 调用索引读取器的 `readIndexDocument` 函数，测试其在各种模式条件下的行为。
- 断言不支持的文件单元被拒绝，并返回指示需要重建的状态。
- 验证测试数据中的模式版本以及响应中的预期模式版本。

## 重要不变性与负面范围
- 测试文件必须以 `.test.ts` 或 `.spec.ts` 结尾（规则：测试文件后缀）。
- 本套件**不**测试索引读取器的性能或并发性。
- **不**使用有效或受支持的模式版本测试读取器。
- **不**测试对格式错误的 JSON 或缺失文件的错误处理。
- **不**与临时目录之外的实际文件系统持久性进行集成测试。

## 导出/外部可见行为
该套件不导出任何公共接口；它纯粹是一个测试工具，用于验证索引读取器的模式拒绝逻辑。关键的外部可见行为是：不支持的模式版本会导致读取器返回需要重建的状态，并且响应中包含遇到的模式版本和预期的模式版本。
