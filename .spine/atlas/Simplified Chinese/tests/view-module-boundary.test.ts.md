# ArchSpine 视图模块边界测试

该 Vitest 单元测试用于强制实施视图服务层的架构模块边界。其主要目的是确保视图运行时和渲染模块正确放置在 `src/services/view` 目录下，从而维护服务层与基础设施层之间的清晰分离。

## 主要职责

- 验证视图运行时和渲染模块位于 `src/services/view` 目录下
- 断言以下特定视图模块文件的存在：
  - `view-renderer.ts`
  - `arch-diagram-renderer.ts`
  - `view-registry.ts`
  - `view-runtime.ts`
- 确保 `view-renderer.ts` **不**被放置在 `src/infra` 目录中，强制实施层分离

## 不涉及范围

- **不**测试视图模块本身的功能
- **不**验证模块文件的内容或正确性
- **不**强制执行除文件位置之外的命名约定

## 不变约束

- 必须是使用 Vitest 框架的测试文件
- 依赖 Node.js 的 `fs` 和 `path` 模块进行文件系统断言
- 对仓库的物理文件结构进行断言

## 架构意图

该测试通过验证文件放置来强制实施服务层（视图）与基础设施层之间的清晰分离。它支持最近为模块化 CLI 和解耦核心基础设施服务而进行的重构工作，通过防止可能违反架构边界的错误模块放置来保护架构完整性。

## 导出行为

该测试文件不导出任何公共接口；它纯粹是在测试套件执行期间运行的验证机制。
