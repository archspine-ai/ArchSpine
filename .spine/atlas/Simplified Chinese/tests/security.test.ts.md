# ArchSpine CLI 安全性集成测试

该文件是用于 ArchSpine CLI 安全性及提示流程行为的 Vitest 集成测试套件。它验证 CLI 是否正确处理 `dry-run` 模式与确认提示，以防止意外进行破坏性操作。

## 主要职责

- 为每个测试场景创建独立的临时目录。
- 编写包装脚本覆盖 `prompts` 模块，模拟用户对确认和选择提示的输入。
- 通过 `execFileSync` 在临时目录内运行已编译的 ArchSpine CLI 二进制文件。
- 在每个测试后通过 `afterEach` 钩子清理所有创建的临时目录。
- 验证 `dry-run` 标志能够阻止确认提示的显示。
- 确认在未启用 `dry-run` 时，确认提示的回答会被正确执行。
- 测试在受控的提示注入下，安全敏感的 CLI 工作流是否按预期运行。

## 重要不变规则

- 所有测试文件必须以 `.test.ts` 或 `.spec.ts` 结尾（架构规则 `test-file-suffix`）。

## 负范围（不涵盖的内容）

- 各个模块或函数的单元测试。
- 涉及真实 LLM 或外部网络调用的端到端测试。
- CLI 的性能或负载测试。
- 与提示安全性或 `dry-run` 行为无关的 CLI 功能测试。

## 变更意图

架构意图是提供安全导向的集成测试，验证 CLI 的提示编排，尤其是 `dry-run` 和确认提示的正确处理。最近的变更将 `dry-run` 检查移到了 `FixTask` 中的确认提示之前，并添加了相应的安全测试，以确保该修复得到充分验证。

## 导出的或外部可见行为

该文件不导出任何公共 API；它仅是一个由 Vitest 执行的测试套件。其外部可见行为是执行一系列集成测试，以断言 CLI 对用户提示输入和 `dry-run` 标志的响应是否符合预期。
