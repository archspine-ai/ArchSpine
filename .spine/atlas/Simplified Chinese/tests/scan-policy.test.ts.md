# ArchSpine – ScanPolicy 集成测试套件

## 角色
这是一个 Vitest 集成测试套件，用于验证 ScanPolicy 模块在临时文件结构和配置模式下的行为。

## 主要职责
- 创建临时目录和文件结构，模拟仓库布局以进行测试。
- 测试 `DEFAULT_SCAN_POLICY` 常量和扫描行为。
- 验证 Scanner 在受控环境中正确应用扫描策略。
- 测试配置模式处理和版本兼容性。
- 每次测试运行后清理测试工件。

## 重要不变项与负面范围
- **不变项：** 必须是后缀为 `.test.ts` 或 `.spec.ts` 的测试文件。必须将测试工件隔离到临时目录。不得影响实际项目配置。
- **负面范围：** 生产部署逻辑、用户界面组件、永久文件系统修改、网络或外部 API 调用。

## 最重要的导出/外部可见行为
- 该套件通过隔离、可重复的测试确保扫描引擎遵循策略和配置模式。
- 最近的更改加强了模式处理并增加了尝试预览功能，这反映在配置验证的更新测试用例中。
