# ArchSpine 仓库策略测试套件

## 角色
用于验证仓库策略配置和漂移检测的 Vitest 测试套件。

## 主要职责
- 创建临时目录，为仓库策略检查提供隔离的测试环境。
- 模拟控制台警告和日志，捕获并断言策略违规消息。
- 验证 `.gitignore` 和 `.gitattributes` 是否正确更新了 ArchSpine 管理的部分。
- 测试 `runRepoCheck` 函数在仓库配置偏离预期状态时的行为。

## 重要不变项与范围外
- **不变项：** 测试文件后缀必须为 `.test.ts` 或 `.spec.ts`（依据规则 `test-file-suffix`）。使用 Vitest 测试框架，遵循 `describe`/`it`/`expect` 模式。依赖模拟的控制台方法以避免污染测试输出。
- **范围外：** 直接 CLI 命令执行或用户交互。在临时测试目录之外对文件系统进行永久修改。与外部服务或网络调用的集成。

## 导出行为
未导出任何公共接口；此模块仅用于测试。
