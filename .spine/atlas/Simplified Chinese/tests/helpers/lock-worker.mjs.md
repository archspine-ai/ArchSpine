# 锁工作进程脚本

## 目的
本文档实现了一个锁工作进程脚本，用于测试 ArchSpine 分布式锁定系统。它提供命令行接口来获取、持有和释放基于文件的锁，并输出结构化状态消息以便与自动化测试套件集成。

## 上下文与受众
适用于 ArchSpine 镜像系统的开发者和质量工程师，需要验证跨多个进程的锁获取和释放行为。该脚本设计为由测试框架调用，而非直接供最终用户使用。

## 主要职责
- 提供基于命令行的锁工作进程，用于获取并可选地持有或释放文件锁
- 支持三种模式：获取-释放、获取-持有和获取-释放-重写
- 向标准输出发送 JSON 格式的结构化状态消息，供测试框架使用
- 处理进程生命周期，包括基于信号的退出和定时持有周期

## 不涉及范围
- 不定义 LockManager 类本身（从 `../../src/utils/lock.ts` 导入）
- 不处理超出锁原语的多进程协调
- 不包含测试断言或验证逻辑

## 关键要点
- 支持三种操作模式：获取-释放、获取-持有和获取-释放-重写
- 向标准输出发送 JSON 格式的状态消息以支持测试自动化
- 处理 SIGINT 和 SIGTERM 信号以实现干净的进程终止
- 从项目的工具层导入 LockManager 类
