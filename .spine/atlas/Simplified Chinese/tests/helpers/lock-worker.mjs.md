# 锁工作进程脚本：测试 ArchSpine 中的 LockManager

## 文档目的

本文档提供用于测试 ArchSpine 镜像系统中 `LockManager` 类的锁工作进程脚本的完整源代码和说明。它是理解文件锁的获取、持有和释放在不同场景下如何工作的参考依据。

## 目标读者

本文档适用于 ArchSpine 项目的**开发者和测试人员**，他们需要验证文件锁定机制的行为，特别是在并发或分布式环境下。对于集成外部监控工具（需要消费 JSON 状态消息）的人员也很有帮助。

## 文档涵盖的内容

- **三种运行模式**：`acquire-release`、`acquire-hold` 和 `acquire-release-rewrite`。每种模式演示不同的锁生命周期模式。
- **与 `LockManager` 的集成**：脚本导入并使用项目源代码中的 `LockManager` 工具类。
- **JSON 状态输出**：脚本向标准输出发送结构化消息，用于调试和外部监控。
- **信号处理**：注册了针对 `SIGINT` 和 `SIGTERM` 的优雅关闭处理程序，以确保正确释放锁。

## 关键要点

- 锁工作进程支持三种模式，每种模式在获取、持有和释放锁方面具有不同的行为。
- 它依赖项目的 `LockManager` 类，而非自定义实现。
- 状态消息采用 JSON 格式，支持自动化测试和可观测性。
- 脚本可以保持进程运行以模拟长期持有的锁，并能干净地处理终止信号。

## 本文档锚定的工作流与决策

- 通过此脚本测试和验证关于锁生命周期的决策（例如何时释放、持有多长时间）。
- JSON 输出格式成为外部监控工具的合同。
- 信号处理模式为 ArchSpine 其他组件的优雅关闭提供了先例。
- 脚本的结构（参数解析、模式选择）为项目中的其他测试工具设定了模板。
