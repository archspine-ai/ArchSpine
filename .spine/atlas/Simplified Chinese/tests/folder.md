`tests` 目录是 ArchSpine 的综合性测试套件，旨在验证系统各个层面，从底层基础设施到端到端的 CLI 工作流。该目录包含 60 多个测试文件和若干按测试范围分组的子目录：常规 Vitest 套件负责单元与集成测试；`e2e` 文件夹专注于 CLI 二进制文件的端到端测试；`engines` 文件夹包含扫描引擎测试；`infra` 文件夹涵盖索引恢复和 LLM 重试机制；`helpers` 文件夹则提供了一个分布式锁工作线程脚本。

重点测试领域包括 CLI 命令编排（例如 `cli.test.ts`、`init-cli.test.ts`、`sync-command.test.ts`）、LLM 集成与配置（`llm.test.ts`、`llm-config.test.ts`、`llm-provider-utils.test.ts`）、引擎逻辑（上下文引擎、扫描器、提示引擎）、数据库完整性（`db-modules.test.ts`、`integrity.test.ts`）、凭据安全性（`credentials-backend.test.ts`、`credentials-store.test.ts`）、错误处理收敛（`error-system.test.ts`）以及架构边界强制执行（`infra-facade-boundary.test.ts`、`view-module-boundary.test.ts`）。像 `arch-diagram-renderer`、`rules-loader`、`spine-gate` 和 `writer-boundary` 这样的具体子模块得到了独立测试，而 `e2e/` 和 `engines/` 文件夹则验证了更高级别的集成方案。
