# ArchSpine CLI 同步命令集成测试

## 文件角色
本文件是针对 ArchSpine CLI 中 `sync` 命令的 Vitest 集成测试。它在隔离环境中验证已构建可执行文件的行为是否正确。

## 主要职责
- 创建临时仓库夹具，确保每个测试在干净、隔离的目录中运行。
- 执行 `dist` 目录中预构建的 ArchSpine CLI 二进制文件，测试 `sync` 命令的行为。
- 验证 CLI 输出是否正确，退出码是否符合预期。
- 每次测试后清理所有临时目录，防止产生副作用。

## 关键不变约束
- 测试**必须**使用位于 `dist` 目录中的已构建 CLI 二进制文件——不针对源代码或开发服务器运行。
- 每个测试**必须**在唯一的临时目录内执行，以保证隔离性。
- 文件**必须**是 Vitest 测试文件，后缀名为 `.test.ts`。

## 不涉及范围（负面范围）
- 本测试**不**对内部函数进行单元测试。
- **不**模拟 CLI 二进制文件或任何外部依赖。
- **不**测试 CLI 的构建或开发过程。
- **不**提供任何生产运行时功能。

## 导出/外部行为
本文件不导出任何公共 API。它纯粹是一个测试套件，作为 Vitest 运行器的一部分执行。其唯一的外部作用是验证 CLI 二进制文件的行为并报告通过/失败结果。
