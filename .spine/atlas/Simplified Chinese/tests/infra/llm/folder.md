# LLM 重试工具 — 错误分类测试套件

本目录包含 `isRetryableError` 函数的单元测试，该函数用于对 LLM 重试工具中的错误进行分类。测试验证哪些错误类型应触发重试、哪些不应触发，从而确保重试逻辑的健壮与正确。

## 重要子项

- **retry.test.ts** — 基于 Vitest 的测试文件，覆盖以下四种关键错误类别：
  - 与 socket 相关的错误（例如 `'socket hang up'`、`ECONNRESET`）——应判定为可重试。
  - 终止错误（例如 `'terminated'`）——应判定为可重试。
  - HTTP 429（限流）——应判定为可重试。
  - HTTP 400（错误请求）——应**不**可重试。

## 重要实现领域

测试的核心逻辑是重试工具内部的错误分类。正确区分暂时性错误（应触发重试）与永久性或客户端错误（应直接上报），对于构建弹性 LLM 客户端至关重要。每个测试用例均针对重试循环在实际运行中必须处理的真实场景。
