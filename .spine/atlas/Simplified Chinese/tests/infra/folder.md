目录 `tests/infra/llm-recovery` 包含一组聚焦的测试套件，用于验证 ArchSpine 系统中两个关键容错机制：**基于索引的断点恢复**与 **LLM 重试逻辑**。测试工件按如下两个主要区域组织：

1. **`index-recovery.test.ts`** – 这是一个 Vitest 测试套件。它通过为每个测试用例创建独立的临时目录、模拟 LLM 摘要任务失败、写入不完整的索引文件，来检验系统能否从部分损坏或损坏的索引状态中正确恢复。同时验证恢复流程中任务状态、遥测状态与协议版本号的集成情况，并在每次运行后清理所有测试工件并还原模拟。

2. **`tests/infra/llm/`** – 该子目录包含针对 LLM 重试工具错误分类函数的测试。这些测试确保重试逻辑能正确区分哪些错误应该触发重试（例如 socket 挂起、`ECONNRESET`、终止、HTTP 429），哪些不应重试（例如 HTTP 400）。

整体而言，这些测试保证了 ArchSpine 能够优雅地应对索引损坏以及 LLM 的临时故障，并具备正确的测试隔离、模拟仿真和清理能力。
