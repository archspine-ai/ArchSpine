# ArchSpine 测试套件：索引恢复与 LLM 重试

本 Vitest 测试套件用于验证 ArchSpine 系统中基于索引的恢复机制和 LLM 重试逻辑。它确保系统能够在摘要任务中优雅地处理故障和损坏状态。

## 主要职责

- **隔离测试环境**：为每个测试用例创建独立的临时目录，避免副作用，确保结果确定性。
- **模拟 LLM 故障**：模拟 LLM 摘要任务失败，以测试系统在受控条件下的重试逻辑。
- **恢复验证**：确认系统能够从部分或损坏的索引状态中正确恢复，恢复正常功能。
- **清理工作**：每次测试运行后移除所有测试工件，保持环境整洁。

## 范围与边界

- **涵盖范围**：使用模拟依赖项对恢复和重试逻辑进行单元级验证。
- **不涵盖范围**：摘要或索引恢复的生产实现；与真实 LLM 服务的集成测试；恢复机制的性能或负载测试。

## 不变规则

- 所有测试文件必须使用 `.test.ts` 或 `.spec.ts` 后缀，以符合项目约定。

## 架构意图

该套件旨在为索引恢复和重试机制提供稳健、确定性的测试覆盖，确保它们在故障场景下行为正确。最近的变更主要解决 lint 错误，并在 v1.0 发布前使测试结构与项目约定保持一致。

## 公开接口

本模块不暴露任何公共 API，仅作为内部验证的测试工具。
