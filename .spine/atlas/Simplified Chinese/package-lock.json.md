# ArchSpine 项目包锁定文件配置摘要

## 概述
此文件（`package-lock.json`）是 ArchSpine 项目的权威锁定文件，用于锁定每个直接和传递依赖项的精确版本，确保在所有环境中实现可重现的依赖解析和确定性构建。该锁定文件通过哈希值强制执行完整性检查，并确保仅安装满足项目引擎要求的包。

## 关键参数
- **`lockfileVersion`**：设为 `3`。此现代格式支持工作区和确定性安装，需要 npm v7 或更高版本。使用旧版 npm 将无法解析该文件。
- **`requires`**：布尔标志，设为 `true`，表示根包（`archspine`）已声明需要解析的依赖项。
- **`engines`**：定义所需的 Node.js 版本范围为 `>=20.18.1 <21 或 >=22`。如果运行时的 Node.js 版本不满足此约束，npm 将拒绝安装或运行该包。这是一个硬性门槛，是旧版 Node.js 部署失败的常见原因。
- **`bin`**：将 CLI 命令 `archspine` 和 `spine` 映射到 `dist/cli/index.js`。这些是安装后的全局入口点。

## 重要依赖
- **`@ast-grep/lang-c`**（及相关 `@ast-grep/lang-*` 包）：用于语言分析的 AST 解析器，对代码理解功能至关重要。
- **`@google/generative-ai`**：生成式 AI SDK，为工具中的 AI 交互提供支持。
- **`better-sqlite3`**：SQLite 绑定，用于本地缓存和持久化状态。
- **`vitepress`**（仅开发依赖）：文档站点生成器，不会在生产环境安装。
- **`vitest`**（仅开发依赖）：单元测试框架，通过自动化测试保证代码质量。

## 稳定性与风险
此锁定文件消除了不同环境间的版本差异，确保依赖树完全一致。但如果添加或更新依赖后未重新生成锁定文件，运行 `npm install` 会因完整性校验失败而报错。安全风险来自可能包含已知漏洞的过时包；建议定期运行 `npm audit` 并更新依赖。**engines** 约束强制了特定 Node.js 版本范围；任何不支持的版本（如 Node.js 18 或 21.x）都将导致部署失败。操作人员应在部署前验证运行时版本，并考虑通过 `.nvmrc` 或 Docker 固定 Node.js 版本。
