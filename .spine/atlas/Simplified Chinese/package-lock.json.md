# ArchSpine 包配置

## 角色
定义 ArchSpine 项目的运行时依赖和构建工具链，这是一个基于 AST 代码分析和 AI 集成的镜像系统。

## 关键职责
- **包依赖管理**：支持多种语言的 AST 解析（C、C++、Go、Java、Python、Rust）
- **AI 模型集成**：通过 Google Generative AI 和 OpenAI SDK 实现
- **本地数据库存储**：使用 better-sqlite3
- **CLI 工具入口点**：`archspine` 和 `spine`
- **开发工具**：代码检查（ESLint）、格式化（Prettier）、测试（Vitest）、文档（VitePress）和 TypeScript 编译

## 重要不变性
- Node.js 引擎版本必须 >= 20
- 所有生产依赖必须通过完整性哈希解析
- Lockfile 版本必须为 3 以确保可重复安装
- 许可证必须保持 Apache-2.0

## 负范围
此配置不定义任何运行时行为、应用程序逻辑或系统架构。它仅管理依赖图和构建工具链。

## 导出的行为
- **`dependencies`**：所有运行时必需的生产包，包括六种语言的 AST 解析器、AI SDK、SQLite 存储和 CLI 工具。
- **`devDependencies`**：构建和开发工具，如 TypeScript、ESLint、Prettier、Vitest 和用于文档的 VitePress。
- **`engines`**：指定运行项目所需的最低 Node.js 版本（>=20）。
- **`bin`**：定义 CLI 入口点 `archspine` 和 `spine`，它们映射到编译后的 JavaScript 文件。
- **`lockfileVersion`**：使用 lockfile 版本 3 确保确定性依赖解析。

## 稳定性与风险
此配置通过控制运行时可用的第三方库直接影响系统稳定性。版本锁定不当或缺少完整性检查可能导致供应链漏洞或运行时故障。广泛的 AST 解析器集合增加了攻击面，而对外部 AI API 的依赖引入了网络依赖风险。Lockfile 确保可重复构建，但任何对 package.json 的手动修改而未更新 lockfile 都可能导致环境不一致。
