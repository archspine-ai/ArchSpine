# ArchSpine 配置摘要

## 作用
`package.json` 文件定义了 ArchSpine 镜像治理系统的项目元数据、依赖关系图、可执行入口点和运行时要求。它编排了静态分析（AST-grep）、LLM 集成（Google Gemini、OpenAI）、持久化存储（better-sqlite3）、CLI 交互和文档生成的依赖树。它还通过 devDependencies（TypeScript、ESLint、Vitest、VitePress）配置了构建、测试和发布流水线，声明了两个 CLI 二进制文件（`archspine` 和 `spine`），并设置了最低 Node.js 引擎版本（>=20），以确保核心 API 的兼容性。

## 关键参数

- **`name`**：`archspine` — 在 npm 生态系统中唯一标识此包；包管理和注册表使用此字段。
- **`version`**：`1.0.0` — 初始发布版本。必须遵循语义化版本，以标明破坏性变更。
- **`bin`**：`archspine` 和 `spine` — 两个 CLI 入口点，必须指向编译后的 JavaScript 入口；配置错误会导致整个命令行界面崩溃。
- **`engines`**：`node >=20` — 运行时门控；如果用户尝试在更旧的 Node.js 上运行，npm 将警告或拒绝，从而强制执行系统要求。
- **`dependencies`**：生产依赖，如 `@ast-grep/*`、`better-sqlite3`、`@google/generative-ai`、`openai`、`undici` 等。这些直接影响功能；必须锁定版本范围以避免解析、数据库或 AI 连接方面的回归。
- **`devDependencies`**：TypeScript、ESLint、Vitest、VitePress — 控制开发工作流和文档构建。版本不匹配可能破坏 CI、构建或测试流水线。
- **`lockfileVersion`**：`3` — 确保跨环境的确定性安装。更改此设置可能导致依赖解析漂移，引发“在我机器上可以运行”的 bug。

## 稳定性与风险

此配置文件是根依赖清单。任何配置错误（如版本范围不正确、缺少锁文件、引擎不匹配）都可能导致完全构建失败、运行时崩溃，或未受审查的传递依赖带来安全漏洞。固定的引擎要求（`node >=20`）防止了静默的 API 兼容性破坏。双二进制入口必须保持一致；重命名或移除它们会静默破坏 CLI 用户体验。定期更新依赖至关重要——尤其是 `undici`、`node-fetch` 和 LLM SDK 等安全关键包。文档生成（VitePress）和测试（Vitest）的 devDependencies 影响文档生成和测试套件的可靠性，进而影响开发者信心和发布质量。
