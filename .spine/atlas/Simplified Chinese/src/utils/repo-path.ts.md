# ArchSpine – 仓库路径标准化工具

## 角色
内部工具函数，提供跨平台的仓库路径标准化，确保数据库键表示的一致性。

## 主要职责
- 将原始文件系统路径标准化为仓库相对逻辑路径。
- 将 Windows 反斜杠转换为 POSIX 正斜杠，实现跨平台兼容。
- 去除开头的 `./` 和 `/` 前缀，确保路径相对于仓库根目录。
- 为数据库查询和写入提供路径表示的唯一真实来源。

## 关键不变性
- 输出始终为 POSIX 风格字符串，使用正斜杠。
- 输出始终相对于仓库根目录（无前导斜杠或 `./`）。

## 排除范围（不负责）
- 路径验证或存在性检查。
- 处理仓库外部的绝对路径。
- URL 或 URI 路径标准化。

## 最重要的导出行为
- **`normalizeRepoPath`** – 唯一的公开函数，将任何原始文件系统路径转换为规范的、相对于仓库的 POSIX 路径，适合用作数据库键。
