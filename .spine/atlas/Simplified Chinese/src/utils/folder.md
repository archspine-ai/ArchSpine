`src/utils` 目录为 ArchSpine 系统提供了基础基础设施和工具模块，支撑文件同步、命令行交互和运行时协调。这些模块分为三个核心组：

- **Agent 指令工具** – 三个文件（`agent-instructions.sync.ts`、`agent-instructions.templates.ts`、`agent-instructions.ts`）负责 Agent 指令块的生命周期管理：将指令块同步到仓库配置文件、提供静态模板和忽略模式、通过 barrel export 暴露统一的公共 API。
- **命令行展示与交互** – `banner.ts` 渲染品牌启动横幅和版本信息；`confirm.ts` 实现交互式用户确认提示和终端输入处理。
- **核心基础设施服务** – `fs.ts` 提供原子写入和安全复制操作；`lock.ts` 实现基于文件的互斥锁，确保关键区间的独占访问；`git-hook.ts` 生成和管理 pre-commit Git 钩子 shell 脚本；`footprint.ts` 计算源码文件的确定性结构指纹；`repo-path.ts` 跨平台规范化仓库路径，用于一致的数据库键表示。

最重要的实现领域包括原子文件安全性（`fs.ts`）、排他锁定（`lock.ts`）、Git 钩子自动化（`git-hook.ts`）以及通过 `footprint.ts` 进行的结构变更检测。Agent 指令同步模块（`agent-instructions.sync.ts`）也处于核心位置，因为它直接管理 ArchSpine 配置如何在多个仓库目标中注入和移除。
