`src/` 目录是 ArchSpine 镜像系统的完整源代码根目录，包含所有核心模块与辅助资源，共同支撑代码分析、架构强制与镜像同步能力。

该目录围绕九个主要功能组进行组织，每组分工明确：

- **`src/assets/`** – 存放可复用的模板资源（文档模板、提示工程组件、配置规范、公共 API 表面文档），为一致性的开发与运维提供基础。
- **`src/ast/`** – 提供解析与语言发现基础设施，包括抽象语法树构建、符号与导入提取、基于文件扩展名的语言检测，以及语言特定的配置与规则集。
- **`src/cli/`** – 实现命令行界面层：参数解析、命令路由、帮助系统、UI 工具，以及 `build`、`sync`、`init` 等子命令的适配器。
- **`src/core/`** – 定义基础契约与运行时编排骨架：类型定义、配置验证模式、错误码、管道编排器、扫描策略、任务状态工厂/修改器，以及依赖注入上下文类型。
- **`src/engines/`** – 封装核心操作逻辑：文件系统扫描（含忽略规则与 Git diff 集成）、架构规则加载与匹配、依赖与关联分析（基于路径启发式与符号证据）、诊断上下文解析、违规修复提示生成、报告生成、健康与使用报告，以及脊柱索引聚合。
- **`src/infra/`** – 提供基础设施层：配置管理、SQLite 数据库生命周期、多提供商 LLM 客户端抽象、跨平台凭据存储、文件完整性/清单跟踪、模型上下文协议（MCP）服务器、提示组装、运行时 I/O，以及旨在检测未授权变更并管理修复操作的治理工具。
- **`src/services/`** – 编排多阶段管道（检查、修复、同步、LLM 管理、仓库管理、运行时执行配置文件、架构视图生成），涵盖会话生命周期管理与检查点可恢复性。
- **`src/tasks/`** – 编入端到端管道各阶段：扫描、AST 提取、摘要、验证、聚合、反向索引、视图推导、文档回填、状态提交与清理。
- **`src/types/`** – 提供所有共享接口与数据契约（如 SpineUnit、SpineIdentity、SpineConfig、语言支持、同步清单、架构规则、版本管理，以及视图工件），并通过 `protocol.ts` 提供稳定的公开导入门面，通过 `view.ts` 提供视图类型契约。
- **`src/utils/`** – 提供可复用的工具模块：文件同步、模板生成、终端交互、加密指纹、安全文件 I/O、Git 钩子管理与系统级锁定。

最关键的实现领域包括：AST 解析引擎（`ast/`）、核心运行时编排（`core/`）、扫描与规则执行引擎（`engines/`）、包含 LLM 与数据库集成的基础设施层（`infra/`），以及服务管道编排（`services/`）。这些模块共同构建了从代码发现到镜像同步的完整 ArchSpine 管道。
