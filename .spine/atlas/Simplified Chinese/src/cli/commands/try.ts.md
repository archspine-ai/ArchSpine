# ArchSpine – `try` 预览命令适配器

## 角色
该模块作为 `try` 预览命令的 CLI 命令适配器，负责验证文件系统、解析配置，并提供即时用户反馈。

## 主要职责
- 使用同步文件系统检查，验证指定目录是否存在且非空。
- 解析并验证 `archspine.json` 配置文件中的必需模式属性。
- 根据验证结果，向控制台输出用户指导和错误信息。
- 协调命令流程，包括显示 UI 横幅和委托使用错误处理。

## 重要不变性与负面范围
- **必须保持为薄 CLI 适配器。** 所有非 UI 逻辑应委托给引擎或服务。
- **不得包含业务规则或领域逻辑。** 该模块不应实现扫描、分析或转换管道。
- **必须依赖注入的工具** 进行文件系统和配置操作，以便于测试。
- **不在范围内：** 扫描或分析管道的业务逻辑、超出基本文件读取的持久化操作、网络或外部服务通信，以及复杂的数据转换或编排。

## 公开接口
- `ExecuteTryCommandOptions` 接口

## 规则违规
- **cli-entrypoint-separation（错误）：** 该模块直接执行文件系统 I/O（`fs.existsSync`、`fs.readdirSync`、`fs.readFileSync`）和 JSON 解析，这些属于管道/持久化逻辑。根据规则，此类逻辑应属于服务、核心、引擎或基础设施层，而非 CLI 入口点。
