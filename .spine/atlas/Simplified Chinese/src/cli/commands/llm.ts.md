# ArchSpine – LLM 配置 CLI 适配器

## 角色
该模块是一个 **CLI 命令适配器**，提供用于配置 LLM 提供商、模型和运行时设置的交互式工作流。它作为薄表示层，将所有持久化和验证委托给服务和基础设施层。

## 主要职责
- 提供交互式提示，用于选择 LLM 提供商、模型、基础 URL 和 API 密钥。
- 配置运行时设置，包括提示策略层级、验证策略、精简模式和实验性分阶段验证。
- 将 LLM 配置和 API 密钥的持久化委托给 LLM 管理服务。
- 导出 LLM 配置写入器和命令执行选项的类型定义。

## 重要不变性与负面范围
- **必须保持为薄 CLI 适配器** – 不得吸收管道或持久化逻辑。
- **不直接持久化** – 所有配置数据均交由 LLM 管理服务处理。
- **不执行 LLM 推理** – 该模块不执行补全、流式或任何 LLM 调用的运行时操作。
- **不处理认证** – API 密钥的验证或授权不在其范围内。
- **使用 prompts 库** 处理所有交互式用户输入。

## 公开接口
- `LLMConfigWriter` 类型（`Pick<Config, ...>` 子集）
- 用于 LLM 设置的交互式提示函数

## 架构意图
提供交互式 CLI 工作流，通过将持久化和验证委托给服务和基础设施层，保持 CLI 层的简洁。最近的变更专注于解决 lint 错误并在 v1.0 前完成管道修复，确保适配器保持正确的分离。
