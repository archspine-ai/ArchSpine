# ArchSpine 配置命令适配器

本模块是 ArchSpine 系统中用于配置管理的**轻量 CLI 适配器**。其唯一职责是在终端用户命令与底层配置服务之间建立桥梁，自身不包含任何业务逻辑。

## 角色

它是读取和写入配置值的命令行入口点。它将用户提供的参数转换为服务调用，并将结果格式化为控制台输出。

## 主要职责

- 定义 `ExecuteConfigCommandOptions` 接口，该接口规定了配置操作命令选项的形态。
- 根据支持的配置模式（例如 `llm.provider`、`llm.model`）解析和验证配置键。
- 将所有配置的读取和修改操作委托给配置服务，具体使用 `getPreCommitResolution`、`getHookSyncMode`、`getSupportedValue` 和 `setSupportedValue` 等方法。
- 使用 CLI 工具将配置值格式化并输出到控制台，为用户提供清晰的反馈。

## 重要不变性

- **必须保持为薄适配器。** 仅处理命令行参数解析、验证和输出格式化。
- **必须将所有配置状态操作委托给配置服务。** 不得实现任何存储或业务逻辑。
- **不得包含**配置解析或持久化的业务逻辑。

## 不涉及范围（负面范围）

- 实现配置存储或业务逻辑（此部分委托给配置服务）。
- 直接管理流水线执行或持久化操作。
- 提供超出简单控制台输出的交互式用户界面。

## 公开表面

- `ExecuteConfigCommandOptions` 接口——主要的导出类型，定义了命令选项的契约。

## 架构意图

本模块遵循严格的关注点分离原则：CLI 模块作为入口点和适配器，所有核心逻辑保留在专用服务中。这确保了 CLI 保持轻量、可测试且易于维护。
