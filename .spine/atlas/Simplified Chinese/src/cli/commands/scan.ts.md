# ArchSpine – `scan` 命令处理器

## 角色
`scan` 操作的 CLI 命令处理器，负责协调源代码扫描和干运行报告。

## 主要职责
- 定义扫描命令选项的接口，包括根目录和配置。
- 解析命令行参数以检测 `--dry-run` 标志。
- 调用 Scanner 引擎执行扫描操作。
- 当存在干运行标志时，格式化并输出干运行报告（将格式化工作委托给 Scanner 引擎）。

## 重要不变性与负面范围
- **必须保持为薄命令适配器** – 扫描和报告逻辑属于引擎，不在此处。
- **不在范围内**：实现扫描逻辑（委托给 Scanner 引擎）、管理配置加载（委托给配置基础设施）、持久化扫描结果、处理非 CLI 执行路径。

## 公开表面
- `ExecuteScanCommandOptions` 接口
- `executeScanCommand` 函数

## 架构意图
将 CLI 入口点与核心扫描逻辑分离，以保持运行时的可重用性。
