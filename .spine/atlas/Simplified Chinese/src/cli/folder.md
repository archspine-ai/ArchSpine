# `src/cli` — 命令行入口与用户交互界面

`src/cli` 是 ArchSpine 镜像系统的命令行入口，负责处理所有终端操作，包括项目初始化、仓库配置、日常同步以及系统诊断。

## 重要子模块

- **`index.ts`** – 主命令路由器。启动运行环境（配置、密钥、LLM 设置），解析 `process.argv`，然后分派到对应的处理器模块（例如 `init`、`sync`、`build`、`publish`、`check`、`fix`、`scan`、`history`）。同时配置全局 HTTP 代理并显示系统横幅。

- **`cli-utils.ts`** – 一系列 UI 展示辅助工具，用于统一的命令输出：横幅显示（完整/精简/静默）、文本换行、值解析、错误格式化以及分步进度消息。

- **`help.ts`** – 渲染通用帮助（按类别列出所有命令）和针对特定命令的详细帮助，如 `init`、`sync`、`view` 等。

- **`document-languages.ts`** – 定义 `DocumentLanguageChoice` 类型以及构建支持文档语言列表的逻辑，包括为高容量语言添加分隔符。

- **`commands/`** – 为每个 CLI 子命令存放专用的处理器模块；`index.ts` 中的路由器在解析参数后将执行委托给这些模块。

- **`init/`** – 编排完整的项目初始化流程：制品策略选择、规则模板安装、Git 钩子、LLM 密钥设置、文件系统扫描、语言发现、清单更新，以及向构建工作流的交接。

- **`repo/`** – 提供 CLI 适配器，用于检查和设置仓库制品策略，并附带进度反馈和结果展示。

## 核心实现领域

- **命令路由与帮助** – 由 `index.ts` 集中解析参数并调度，配合 `help.ts` 提供全面的帮助文本，及时追踪实验性命令。
- **UI 格式化** – 借助 `cli-utils.ts` 实现终端输出的统一风格与体验。
- **初始化流程** – 通过 `init/` 模块完成从零开始搭建 ArchSpine 项目的多步引导。
- **仓库配置** – 通过 `repo/` 适配器以交互方式管理制品策略。
