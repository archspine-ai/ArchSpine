# ArchSpine 基础设施桶导出

## 角色
基础设施子系统的公共 API 外观（桶导出），聚合并重新导出所有基础设施层模块。

## 主要职责
- 集中并暴露基础设施层的公共契约给其他子系统
- 为需要基础设施服务（配置、文档、语言、清单、规则、版本）的消费者提供单一导入点

## 不涉及范围
- 不包含任何业务逻辑或实现细节
- 本身不定义或强制执行任何架构规则

## 重要不变性
- 所有重新导出的模块必须存在且为有效的 TypeScript 模块
- 桶导出不得引入循环依赖

## 公共表面
- `config.js`
- `index-documents.js`
- `languages.js`
- `manifest.js`
- `rules.js`
- `versions.js`

## 架构意图
为基础设施层提供清晰、集中的公共 API 表面，使其他子系统能够导入基础设施服务而无需耦合到内部模块路径。此文件未检测到更改；最近的代码重构（c4ed85e）很可能建立了这种外观模式，作为子系统边界强制实施的一部分。
