`src/types/protocol` 目录是整个 ArchSpine 镜像系统的类型基础。它定义了所有核心数据结构的规范模式与接口，涵盖配置、文档、语言、清单、规则、版本和视图。这些类型契约为所有子系统提供了稳定的类型基础，确保数据的一致性与互操作性。

该目录下的主要子结构分为两个关键部分：

- **`protocol.ts`**：公共协议外观模块，提供单个稳定的导入路径来获取所有协议类型。它负责重新导出内部 `./protocol/index.js` 模块中的所有类型定义，将外部使用者与内部模块结构解耦。
- **`view.ts`**：专用类型定义模块，为视图生成系统建立视图工件契约。它定义了视图信封、内容结构、标识符（ViewId、ViewType）、架构图规范（节点、边、摘要卡片）以及特定视图项类型（公共表面、风险热点）的 TypeScript 接口。

在内部，`protocol/` 子目录（通过 `protocol.ts` 和 `./protocol/index.js` 访问）集中存放了配置、文档、语言、清单、规则和版本的类型定义。这种模块化分离使类型系统既易于扩展又便于维护。

最重要的实现领域包括：（1）提供稳定 API 表面的协议外观模块；（2）驱动架构图和风险摘要生成的视图类型契约；（3）为验证和镜像工作流奠定基础的规则与清单的规范模式。
