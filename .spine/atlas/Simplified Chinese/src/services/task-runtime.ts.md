# ArchSpine – 核心服务编排器

## 角色
该模块作为 ArchSpine 系统的核心服务编排器，负责准备并协调所有任务执行引擎及支撑基础设施，确保分析流水线在策略合规且线程安全的环境下运行。

## 主要职责
- 从配置和环境设置中解析提示策略层级与验证策略。
- 创建并初始化执行流水线的任务状态与工件状态。
- 编排扫描器、聚合器、规则引擎、上下文引擎及 LLM 客户端，以执行协同分析。
- 管理锁与扫描策略，保障线程安全且符合策略的操作。

## 不涉及范围
- 视图相关的渲染或 UI 逻辑。
- 直接的低层级文件 I/O 操作。
- 各分析引擎（如扫描器、聚合器、规则引擎）的内部实现。

## 不变约束
- 必须编排核心、引擎及基础设施模块，但不实现其内部逻辑。
- 必须从环境和基础设施中解析配置策略（提示、验证、扫描）。
- 必须生成供下游执行使用的 `PreparedTaskContext`。

## 公开接口
- `ServiceOptions` 接口
- `PreparedTaskContext` 接口

## 变更意图
架构意图是提供一个集中化的服务层，用于准备和协调所有分析引擎与基础设施，确保策略合规与状态管理。最近的变更表明对同步模式与回填功能进行了增强，可能影响编排逻辑。
