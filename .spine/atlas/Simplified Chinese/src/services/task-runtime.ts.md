# ArchSpine – TaskContextFactory（任务上下文工厂）

**角色：**  
服务级别的工厂和编排器，为分析管道准备完全配置的任务上下文。

**职责：**  
- 从选项和环境解析提示策略层级和验证策略。  
- 创建并初始化任务状态和工件状态，供执行管道使用。  
- 实例化并协调扫描器、聚合器、AST提取器、规则引擎、上下文引擎和输出管理器。  
- 管理锁管理器和扫描策略，确保线程安全和策略合规的操作。  
- 构造并返回封装所有已初始化组件的 `PreparedTaskContext`。

**不在范围内：**  
- 直接与 LLM 客户端交互（仅导入类型 `LLMClient`）。  
- 执行扫描、分析或输出——本文件仅负责上下文准备。  
- 处理视图特定的服务逻辑（应位于 `src/services/view/` 下）。

**不变约定：**  
- 解析后的提示策略层级必须与推导出的任务模式（`validate` 或 `summarize`）匹配。  
- 任务状态和工件状态必须在任何引擎实例化之前创建。  
- 清单文件必须在使用前通过 `rootDir` 和 `scanPolicy` 打开。  
- `LockManager` 在初始化期间确保独占访问。

**公开接口（最重要的导出）：**  
- `createTaskContext(...)` – 主要的入口点。  
- `ServiceOptions` – 工厂消费的配置类型。  
- `PreparedTaskContext` – 封装所有组件的返回对象。

该模块是核心工厂，确保所有必需的分析组件和策略在管道开始工作之前正确装配。它本身不执行任何分析，但保证每个依赖项都已初始化且保持一致。
