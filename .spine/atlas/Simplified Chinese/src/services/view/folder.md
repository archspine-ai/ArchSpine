`src/services/view` 目录实现了 ArchSpine 系统的**视图层**，负责从已索引的项目元数据生成和渲染架构可视化与分析报告。它提供了一个统一的子系统，用于定义、评分、生成和渲染架构视图，例如架构图、风险热点和公共表面分析。  

关键子模块按功能分组：  
- **图表生成**：`architecture-diagram-view.ts` 利用基于 LLM 的规范生成功能产生交互式图表规范，而 `arch-diagram-renderer.ts` 是一个纯工具，将这些规范转换为 SVG 标记。  
- **评分与分析**：`risk-hotspots-view.ts` 从索引的代码单元计算架构风险热点；`public-surface-view.ts` 根据源文件成为公共 API 表面的可能性对其进行排序。  
- **渲染与报告**：`view-renderer.ts` 消费视图产物（风险热点、模块条目、CLI 条目、公共表面），并使用基于文件系统加载的模板将其渲染为格式化 Markdown 报告。  
- **注册表与运行时**：`view-registry.ts` 集中定义所有可视化视图的类型定义和注册；`view-runtime.ts` 解析实验性视图层和已启用视图的配置。  
- **基础设施与实用工具**：`index-loader.ts` 从 `.spine/index` 目录加载并缓存已索引的 Spine 单元；`common.ts` 提供评分和路径抑制功能；`types.ts` 定义带有行数的元数据包装。  
- **公共入口**：`index.ts` 为 MCP 资源层重新导出所有视图特定的运行时和渲染组件。  

最重要的实现领域包括：基于 LLM 的图表生成、SVG 渲染、风险评分启发式算法、公共表面分析算法以及基于模板的 Markdown 报告生成。具体子模块如 `architecture-diagram-view` 和 `risk-hotspots-view` 是视图推导的主要入口点。
