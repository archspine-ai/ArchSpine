# ArchSpine – 视图子系统公共外观模块（`resources.ts`）

## 角色
该文件是视图子系统的**公共外观模块**。它在 MCP（模型上下文协议）资源层内集中导出所有视图特定的运行时和渲染组件，为外部调用者（尤其是 MCP 客户端）提供稳定的导入接口。

## 主要职责
- **聚合并重新导出**视图子系统的所有公共模块，提供单一、稳定的入口点。
- **封装内部结构**，使得 `src/services/view/**` 下的实现细节可以独立演化，而不影响外部消费者。
- **作为架构可视化和分析视图的主要 MCP 资源入口点**。

## 重要不变性
- 仅包含视图子系统公共模块的**重新导出**，不允许内联实现。
- 不得**依赖视图子系统的内部未导出模块**，以保持封装性。
- 导出表面是**稳定的**；对导出的任何更改都应视为对外部消费者的破坏性变更。

## 负面范围（不涉及）
- **不**实现视图特定的业务逻辑或渲染逻辑（委托给导出的模块）。
- **不**处理 HTTP 请求或直接用户交互（由上游服务或 MCP 服务器处理）。
- **不**管理数据持久化或核心领域操作（由其他子系统负责）。

## 公共表面（导出的模块）
- `arch-diagram-renderer.js`
- `architecture-diagram-view.js`
- `common.js`
- `index-loader.js`
- `public-surface-view.js`
- `risk-hotspots-view.js`
- `types.js`

## 架构意图
该外观将外部消费者与内部重构解耦，与解决层反转的更大规模重构保持一致。最近的变更整合了视图导出，以提供稳定的接口。

## 规则违反（警告）
- **ID：** `service-runtime-boundary`
- **原因：** 此视图特定服务模块位于 `src/infra/mcp/` 下，而非 `src/services/view/**`，模糊了运行时编排边界。
