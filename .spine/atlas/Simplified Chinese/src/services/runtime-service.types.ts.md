# ArchSpine – `RuntimeServiceDraft` 接口摘要

## 角色
该 TypeScript 接口在 ArchSpine 类型系统中定义了一个**占位边界**，用于未来运行时服务能力的扩展。它是一个设计阶段的契约，预定义了运行时服务的形状，但不承诺任何具体实现。

## 主要职责
- 声明一个草案接口，用于集中未来运行时服务的方法签名，涵盖 LLM 设置和核心服务。
- 作为设计边界，防止在主要运行时文件（`infra/llm/runtime.ts`）中扩展提供者解析逻辑。

## 关键不变性
- 必须保持为**接口**（不能是类或具体实现）。
- 必须位于 `infra/llm` 模块内，以作为运行时系统的边界。

## 负面范围（不在范围内）
- **不**实现具体的运行时服务。
- **不**处理提供者解析或实例化。
- **不**直接被视图或 UI 层使用。

## 最重要的导出行为
- 导出 `RuntimeServiceDraft` 接口作为其唯一的公开表面。
- 该接口旨在由未来的运行时服务类实现，但目前仅作为类型级别的占位符。

## 变更意图
- **架构意图：** 定义一个稳定的接口边界，以封装未来的运行时服务编排，将契约与实现分离。
- **近期变更意图：** 该文件近期无变更；Git 提交信息 `feat: freeze runtime and prompt engine phases` 表明运行时接口已趋于稳定，与该文件的目的一致。
