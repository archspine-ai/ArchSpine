# ArchSpine – 相对导入路径解析器

## 角色
扫描引擎的路径解析工具，专门用于解析源代码树中的相对导入目标。

## 主要职责
- 根据给定的根目录和包含目录，将相对导入源字符串解析为绝对文件系统路径。
- 查询语言注册表以确定有效的源文件扩展名，用于存在性检查。
- 对候选路径执行文件存在性和类型验证（文件 vs 目录）。
- 如果找到有效的源文件，则返回相对于根目录的路径；否则返回 null。

## 重要不变性
- 仅处理以 `.` 开头的导入源。
- 必须依赖 `LangRegistry` 获取语言特定的源文件扩展名。
- 必须验证解析后的路径存在且为文件。
- 必须与 CLI 入口点和表示逻辑解耦（遵循引擎独立性规则）。

## 排除范围（不负责）
- 处理非相对导入（例如包导入）。
- 模块打包或依赖图构建。
- CLI 参数解析或用户交互。
- 服务级编排或状态管理。

## 最重要的导出行为
- **`resolveRelativeImportTarget(rootDir: string, dir: string, source: string): string | null`**  
  这是唯一的公开函数。它接受根目录、包含目录和相对导入源字符串，如果找到有效的源文件，则返回相对于根目录的解析路径，否则返回 null。
