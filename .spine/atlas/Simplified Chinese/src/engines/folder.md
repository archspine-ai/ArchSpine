# `src/engines` — 核心引擎模块

`src/engines` 目录是 ArchSpine 镜像系统的运行中枢，包含了实现文件扫描、规则实施、依赖分析及报告生成的所有引擎组件。这些模块按功能分为以下几个区域：

- **扫描器基础设施**（`scanner.ts`、`scanner-git.ts`、`scanner-utils.ts`）：`scanner` 引擎利用分层忽略规则和 Git diff 集成发现、过滤并报告仓库文件。辅助模块提供 Git 执行支持及路径/模式归一化功能。
- **上下文与规则解析**（`context.ts`、`context-path-resolver.ts`、`context-relevance.ts`、`rules.ts`）：`context` 引擎为依赖分析解析架构上下文；`context-path-resolver` 专用于解析相对导入目标；`context-relevance` 从规则文本中提取关键词；`rules` 负责加载、存储架构规则并根据文件路径进行匹配。
- **违规修复系统**（`fix-prompt.ts`、`fix.ts`）：`fix-prompt` 生成用于提出架构违规修复建议的大语言模型提示词；`fix` 是修复服务模块的公开门面。
- **报告与健康检查**（`god.ts`、`info.ts`、`usage.ts`）：`god` 生成全面的“上帝模式”架构档案；`info` 检查同步状态和系统健康度；`usage` 从清单数据库生成格式化的使用与审计报告。
- **聚合与门面**（`aggregator.ts`、`check.ts`）：`aggregator` 遍历脊柱文件系统以构建结构化的 `SpineUnit` 集合；`check` 重新导出检查子系统供 CLI 和服务使用。

需要关注的具体子模块包括扫描引擎（`scanner.ts`）、上下文解析引擎（`context.ts`）、规则引擎（`rules.ts`）、聚合器（`aggregator.ts`）以及上帝模式报告器（`god.ts`）。这些模块共同构成了镜像和分析仓库架构的端到端流水线。
