# ArchSpine 核心引擎模块

该目录包含 ArchSpine 镜像系统的核心引擎，提供扫描、上下文解析、规则执行和报告生成功能。这些组件共同实现了系统的核心操作逻辑，包括：基于忽略规则和 Git 差异集成的文件系统扫描、架构规则加载与匹配、利用路径启发式和符号证据的依赖与相关性分析、用于架构诊断的上下文解析、违规修复提示与报告生成、系统健康与使用情况报告，以及将 Spine 索引数据聚合成结构化单元集合。

## 主要组件

该模块按功能分为以下几个组：

### 扫描与文件发现
- **`scanner.ts`** — 核心文件系统扫描器，通过分层忽略规则（`.gitignore`、`.spineignore`）、`picomatch` 模式匹配以及 Git 差异集成来发现、过滤和报告仓库文件，支持增量扫描。生成包含完整文件和变更文件列表的 `ScanResult` 对象。
- **`scanner-utils.ts`** — 工具模块，提供路径规范化、基于 `picomatch` 的模式匹配以及扫描操作的试运行报告格式化功能。
- **`scanner-git.ts`** — Git 命令执行客户端接口及默认实现，委托给 `child_process.execFileSync`。

### 规则引擎
- **`rules.ts`** — 加载、存储和匹配架构规则（`SpineRuleDocument`），使用 glob 模式（包括通过 `!` 取反）针对文件路径进行匹配。利用 `picomatch` 进行高效模式匹配。

### 上下文与相关性分析
- **`context.ts`** — 架构上下文解析引擎，解析相对导入目标、从源文件骨架中提取规则关键词、计算相关性分数，并为依赖候选生成结构化诊断信息。
- **`context-path-resolver.ts`** — 路径解析工具，将相对导入源字符串解析为绝对文件路径，并咨询 `LangRegistry` 以确定有效扩展名。
- **`context-relevance.ts`** — 相关性评分引擎，对依赖候选和目标路径使用规则派生关键词、符号证据和路径距离启发式算法进行综合评分。

### 报告与诊断
- **`fix.ts`** 和 **`fix-prompt.ts`** — 修复服务的公共 API 外观模块，以及用于架构违规修复的 LLM 提示模板生成器。
- **`check.ts`** — 公共 API 桶模块，重新导出 `CheckService`、`runCheck` 和 `ValidationSummary`。
- **`info.ts`** — 生成全面的系统信息与健康报告，检查清单、配置、密钥和 LLM 配置，检测受保护输出变更并报告同步状态。
- **`usage.ts`** — 从 Manifest 数据存储生成格式化的使用情况与审计报告，包括令牌计数、预估成本和违规详情。
- **`god.ts`** — CLI 编排工具，从 `.spine/index` 生成全面的架构报告（上帝模式报告），输出包含文件分类账条目的 Markdown 报告。

### 聚合
- **`aggregator.ts`** — 核心引擎类，遍历 `.spine/index` 和 `.spine/atlas` 目录，读取并验证 Spine 索引文档，构建结构化的 `SpineUnit` 集合（如 `SpineFolderUnit`、`SpineProjectUnit`），供下游同步和视图操作使用。

## 关键实现领域

- **扫描与忽略逻辑**：扫描子系统集成了多层忽略规则和 Git 差异以实现增量更新。
- **规则匹配与执行**：规则引擎支持丰富的 glob 模式和取反，以实现精确的架构规则应用。
- **上下文解析**：上下文管道解析导入、提取关键词并评分相关性，提供可操作的诊断信息。
- **报告**：多个报告模块涵盖健康、使用、违规和全面架构报告。
- **聚合**：聚合器将原始脊柱数据转换为结构化集合，供同步和视图层使用。
