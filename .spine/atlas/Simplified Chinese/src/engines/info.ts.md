# ArchSpine – 同步与健康检查 CLI 模块

## 角色
该模块提供 CLI 命令，用于检查项目同步状态、验证受保护输出以及报告系统整体健康情况。它是一个面向用户的诊断工具，而非引擎或编排器。

## 主要职责
- 加载并检查项目清单，获取同步状态、语言快照和索引单元计数。
- 使用 spine-gate 检测验证受保护输出的变更，并格式化警告信息供用户查看。
- 通过运行时 IO 报告同步完成状态、反向索引状态和总使用统计信息。
- 与运行时服务集成，解析 LLM 设置和执行配置文件，以进行上下文感知检查。

## 不涉及范围
- 执行实际的同步操作或修改项目清单。
- 直接进行 LLM 推理或内容生成。
- 编排多个引擎或执行分析转换。

## 不变约束
- 必须保持为 CLI 命令模块，而非引擎。
- 依赖于基础设施模块（配置、清单、密钥、运行时 IO）和运行时服务进行编排。
- 不应导入 CLI 入口点或运行时 IO 之外的表示层。

## 架构意图
该模块提供面向用户的 CLI 命令，用于健康检查和同步状态报告，利用运行时服务和基础设施实现一致的项目状态检查。最近的代码重构侧重于建立子系统外观并解决层反转问题，使该模块与引擎逻辑的分离更加清晰。

## 公开接口
未定义明确的公开接口；所有行为通过 CLI 调用暴露。
