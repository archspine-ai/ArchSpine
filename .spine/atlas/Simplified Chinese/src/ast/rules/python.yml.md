# ArchSpine Python AST 提取规则

## 目的

本文档定义了 ArchSpine 扫描器用于从 Python 源文件中提取**导入**、**导出**和**使用关系**的语法模式。它是语言特定的规则集，支撑 ArchSpine 构建 Python 项目语义索引的能力。如果没有这些规则，扫描器将无法识别 `import` 语句、`def` 和 `class` 定义等 Python 代码的基本结构。

## 目标读者

- **参与 ArchSpine AST 提取模块开发的维护者** – 本文档是 Python 代码分析的唯一权威依据。  
- **希望自定义或扩展 Python 语言支持的项目负责人** – 规则以 YAML 模式 DSL 编写，可通过添加或修改条目来调整，无需改动扫描器核心逻辑。  

这些规则遵循典型的 Python 编码约定（如 PEP 8 风格的导入、函数定义等）。

## 本文锚定的关键决策

- 扫描器同时处理**标准导入**（`import X`）和 **from‑导入**（`from A import B`）。  
- 覆盖所有常见的 Python 导出结构：**函数**、**类**、**异步函数**和**模块级变量**。也包含特殊的 `__all__` 列表模式。  
- 使用检测仅限于**直接调用**和**属性访问**；动态或反射式使用（例如 `getattr`）明确不在范围之内。  
- 规则集是可扩展的 – 可通过添加新的模式条目来支持更多语法形式。

## 锚定的工作流程

- **代码分析流水线** – 提取规则是 Python 分析工作流的第一个步骤，生成包含导入、导出和使用关系的语义索引。  
- **依赖图生成** – 提取出的关系数据供 ArchSpine 的镜像系统可视化和查询工具使用。  
- **规则维护** – 任何对 Python 语法支持的变化（例如添加装饰器模式）都从更新本文档开始。

## 模式概览

规则分为三类：

- **导入** – `from_import`（`from $SOURCE import $$$SYMBOLS`）和 `simple_import`（`import $$$SYMBOLS`）。  
- **导出** – `function`、`class`、`async_function`、`variable` 以及 `__all__` 列表。  
- **使用** – `call`（`$NAME($$$ARGS)`）和 `property`（`$OBJ.$NAME`）。

这些模式通过一个简单的 DSL 表达，使用 `$` 前缀的占位符和 `$$$` 表示多部分捕获。本文档不涉及其他编程语言（TypeScript、Go、Rust 等），也不涵盖运行时行为或文档生成等内容。
