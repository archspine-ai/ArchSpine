# ArchSpine 镜像系统模式配置

## 目的

本文档定义了 ArchSpine 镜像系统的模式配置，用于从 Go 源代码中自动提取结构和关系信息。它描述了解析 Go 代码并识别导入、导出和使用等语义元素的规则和模式。

## 目标读者

本文档面向维护 ArchSpine 或集成其解析能力的开发者。这是一份技术规范，将 Go 语法结构映射到通用模型，驱动代码分析和文档生成工作流。

## 要点总结

- 配置使用基于模式的 DSL 来描述 Go 代码结构。
- **导入**、**导出**和**使用**是提取的三个核心类别。
- 每个模式包含标识符、类似正则的模板以及可选的类型分类。
- 系统支持单行和分组两种导入声明。
- 导出模式涵盖了 Go 的主要声明类型：函数、结构体、接口、类型别名、变量和常量。

## 本文档锚定的决策与工作流

- **模式定义**：在提供的上下文数据中，详细指定了匹配导入语句（`import $SOURCE`）、分组导入、函数声明、结构体类型、接口、类型别名、变量、常量、函数调用和选择器表达式的模板。这些模式是所有提取逻辑的基础。
- **分类规则**：每个导出模式包含一个 `kind` 字段（例如 Function、Class、Interface、Type、Variable），用于决定提取的元素在通用模型中的分类方式。
- **范围边界**：本文档明确排除了 Go 以外的语言语法、运行时行为、项目特定逻辑以及文档的呈现/样式，确保只关注解析规则。

## 模式概览

提供的上下文数据定义了以下模式组：

- **导入**：两个模式——单行导入（`import $SOURCE`）和分组导入（`import ( $$$SYMBOLS )`）。
- **导出**：六个模式——函数、结构体、接口、类型别名、变量和常量，每个都有独特的标识符和模板。
- **使用**：两个模式——函数调用（`$NAME($$$ARGS)`）和选择器表达式（`$OBJ.$NAME`）。

这些模式构成了 ArchSpine 生态系统中自动化代码分析和文档生成的基础。
