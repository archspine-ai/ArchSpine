# 文档摘要：ArchSpine 的 Go 语言 AST 提取规则

## 目的

本文档定义了 ArchSpine 抽象语法树提取器用于解析 Go 源代码的导入、导出和引用的语言模式。它是多语言规则集的一部分，帮助镜像系统建立项目的结构性认知。

## 目标读者

本文件面向 ArchSpine 抽象语法树引擎的维护者以及添加或更新语言提取规则的贡献者。每当需要支持新的 Go 语法或惯用法时应对其进行审查。

## 锚定的关键决策与工作流

- **导入检测**：通过两种不同的模式（`import` 用于单行导入，`import_group` 用于带括号的分组导入）捕获导入。
- **导出提取**：从以下位置提取导出：
  - 函数（`func $NAME...`）
  - 结构体（`type $NAME struct...`）—— 标记为 `Class` 类型
  - 接口（`type $NAME interface...`）—— 标记为 `Interface` 类型
  - 类型别名（`type $NAME = $VAL`）—— 标记为 `Type` 类型
  - 变量（`var $NAME = $VAL`）—— 标记为 `Variable` 类型
  - 常量（`const $NAME = $VAL`）—— 标记为 `Variable` 类型
- **引用捕获**：两种模式覆盖函数调用（`$NAME(...)`）和选择器表达式（`$OBJ.$NAME`），支持检测函数调用和方法/字段访问。
- **模式语法**：所有规则采用基于 YAML 的模板匹配语法，使用命名捕获组（如 `$NAME`、`$$$SYMBOLS`），与多语言框架保持一致。

## 不涉及的范围

- 其他编程语言（TypeScript、Python、Java 等）
- 高层架构或命名约定规则
- 运行时行为或执行语义
- 代码转换或修复逻辑
