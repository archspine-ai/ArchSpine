# ArchSpine 模式提取规则文档摘要

## 文档目的
本文档定义了支撑 ArchSpine 镜像系统的模式提取规则。它规定了系统如何从源文件中发现并呈现代码结构 —— 包括导入、导出（函数、结构体、变量）以及用法（调用、字段访问、指针字段访问）。这些规则是构建精准、自动同步的代码镜像的配置基础。

## 目标读者
主要读者是 **ArchSpine 系统的开发者和维护者**，他们需要理解、定制或扩展提取逻辑。对于希望适配镜像系统到新语言或代码库的集成人员而言，本文档同样有用 —— 他们可以通过修改模式定义来适应不同场景。

## 本文档锚定的关键决策
- **导入检测**：`#include` 模式（例如 `#include $SOURCE`）规定了定位外部依赖的标准。
- **导出规则**：针对函数、结构体和变量定义了三类不同的模式，用以识别代码元素作为公开 API 的组成部分。
  - 函数：`$RET $NAME($$$ARGS) { $$$BODY }`
  - 结构体：基于 `struct_specifier` 和类型标识符的语法规则。
  - 变量：`$TYPE $NAME = $VAL;`
- **用法模式**：三种模式（调用、字段访问、指针字段访问）规定了如何追踪代码交互，从而在镜像内建立连接。
  - 调用：`$NAME($$$ARGS)`
  - 字段访问：`$OBJ.$NAME`
  - 指针字段访问：`$OBJ->$NAME`

## 支撑的工作流
本文档中定义的模式直接被 ArchSpine 镜像构建引擎消费。当开发者运行镜像生成流程时，这些规则驱动对源文件的分析，生成一个关于导入、导出符号和用法关系的结构化表示。任何对这些模式的更改都会立即影响镜像的覆盖范围和准确性。
