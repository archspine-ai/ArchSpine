# 文档概要：ArchSpine 的 Java AST 提取规则

## 目的
本文档是一个机器可读的规则集，用于指导 ArchSpine 如何解析 Java 源代码并提取结构化知识。通过定义导入、导出（类和接口）以及使用（方法调用和实例化）的模式，它使系统能够构建代码依赖关系、公共 API 表面和使用关系的详细索引。这是 ArchSpine 自动分析、可视化和治理代码库的基础构建模块。

## 读者对象
该规则文件面向为 ArchSpine 语言支持做出贡献的开发者，以及读取这些模式以执行静态分析的 ArchSpine 运行时引擎。它并非面向最终用户，而是 `.spine/atlas` 和 `.spine/index` 目录中的一个配置工件。主要受众是维护 ArchSpine AST 提取功能的开发团队以及任何为添加新语言支持做出贡献的人。

## 关键要点
- 该文件以紧凑的 YAML 格式定义了 Java 特定的 AST 提取规则。
- 它涵盖三个类别：导入（imports）、导出（exports，包括类和接口）和使用（usages，包括方法调用和实例化）。
- 模式使用模板语言，包含 `$NAME`、`$BODY` 和 `$ARGS` 等占位符来匹配代码结构。
- 该文件是众多语言特定规则文件之一；每种语言（TypeScript、Python、Go 等）都有其对应的 YAML 文件。
- 没有此类规则文件，ArchSpine 无法解析相应语言，因此无法提供该语言代码库的洞察。

## 本文档锚定的决策与工作流程
- **模式设计**：规则精确规定了如何提取导入、类/接口声明和方法调用。对这些模式的任何更改都会影响整个依赖图和公共 API 索引。
- **扫描器工作流程**：ArchSpine 扫描器将这些模式作为 Java 的唯一定义语法；YAML 结构决定了捕获哪些节点以及如何标记它们（例如 `kind: Class`）。
- **聚合器工作流程**：聚合器依赖提取的使用关系（调用和实例化）来计算跨文件引用，因此该文件是依赖解析的核心契约。
