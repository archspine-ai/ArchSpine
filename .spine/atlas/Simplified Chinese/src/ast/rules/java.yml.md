# ArchSpine 模式定义：结构性代码分析语法

## 目的
本文档规定 ArchSpine 镜像系统用于识别结构性代码元素的模式语法。它定义了三大类别（导入、导出、使用）的模式，构成该工具反射引擎的配置语法。这些模式使自动化代码分析、架构验证和镜像构建成为可能，而无需依赖运行时执行。

## 目标读者
本文档面向集成或扩展 ArchSpine 的开发者与架构师。读者应熟悉系统的镜像与反射概念。模式匹配引擎的实现细节、运行时语义以及超出 Java 类模式的语言特定语法不在本文范围内。

## 文档存在的理由
ArchSpine 的镜像系统需要一种声明式方式来识别结构性代码模式。这些定义为以下工作流程提供支持：
- 配置反射引擎以扫描源代码。
- 一致地识别导入、类/接口声明以及方法调用或实例化。
- 可扩展性：可以按照既定语法添加新模式。

## 模式类别

### 导入 (Imports)
- **标识符 (id)**：`import`
- **模式 (pattern)**：`import $SOURCE;`  
  匹配导入语句。

### 导出 (Exports)
用于类和接口声明的模式，可选可见性修饰符。
- **标识符**：`class`，模式：`class $NAME $$$ { $$$BODY }` (默认可见性)
- **标识符**：`public_class`，模式：`public class $NAME $$$ { $$$BODY }`
- **标识符**：`interface`，模式：`interface $NAME $$$ { $$$BODY }`
- **标识符**：`public_interface`，模式：`public interface $NAME $$$ { $$$BODY }`

### 使用 (Usages)
用于方法调用和对象实例化的模式。
- **标识符**：`call`，模式：`$OBJ.$NAME($$$ARGS)` — 方法调用。
- **标识符**：`new`，模式：`new $NAME($$$ARGS)` — 构造函数实例化。

## 关键要点
- 语法涵盖三个领域：导入、导出（类/接口）和使用（调用/实例化）。
- 导出模式区分公有和默认可见性，并区分类与接口种类。
- 每个模式定义包含唯一的 `id` 和包含占位符（`$SOURCE`、`$NAME`、`$$$BODY`、`$$$ARGS` 等）的 `pattern` 字符串。
- 这些模式是 ArchSpine 静态代码分析与架构强制的基础。
