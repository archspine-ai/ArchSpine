# ArchSpine 代码模式配置：文档概要

## 目的
本文档定义了 ArchSpine 静态分析引擎用于识别 JavaScript/TypeScript 源代码中导入（import）、导出（export）和使用（usage）的模式匹配规则。通过精确描述每种模式的语法，它为自动提取符号及其依赖关系提供了基础——支撑 ArchSpine 系统的代码镜像和依赖追踪功能。

## 读者对象
- **ArchSpine 静态分析引擎的开发者与维护者**，需要理解、修改或扩展针对不同代码结构的识别规则。
- **贡献者**，例如希望集成新的模式类型（如额外的导入或导出形式）或调整现有模式以适应语言特性或项目惯例的变化。

## 关键要点
1. **导入模式**涵盖三种标准形式：具名导入（`import { ... } from ...`）、默认导入（`import X from ...`）和命名空间导入（`import * as X from ...`）。
2. **导出模式**覆盖多种结构：函数、类（包括默认导出）、接口、类型、变量（`const`、`let`、`var`）、默认导出和列表导出（`export { ... }`）。每个导出均附带符号种类（如 `Function`、`Class`、`Variable`），便于后续分析中统一处理。
3. **使用模式**捕获三种基本操作：函数调用、构造函数调用（`new X(...)`）和属性访问（`obj.prop`）。
4. 每个模式都被分配了唯一标识符，并在适用情况下分配了种类标签。这些标识符作为整个分析流程中的标准引用，确保下游工具（如依赖图、镜像生成）对符号的处理保持一致。

## 工作流锚点
本配置文件是静态分析阶段的唯一事实来源。任何对识别语法的修改（例如支持重导出或动态导入）都须首先在此处体现。这些定义直接馈入扫描代码库的模式匹配引擎，因此需要修改识别逻辑的工程师应参照本文档，以确保新规则与现有约定对齐。
