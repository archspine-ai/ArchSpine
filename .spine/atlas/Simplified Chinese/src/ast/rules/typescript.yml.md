# ArchSpine TypeScript AST 提取规则

## 文档存在的意义
本文档是 ArchSpine 项目中的一份配置文件，用于定义 TypeScript 源代码的抽象语法树（AST）提取规则。它详细规定了如何从 TypeScript 文件中识别和提取三类代码结构：
- **导入声明**（具名导入、默认导入、命名空间导入）
- **导出声明**（函数、类、接口、类型、变量、默认导出、导出列表）
- **使用引用**（函数调用、构造函数调用、属性访问）

如果没有这些规则，分析引擎将无法确定需要捕获哪些语法形式，从而导致用于自动化文档生成、依赖追踪和治理执行的代码索引不完整或不准确。

## 目标读者
- 负责维护 ArchSpine AST 提取引擎（位于 `src/ast/` 目录下）的**开发者**。
- 需要理解 TypeScript 元数据提取流程的 **AI 代理**。
- 希望扩展或修改 TypeScript 提取能力的**贡献者**。

## 锚定的工作流程与决策
- **提取准确性**：任何需要改变导入、导出或使用捕获方式的操作都需在此文件中反映。
- **语法同步**：本文件必须与 ArchSpine 所使用的 tree‑sitter 解析器支持的 TypeScript 语法保持同步。
- **语言隔离**：其他编程语言在同一目录下有各自的规则文件；对 TypeScript 规则的修改不会影响其他语言。

## 关键要点
- 模式使用 tree‑sitter 查询语法编写。
- 每个模式都有一个标识符，并可选地指定一个分类（例如 `Function`、`Class`、`Variable`）以便于上层分析。
- 导入模式涵盖了三种标准形式：具名导入、默认导入和命名空间导入。
- 导出模式处理多种 TypeScript 结构，包括默认导出和导出列表。
- 使用模式捕获常见的引用形式：函数调用（`call`）、对象实例化（`new`）和属性访问（`property`）。
- 规则是语言特定的，仅适用于 TypeScript。
