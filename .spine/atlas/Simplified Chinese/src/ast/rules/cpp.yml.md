# ArchSpine C/C++ AST提取规则：文档概要

## 目的
本文档定义了 ArchSpine 系统中用于解析和索引 C/C++ 源代码符号的 AST 提取规则。它规定了系统如何从 C/C++ 源文件中识别导入项、导出符号和使用模式，从而为代码分析和镜像构建提供知识图谱支持。

## 适用读者
适用于需要了解如何分析 C/C++ 代码以实现导入检测、符号导出和使用跟踪的开发者和 AI 代理。这些规则帮助读者理解 ArchSpine 知识图谱中符号关系的提取逻辑，无论是实现提取流程还是使用提取后的数据均适用。

## 文档定义的内容
本文档明确了三个核心提取工作流：

- **导入检测**：如何将 `#include` 指令识别为源导入（模式：`#include $SOURCE`）。
- **符号导出**：如何将函数、类、结构体、命名空间和模板类提取为导出符号。每条导出规则指定了 AST 节点类型、捕获符号名称及可选签名的模式，以及分配的符号类别（如 Function、Class、Type）。
- **用法检测**：如何识别三种使用模式：直接函数调用（`$NAME($$$ARGS)`）、作用域访问（`$OBJ::$NAME`）和方法调用（`$OBJ.$NAME($$$ARGS)`）。

## 锚定的决策
本文档固定了以下内容：
- 哪些 C/C++ 构造被视为导入、导出或使用。
- 每种符号类型使用的精确 AST 模式。
- 从 AST 节点类型到语义符号类别（Function、Class、Type）的映射。
- 提取范围：明确**不覆盖** lambda 表达式、`constexpr`、高级模板元编程、宏、除 include 以外的预处理指令、属性注解等。此外也不强制执行任何架构规则、命名约定或项目特定策略。

## 关键要点
- 导入项通过 `#include` 指令提取。
- 导出项包括函数、类、结构体、命名空间和模板类，每个都包含名称及（如适用）签名。
- 使用模式涵盖函数调用、作用域访问（::）和方法调用（.）。
- 规则仅限定于基础 AST 模式，有意排除了更高级的 C++ 特性。
