# ArchSpine 符号提取规则规范

## 目的
本文档定义了驱动 ArchSpine 镜像系统的核心提取规则。它详细说明了系统如何导入、导出以及识别 C++ 源代码中的符号使用。通过采用声明式模式语言，镜像系统无需执行或分析运行时行为，即可生成精准的 API 文档和代码映射。

## 读者对象
本规范面向**从事 ArchSpine 镜像系统开发与集成的工程师**。读者需要：
- 配置从 C++ 代码库中提取哪些符号。
- 定义如何匹配 `#include` 指令以完成导入。
- 设定函数、类、结构体、命名空间及模板类的导出模式。
- 建立识别函数调用、作用域访问和方法调用的使用模式。

## 本文档锚定的决策与工作流
- **导入逻辑**：`include` 模式 (`#include $SOURCE`) 确定了头文件包含的唯一入口。后续所有提取操作都依赖于正确匹配这些指令。
- **导出定义**：五个独立的导出规则类别（函数、类、结构体、命名空间、模板类）决定了哪些 C++ 结构会成为镜像中的符号实体。每条规则都使用基于 tree–sitter 的规则模式，并通过占位符（`$NAME`、`$ARGS`）提取名称和签名。
- **使用检测**：三种使用模式（调用、作用域访问、方法调用）使镜像能够记录符号在整个代码库中的使用情况，这对于依赖分析和交叉引用生成至关重要。
- **范围与边界**：本规范明确排除了运行时语义、构建系统配置以及非 C++ 语言。任何依赖执行行为或构建依赖的工作流，请参考其他文档。

## 关键要点
- 规则覆盖函数、类、结构体、命名空间和模板类。
- 使用模式涵盖函数调用、作用域成员访问（`::`）和方法调用（`.`）。
- 导入模式匹配 `#include` 指令以引入头文件。
- 所有模式遵循一致的 tree–sitter 语法，确保提取结果具有确定性。
