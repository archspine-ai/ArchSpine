# ArchSpine Rust 源代码 AST 模式规则文档概要

## 文档目的

本文档规定了 ArchSpine 扫描流水线在分析 Rust 源代码时，用于提取结构化元数据的 AST 模式规则。它定义了扫描引擎需要识别的导入、导出和使用模式，是连接底层语法树与上层架构分析之间的桥梁。

## 目标读者

本概要面向维护或扩展 ArchSpine 扫描器 Rust 分析功能的开发者，以及参与该工作流的 AI 代理。您需要了解 Rust 语法和 tree-sitter 查询语言，以便理解并应用这些规则。

## 核心模式定义

文档明确了三类主要模式，每个模式对应一个 tree-sitter 查询规则，用于从代码中提取结构信息。

### 导入模式

- **规则代号：** `use`
- **匹配形式：** `use $SOURCE;` —— 识别所有 `use` 语句，将 `$SOURCE` 作为依赖来源。

### 导出模式

导出模式定义了五种 Rust 语言结构的识别方式：

| 规则代号 | 匹配目标 | 匹配形式（简化） | 类型标签 |
|---------|---------|------------------|----------|
| `function` | 独立函数 | `fn $NAME($ARGS) { ... }`（排除在 `impl` 块内部的函数） | `Function` |
| `struct` | 结构体 | `struct $NAME { ... }` | `Class` |
| `trait` | 特征 | `trait $NAME { ... }` | `Interface` |
| `impl` | 实现块 | `impl $NAME { ... }`（支持泛型类型） | `Type` |
| `variable` | 变量声明 | `let $NAME = $VAL;` | `Variable` |

### 使用模式

使用模式描述了对已定义元素的引用方式：

- **调用模式（`call`）：** `$NAME($$$ARGS)` —— 匹配函数调用表达式。
- **路径模式（`path`）：** `$OBJ::$NAME` —— 匹配通过命名空间或路径访问的标识符（如 `std::vec`）。

所有模式均采用 tree-sitter 查询语法，并辅以规则条件（如 `inside`、`not`、`has`、`any` 等）来精确限定匹配范围。

## 支撑的工作流与决策

- **扫描决策：** 当扫描器遍历 Rust 文件时，AST 提取器根据这些规则判断节点是否属于导入、导出或使用，从而决定是否将其加入依赖图。
- **依赖分析：** 导出的 `Type` 和 `Function` 类节点用于构建模块接口；`use` 导入和路径引用则形成调用依赖边。
- **架构验证：** 生成的元数据最终用于架构规则检查（如层间依赖是否合规），本规则集定义了度量基础。
- **性能优化：** 规则中的 `stopBy: end` 等选项控制了查询的深度，避免过度递归，平衡分析速度与精度。

通过以上规则，ArchSpine 能够自动化识别 Rust 代码中的模块结构和依赖关系，为后续的架构分析提供可靠的数据基础。
