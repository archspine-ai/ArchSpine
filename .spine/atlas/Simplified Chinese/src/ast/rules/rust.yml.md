# ArchSpine 镜像系统：模式配置概览

## 目的
本文档定义了 ArchSpine 用于从 Rust 源代码中识别并镜像代码元素的抽取模式。这些模式指定了如何通过 tree-sitter 查询来定位导入语句、导出项（函数、结构体、特征、实现块、变量）以及用法（函数调用和成员路径）。这些模式是 ArchSpine 构建基于抽象语法树（AST）的代码镜像的核心。

## 目标读者
本文档面向构建或扩展 ArchSpine 镜像系统的开发者，特别是负责以下工作的人员：
- 实现或修改代码分析逻辑。
- 生成或验证基于 AST 的镜像。
- 集成新的 Rust 语言特性或自定义模式匹配。

同时，它也是架构师审查系统结构化抽取规则的参考资料。

## 关键要点
- **基于 tree-sitter 的模式匹配**：所有模式均以 tree-sitter 查询规则表达，精确且可组合。例如，导出模式使用嵌套的 tree-sitter 条件来匹配不在 impl 内部的函数项。
- **导出覆盖范围**：系统覆盖函数、结构体、特征、实现和变量定义。每个导出项被赋予一个语义类别（如 `Function`、`Class`、`Interface`、`Type`、`Variable`），便于下游工具解读镜像节点。
- **用法捕获**：两种用法模式——`call` 和 `path`——捕获关系：函数调用 `$NAME($$$ARGS)` 和成员访问 `$OBJ::$NAME`。这些为依赖分析和调用图分析提供基础。
- **范围界定**：本文档明确排除了运行时行为、用户界面设计以及 Rust 以外的语言语义，只关注静态代码结构。

## 本文档锚定的决策与工作流
- **模式治理**：所有关于代码元素抽取规则的变更必须在此文档中反映；它是抽取语法的唯一权威来源。
- **镜像生成工作流**：构建流水线读取这些模式以生成代码库的镜像表示。模式与实际代码结构不匹配将导致镜像失效。
- **交叉引用**：依赖于镜像数据的工具（如依赖关系图、文档生成器）依赖于这些模式的一致性。
