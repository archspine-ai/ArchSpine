# ArchSpine Rust 模式定义

## 目的
本文档定义了 ArchSpine 用于内省 Rust 源代码的模式匹配规则。它规定了如何识别和提取代码中的导入、导出（函数、结构体、特征、实现块、变量）以及使用（调用、路径）信息。

## 上下文与受众
本文档面向需要理解或修改 ArchSpine 如何分析 Rust 代码的开发者及 AI 代理。它是代码反射引擎的核心配置，定义了 ArchSpine 在 Rust 项目中能够“看到”的语法结构。

## 主要职责
- 指定源文件包含的导入模式
- 定义 Rust 构造（函数、结构体、特征、实现块、变量）的导出模式
- 定义函数调用和路径引用的使用模式
- 作为代码分析与反射的配置模式

## 不涉及范围
- 实际的代码执行或编译逻辑
- 语言无关或多语言的模式定义
- 用户界面或文档生成

## 关键要点
- ArchSpine 使用类似 tree-sitter 的模式规则来识别代码结构。
- 导出项按 Rust 构造类型（函数、类、接口、类型、变量）进行分类。
- 使用项追踪导出项如何被引用（调用和路径）。
- 本文档是一个配置文件，而非叙述性指南。
