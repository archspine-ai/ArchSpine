# ArchSpine 配置模式 – 核心验证契约

本模块定义了 ArchSpine 配置模式的基础验证契约和常量集合。它是允许配置值和验证结果结构的唯一权威来源。

## 角色

ArchSpine 配置模式的核心验证契约与常量定义。

## 主要职责

- 定义 `SpineConfigValidationResult` 接口，为所有验证操作提供结构化输出格式。
- 声明关键配置字段（包括 LLM 模式、提示层级、验证策略等）的允许值常量集合。

## 重要不变性

- 提供**集中化**的允许值集合，所有验证逻辑必须引用此集合。
- 导出**类型化结果结构**（`SpineConfigValidationResult`），确保整个验证管道的一致性。

## 范围外（负面范围）

- **不实现**验证逻辑——该职责由独立模块（如 `config-validation.ts`）处理。
- **不处理** CLI 命令入口或用户交互。
- **不读取或解析**配置文件。

## 最重要的导出行为

- **`SpineConfigValidationResult`** – 所有验证函数必须返回的主要公共接口。它标准化了验证结果在系统中的传递方式。

## 架构意图

本模块为整个 ArchSpine 管道的配置验证建立了共享词汇和契约。通过集中化允许值和结果结构，确保所有验证组件保持一致性和互操作性。

## 近期变更意图

最近的提交表明正在收紧模式处理并增加预览能力，暗示验证契约正在持续优化。
