# ArchSpine 架构约束规范

## 目的
本文档为 ArchSpine 仓库建立并强制执行分层架构约束。它明确定义了 CLI 入口、服务编排、核心流水线契约、任务阶段、引擎和基础设施外观之间的边界。目标是维护一个模块化、可测试且可复用的代码库，其中每一层都有明确的职责和依赖关系。

## 上下文与受众
本文档面向 ArchSpine 项目的所有贡献者，特别是从事 CLI 命令、服务编排、流水线逻辑或基础设施模块开发的开发者。它作为理解架构模型并确保新代码遵循关注点分离原则的参考。相关规则通过自动化 lint 或审查流程强制执行。

## 关键要点
- CLI 模块必须保持为薄入口，不得吸收流水线或持久化逻辑。
- 服务层负责编排核心、任务、引擎和基础设施；视图相关的服务应位于 `src/services/view/` 下。
- 核心模块定义流水线契约，不得依赖 CLI 入口代码。
- 任务模块实现阶段本地工作，不得接管 CLI 命令解析或不相关的服务编排。
- 引擎提供可复用的分析和转换逻辑，必须与表示层和编排层解耦。
- 基础设施模块暴露稳定的外观接口；调用者应优先使用公共外观，而非深入私有实现路径。
