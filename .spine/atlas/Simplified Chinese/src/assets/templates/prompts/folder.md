`prompts/`目录是 ArchSpine 系统的提示工程核心。它提供了用于生成结构化架构图与语义摘要的提示模板模式、渲染工具以及示例数据，使得 AI 代理能够以统一的 JSON 格式输出结果。该目录整合了三个主要实现领域：提示构建模块、模式定义和少样本示例。

显著子组件分组如下：

- **`arch-diagram.ts`** – 用于从项目与文件夹摘要生成 JSON 架构图规格的 LLM 提示模板。它嵌入了 `archDiagramSchema` 以强制输出结构（节点、边、摘要卡片），并提供了关于允许节点类型与约束的明确指导。
- **`blocks.ts`** – 一个纯渲染函数库，用于构造提示构建中的结构化文本块。涵盖身份、指令、上下文、环境、规则违反、Git 意图、JSON 模式与源内容块，每个块都具备条件性渲染逻辑。
- **`schemas.ts`** – 所有 ArchSpine 语义摘要单元的模式定义：源文件、文档、配置文件、文件夹及项目级摘要。这些模式提供了 JSON 模板，确保 AI 代理输出的一致性。
- **`examples.ts`** – 包含语义角色描述的优秀/不良示例的静态内容，作为提示中的少样本参考，提升输出质量。
- **`index.ts`** – 公共 API 门面，将所有模式、渲染函数和示例数据聚合并重新导出，为下游模块提供单一导入点。

最重要的实现领域是 `arch-diagram.ts` 中的提示模板（驱动架构图生成）以及 `schemas.ts` 中的模式定义（保证所有摘要类型遵循可预测的结构）。`blocks.ts` 中的渲染工具是跨多个 ArchSpine 功能构建提示的可复用基础。
