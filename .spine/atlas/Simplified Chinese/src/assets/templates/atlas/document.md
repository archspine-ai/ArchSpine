# ArchSpine 文档摘要

## ArchSpine 为何存在

ArchSpine 是一个文档镜像系统，它在代码旁创建一个 `.spine` 目录，形成一份与代码库保持同步的架构文档“地图”。该系统旨在解决架构文档随代码演变而逐渐脱离实际这一长期问题。通过镜像文件树并将注解直接存储在源代码旁，ArchSpine 使架构知识保持鲜活、准确，并可供开发者和 AI 代理访问。

## 谁应该阅读本文档

本文档面向以下受众：

- **开发者**和**团队**，他们希望在无需重复劳动的前提下保持架构文档的准确性，从而了解镜像系统的工作原理、覆盖范围以及如何融入日常工作流程。
- **技术文档撰写人员**，负责维护反映架构真实状态的文档交付物。
- **AI 代理**，需要理解并与 ArchSpine 文档层进行交互，例如用于检测漂移、注解代码或生成报告的代理。

读者应熟悉软件架构概念，并能适应使用镜像文档结构的项目环境。

## 核心概念与设计决策

本文档锚定了以下关键决策和工作流程：

- **镜像**：`.spine` 目录镜像项目的文件树，将架构注解直接存储在对应的源代码文件旁。这确保了文档始终紧邻其所描述的代码。
- **漂移检测**：系统自动检测代码与文档之间的不一致之处，当架构偏离实际情况时向团队发出警报。
- **多语言地图**：该“地图”支持多种语言（如英语、简体中文）的文档交付，让全球团队能够使用各自偏好的语言贡献和查阅架构文档。

## 维护边界

为了保持系统的专注性和可管理性，本文档明确界定了范围：

- **范围内**：文档镜像、架构规则与注解、视图、漂移检测以及多语言地图。
- **范围外**：源代码修改、CI/CD 配置、ArchSpine 内部机制的详细实现或 API 参考、安装或使用说明（这些内容涵盖在独立的操作指南文档中）。
