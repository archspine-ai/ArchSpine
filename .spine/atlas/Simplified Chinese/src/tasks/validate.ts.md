# ArchSpine 验证任务编排器

## 角色
该模块是 ArchSpine 镜像系统中基于大语言模型（LLM）的架构规则合规性检查的核心编排器。它协调源文件与已配置架构规则之间的验证过程，利用 LLM 生成的提示来完成检查。

## 主要职责
- **验证编排**：管理源文件与架构规则之间的端到端验证流程，使用 LLM 生成的提示进行合规性检查。
- **提示构建**：从源文件和规则上下文中构建提示工件，为 LLM 验证过程提供输入。
- **并发执行**：通过并发控制执行 LLM 验证提示，并处理 LLM 响应的解析。
- **数据规范化**：规范化并过滤 LLM 生成的规则违规条目，确保结构化的合规数据。
- **诊断与记录**：记录验证诊断信息，并将违规情况记录到清单中，以便追踪。

## 关键不变性
- 严格作为管道阶段运行，基于核心任务合约和引擎，而非独立服务。
- 将提示生成和 LLM 执行委托给专用基础设施模块，而非直接处理。
- 输出与 `RuleViolation` 协议完全兼容的结构化验证结果。

## 排除范围（不负责）
- **不**处理 CLI 命令解析或无关的服务编排。
- **不**管理直接的 LLM API 通信或令牌管理。
- **不**定义或存储架构规则，仅消费它们。

## 公开接口
- **`ValidationSummary` 接口**：提供验证结果的摘要。
- **`ValidationTaskResult` 接口**：定义单个验证任务结果的结构。

## 架构意图
实现一个验证阶段，使用 LLM 提示检查源代码是否符合架构规则，并与任务执行图无缝集成。最近的更改侧重于强化扫描恢复验证和修复回退路径，以提高可靠性。
