# ArchSpine 流水线任务模块

本目录包含驱动 ArchSpine 执行图的具体任务实现。每个文件代表一个流水线阶段，负责从文件系统扫描到状态持久化的特定处理步骤。这些任务按既定顺序执行，通过检查点机制支持断点续传，并可适当并发运行。

## 角色与职责

这些任务共同实现核心流水线阶段：文件扫描、清单同步、AST 提取、语义摘要生成（包括完整版与轻量版）、架构规则验证、自动修复、内容聚合、反向索引构建、视图推导以及最终的状态提交。它们确保流水线按序执行，并在发生中断时能够从任意检查点恢复。

## 主要子模块分组

任务可按流水线阶段分组如下：

### 扫描与同步
- **`scan-cleanup.ts`** – 扫描文件系统，检测已变更或已删除的文件，并更新清单。
- **`reconcile.ts`** – 将清单与仓库实际状态同步，验证索引文档兼容性，并更新缓存元数据。

### 抽象语法树提取
- **`ast-extra.ts`** – 从源文件中提取 AST 骨架，注册到运行时缓存，并处理不支持的源文件。

### 语义摘要生成
- **`summarize.ts`** – 基于大语言模型，从提取阶段输出生成完整的语义摘要。
- **`lite-summarize.ts`** – 轻量级摘要生成，处理筛选后的文件并委托给专用方法。

### 验证与修复
- **`validate.ts`** – 基于大语言模型执行架构规则合规性检查，并对违规结果进行规范化处理。
- **`fix.ts`** – 利用大语言模型和 AST 分析生成并应用自动化补丁，修复架构违规。

### 聚合与索引
- **`aggregate.ts`** – 对目录和项目进行分层内容聚合。
- **`reverse-index.ts`** – 从前向依赖索引文件构建反向依赖边。

### 视图推导与状态提交
- **`view-derivation.ts`** – 根据配置推导架构视图（公共接口、风险热点、架构图）。
- **`post-commit-derivation.ts`** – 在主要提交阶段之后编排视图推导、聚合和反向索引。
- **`state-commit.ts`** – 在提取和摘要完成后，将文件状态和元数据提交到 SQLite 数据库。

### 辅助模块
- **`document-backfill.ts`** – 通过大语言模型，从 JSON 数据生成 markdown 文档，支持去重（基于内容哈希）。

## 关键实现领域

最重要的实现关注点包括：
- **基于检查点的可恢复性** – 每个任务都与执行检查点系统集成，允许流水线从已保存状态恢复。
- **并发控制** – `summarize.ts`、`fix.ts`、`document-backfill.ts` 等任务使用 `p-limit` 调控并行执行，避免超出 API 速率限制。
- **幂等性与状态追踪** – 任务精细管理文件状态（开始、跳过、完成、失败），通过哈希比较避免重复工作。
- **与外部大语言模型服务的集成** – 多个任务依赖 AI 模型进行摘要、验证、修复和文档生成；包括提示词构建、结构化响应解析和令牌用量跟踪。

具体子模块如 `aggregate.ts`、`ast-extra.ts`、`fix.ts` 和 `state-commit.ts` 被视为流水线的核心构建块，在讨论流水线架构时应直接引用。
