# LiteSummarizationTask

**角色：** 轻量级源代码摘要的流水线任务，处理来自提取阶段的过滤文件，并将摘要工作委托给专门的摘要方法。

## 主要职责

- 处理提取阶段输出的已过滤源文件。
- 识别不支持的文件和骨架文件，跳过处理。
- 使用 `LangRegistry` 判断文件是否为可进行摘要的源文件。
- 读取文件内容并将摘要工作委托给专门的 `summarizeFile` 方法。
- 通过任务状态函数跟踪任务使用情况、跳过的文件、失败的文件和已处理的文件。
- 使用 `pLimit` 管理并发，实现并行文件处理。

## 不涉及范围

- CLI 命令解析或参数处理。
- 与摘要流水线阶段无关的服务编排。
- 超出文件系统读取范围的直接外部 API 或数据库交互。

## 不变约束

- 任务模块应在核心契约和引擎之上实现阶段本地工作。
- 任务模块不应接管 CLI 命令解析或无关的服务编排。

## 公开接口

- `LiteSummarizationTask` 类（已导出）

## 变更意图

**架构意图：** 提供一个轻量级、受并发控制的摘要任务，严格在流水线阶段边界内运行，仅处理来自提取阶段的文件，并将摘要工作委托给专用方法。

**近期变更意图：** 修复 lint 警告和类型错误，并添加 CI 工作流支持。
