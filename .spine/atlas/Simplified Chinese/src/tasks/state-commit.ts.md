# StateCommitTask — 流水线阶段摘要

**角色：**  
一个专用的流水线阶段任务，负责在 AST 提取和 LLM 摘要生成成功完成后，将同步的文件状态（哈希值和元数据）提交到 SQLite 数据库。

**主要职责：**
- 通过清单系统将批量的文件状态（哈希值和元数据）提交到 SQLite 数据库。
- 确保仅在 AST 提取和 LLM 摘要生成阶段成功完成后才执行数据库写入。
- 通过与执行检查点协调，防止写入竞争和部分失败污染。
- 记录提交进度并更新检查点完成状态。

**不包含的范围：**
- CLI 命令解析
- 超出自身阶段的服务编排
- AST 提取或 LLM 摘要生成逻辑
- 直接文件 I/O 或哈希计算

**不变约束：**
- 必须在核心契约和引擎（`SpineTask`、`TaskContext`）之上实现阶段本地工作。
- 不得接管 CLI 命令解析或无关的服务编排。

**公开接口：**
- `StateCommitTask` 类

**架构意图：**  
提供一个专用的流水线阶段，用于原子化数据库提交，确保上游处理后的数据完整性。核心任务状态契约的近期重构不会改变此阶段的本地焦点。
