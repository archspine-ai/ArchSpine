# ArchSpine 配置子系统入口文件 (`index.js`)

## 角色
该文件是配置子系统的**公开 API 聚合文件**。它汇总并重新导出配置外观（facade）及所有类型定义，为使用者提供单一、稳定的入口点。

## 主要职责
- 从 `./config/facade.js` 重新导出 `Config` 外观。
- 从 `./config/types.js` 重新导出所有类型定义，提供统一的导入路径。
- 作为任何需要配置基础设施的模块的主要接口。

## 重要不变性与排除范围
- **仅重新导出**配置子系统的公开符号，不暴露任何私有实现细节。
- **不包含任何可执行逻辑**或状态行为——纯粹是重新导出的枢纽。
- **保持稳定的导出表面**，以保护使用者免受内部重构的影响。
- **排除范围：** 不实现配置逻辑、验证、运行时编排或服务层关注点。不直接导入深层私有模块。

## 导出的外部可见行为
- `Config` — 主要配置外观（来自 `./config/facade.js`）。
- 所有来自 `./config/types.js` 的类型定义均可通过此入口文件获取。
