# ArchSpine – `generateSourcePrompt` 外观模块

## 角色

该模块是一个**基础设施外观**，负责组装用于源代码分析的结构化 LLM 提示。它不执行推理、不解析 AST、也不强制执行规则——它只负责构建提示。

## 主要职责

- 构建 `PromptBuilder`，包含身份、指令、示例以及环境上下文（分支、状态）。
- 集成本地化语言指令和输出契约渲染，以支持多语言。
- 将架构规则、Git 意图、依赖上下文和先前的语义数据注入提示。
- 应用 JSON 模式（`sourceFileSchema`）以强制输出结构。
- 支持可配置的任务模式（例如 `'summarize'`、`'audit'`）和提示策略层级（例如 `'balanced'`）。

## 重要不变性

- **纯函数**：必须无副作用——仅进行提示组装。
- **委托**：所有提示渲染逻辑委托给 `PromptBuilder` 和共享工具。
- **不涉及编排**：不得吸收服务、任务或引擎编排的关注点（遵循基础设施外观导入规则）。

## 不涉及范围

- 编排 LLM 推理或 API 调用。
- 解析或分析超出提供内容的源代码 AST。
- 强制执行超出注入提示范围的架构规则。
- 管理提示模板或模式定义（委托给资产）。

## 公开接口

- `generateSourcePrompt` —— 唯一导出的函数。

## 变更意图

架构意图是提供一个稳定、可配置的外观，用于生成遵循架构策略和输出契约的提示，从而实现系统范围内一致的源代码分析。最近的变更重构了该模块，以建立子系统外观并解决层反转问题，将提示生成逻辑整合到专用的基础设施外观中。
