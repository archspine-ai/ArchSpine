# `src/infra` — 基础设施层

`src/infra` 目录是 ArchSpine 镜像系统的核心基础设施层，提供系统其余部分所依赖的所有核心服务、工具和持久化机制。该层负责配置管理、凭据存储、数据库持久化、LLM 客户端抽象、清单追踪、MCP 服务器集成、提示词构建、文件 I/O 和运行时工具。

## 主要子模块

- **`config/`** — 配置管理子系统。包含用于解析和验证的门面模块（`config-validation.ts`）以及重新导出类型和门面的桶文件（`config.ts`）。
- **`credentials/`** — 平台特定的安全凭据存储和密钥管理。
- **`db/`** — SQLite 持久化层，包含生命周期管理、模式演进和索引系统的数据访问。`db.ts` 门面聚合了所有运行时数据库操作。
- **`llm/`** — 核心 LLM 基础设施：客户端抽象、配置和语义生成的弹性执行。`llm.ts` 门面提供稳定的公共接口。
- **`manifest/`** — 清单状态持久化、文件完整性验证和状态追踪。`manifest.ts` 门面聚合了核心类型和查询。
- **`mcp/`** — 通过模型上下文协议桥接 ArchSpine 内部能力与外部 AI 代理。
- **`prompt/`** — 提示词构建和组装基础设施。
- **`prompt-context/`** — 提示词策略系统的编排和配置。`prompt-context.ts` 门面导出策略常量、预算配置和工件构建函数。
- **`__mocks__/`** — LLM 提供者模拟基础设施的公共 API 表面。

## 关键实现领域

- **配置与验证** — `config-validation.ts` 和 `config.ts` 提供用于解析和验证系统配置的稳定 API。
- **数据库持久化** — `db.ts` 和 `db/` 文件夹管理 SQLite 生命周期、模式演进和数据访问。
- **LLM 集成** — `llm.ts` 和 `llm/` 文件夹处理客户端创建、配置和弹性执行。
- **清单追踪** — `manifest.ts` 和 `manifest/` 文件夹管理状态持久化和文件完整性。
- **MCP 服务器** — `mcp/` 文件夹实现模型上下文协议桥接。
- **提示词系统** — `prompt/`、`prompt-context/` 及相关文件处理提示词构建、策略和渲染。
- **安全与密钥** — `credentials/`、`secrets.ts` 和 `auth.ts` 管理安全凭据存储和身份验证。
- **文件 I/O 与工具** — `output.ts`、`repository-artifacts.ts`、`rules-loader.ts`、`runtime-io.ts`、`spine-gate.ts`、`writer-boundary.ts` 和 `ui.ts` 提供底层文件系统操作、运行时 I/O 和工具函数。
- **修复与验证** — `repair-policy.ts` 定义违规处理逻辑，`index-reader.ts` 提供索引文档读取和验证。
- **精简模式** — `lite-prompt.ts` 为令牌受限的 API 构建轻量级提示词。
- **渲染** — `renderer.ts` 从布局类型生成结构化 Markdown 文档。
