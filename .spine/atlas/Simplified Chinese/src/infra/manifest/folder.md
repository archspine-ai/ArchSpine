`infrastructure` 目录为 ArchSpine 镜像系统提供了文件系统交互、完整性验证和清单状态持久化的基础设施层。其核心职责是封装底层操作，包括计算 SHA-256 文件哈希、解析指向清单和语言快照文件的绝对路径、读写基于 JSON 的状态文件，以及管理文件元数据快照。

该目录根据职责对组件进行分组：

- **完整性验证** – `integrity.ts` 负责计算文件 SHA-256 哈希，验证文件是否存在，并与 SpineDB 状态追踪集成。
- **文件 I/O 与路径解析** – `io.ts` 解析到脊柱清单（`.spine/manifest.json`）和语言快照（`.spine/languages.json`）的绝对路径，以 null 安全的方式读取 JSON 文件，并使用 `fs.statSync` 捕获文件的修改时间和大小。
- **状态持久化管理** – `state.ts` 定义了运行时状态接口，加载/保存清单和语言快照 JSON 文件，追踪反向索引完成状态和基线检测。`facade.ts` 将这些操作封装为更高级的接口，支持批量提交、文件状态追踪、变更历史管理以及清单基线处理。
- **类型定义** – `types.ts` 提供了文件元数据快照（包含修改时间和文件大小）的 TypeScript 接口，确保整个代码库中的数据结构一致。

这些模块共同构成了可靠的文件状态追踪、清单基线检测和持久化镜像状态的基础，使 ArchSpine 的其他部分能够有效地评估文件完整性并追踪变更历史。
