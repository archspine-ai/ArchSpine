# SpineDB — 数据库门面 / 仓库聚合器

**角色：**  
SpineDB 为 ArchSpine 镜像系统中的所有 SQLite 运行时数据库操作提供统一接口。它是任何需要持久化或查询运行时数据的代码的唯一入口。

**主要职责：**
- 管理运行时 SQLite 数据库的连接生命周期和模式初始化。
- 提供文件跟踪操作的统一 API：创建、读取、更新状态、删除和列出。
- 处理符号导出注册、缓存失效以及跨文件符号解析，以支持依赖分析。
- 记录和查询 LLM 使用指标，用于成本监控和分析。
- 管理架构规则违规的记录和查询。
- 提供批量事务提交支持，以实现原子操作。

**重要不变性与负面范围：**
- SpineDB 为所有运行时数据库操作暴露一个稳定、聚合的公共 API。
- 它将特定领域操作（文件、符号、使用情况、违规）委托给内部仓库模块。
- 数据库连接作为私有资源进行管理。
- SpineDB **不**编排高级业务逻辑或引擎工作流。
- 它**不**直接执行 SQL 查询（这委托给仓库模块）。
- 它**不**提供 HTTP 或网络接口。

**最重要的导出行为：**
- `SpineDB` 类是唯一的公共表面。所有运行时数据库交互都通过此类进行。
