# ArchSpine – 基础设施工具模块

## 角色
本模块提供底层文件系统和 Git 操作，用于读取受管理的文件块以及检查仓库状态。它作为稳定的基础设施工具，将这些关注点与高层服务和引擎层隔离。

## 主要职责
- 通过同步文件操作读取由起始/结束标记定义的受管理文件块。
- 通过 `execFileSync` 同步执行 Git 命令以获取仓库状态和文件列表。
- 使用 `path.join` 和 `fs.existsSync` 提供仓库文件的路径解析和存在性检查。

## 不包含范围
- 服务编排或任务执行逻辑。
- 引擎级别的扫描或处理流水线。
- 高层业务逻辑或领域特定操作。

## 不变约束
- 不得导入或依赖服务、任务或引擎模块，以避免吸收编排关注点（依据 `infra-facade-imports` 规则）。
- 必须对外暴露稳定的底层能力和外观接口。

## 公开接口
- `ManagedBlockMarkers`（接口）
- `readManagedFileBlock`（函数，推断）
- Git 命令执行函数（推断）

## 变更意图
- **架构意图：** 提供稳定的底层基础设施工具，用于文件系统和 Git 操作，将这些关注点与高层服务和引擎层隔离。
- **近期变更意图：** 重构 CLI 以提取仓库和 LLM 管理服务，这可能涉及将此类工具函数迁移或整合到专门的基础设施模块中。
