`providers/` 目录包含了 ArchSpine 镜像系统的所有具体 LLM 供应商实现和共享工具模块。其主要职责是通过与外部 AI 模型交互，为不同文件类型（配置文件、文档、源代码、文件夹、项目）生成语义摘要。

值得注意的子文件分为两类：

- **LLM 客户端实现**：`gemini.ts`（Google Gemini API）、`openai.ts`（OpenAI 兼容 API）和 `mock.ts`（用于测试的确定性模拟客户端）。每个文件都实现了 `LLMClient` 接口，并处理供应商特定的请求/响应周期。
- **共享工具**：`utils.ts` 提供响应解析（结构化 JSON 和 Markdown 区块）、通过导入的生成器组装提示词，以及聚合多次 API 调用的令牌使用情况。

关键的实现领域包括：按文件类型生成结构化摘要、使用 ArchSpine 提示词引擎格式化提示词，以及将模型输出稳健解析为本地化内容区块。模拟客户端还支持为测试规则引擎而模拟架构规则违规行为。
