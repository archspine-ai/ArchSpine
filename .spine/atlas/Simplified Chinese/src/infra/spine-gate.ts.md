# ArchSpine 变异检测工具模块

本模块是一个纯基础设施工具，负责检测和报告 ArchSpine 受保护输出目录中的未授权变异。它在系统的最底层运行，将文件系统和哈希操作与上层编排逻辑隔离。

## 角色

用于检测和报告 ArchSpine 受保护输出目录中未授权变异的基础设施工具模块。

## 主要职责

- 定义基线文件位置和受保护目录（`.spine/index`、`.spine/atlas`、`.spine/view`）的常量。
- 导出 `ProtectedOutputMutationReport` 接口，用于结构化变异报告。
- 提供读取基线状态、收集文件哈希以及计算新增、变更和删除路径的函数。
- 生成变异报告，检测生成的受保护输出中的未授权更改。

## 不涉及范围

- 编排服务、任务或引擎工作流。
- 为其他基础设施能力提供公共外观。
- 处理身份验证、授权或用户会话。
- 管理数据库查询或外部 API 调用。

## 不变约束

- 仅依赖 Node.js 内置模块（`crypto`、`fs`、`path`）。
- 不得导入其他项目模块，以避免吸收编排关注点。
- 必须暴露稳定的底层文件哈希比较和变异检测功能。

## 公开接口

- `ProtectedOutputMutationReport` — 用于结构化变异报告的主要接口。
