# ArchSpine – `commitBatch` 模块摘要

**角色**  
基础设施层的批量提交函数，用于将文件元数据原子同步到 SQLite 索引数据库。

**主要职责**  
- 原子地批量插入和更新 `FileCommitRecord` 对象到 SQLite 的 `files` 表。  
- 通过漂移仓库记录文件角色或职责变更时的语义漂移事件。

**不涉及范围**  
- 编排高级扫描或索引工作流。  
- 为其他基础设施服务提供公共 API 外观。  
- 直接处理文件 I/O 或文件系统操作。

**重要不变性**  
- 必须通过 SQLite 事务维护批量提交的原子性。  
- 不得吸收超出数据持久化和漂移记录范围的业务逻辑或编排职责。

**最重要的导出行为**  
- `commitBatch(db: Database.Database, commits: FileCommitRecord[]): void` – 唯一的公共函数，用于原子地持久化一批文件提交记录。

**负面范围 / 规则违规**  
- 导入 `./repositories/drift.js` 以记录语义漂移事件，这可能属于服务/编排关注点而非低级基础设施能力。这可能违反基础设施模块不应吸收服务/任务/引擎编排关注点的规则。

**变更意图**  
- 架构意图：提供稳定、原子的批量提交机制用于文件元数据持久化，将数据库操作与更高级的扫描引擎解耦。  
- 近期变更：模块化 CLI 并解耦核心基础设施服务（与重构以隔离基础设施批量提交逻辑一致）。
