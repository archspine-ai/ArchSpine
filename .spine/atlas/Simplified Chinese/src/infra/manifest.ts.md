# ArchSpine – 清单外观模块 (`index.ts`)

## 角色
该文件是 Spine 清单子系统的**公共 API 入口（外观）**，负责聚合并重新导出核心清单类型和查询，为使用者提供单一、稳定的导入路径。

## 主要职责
- 公开暴露 `Manifest` 类和 `hasManifestBaseline` 查询函数作为公共接口。
- 将使用者与清单实现模块的内部文件结构解耦，允许内部重构而不影响外部代码。

## 重要不变性与负面范围
- **只能重新导出**底层外观实现中的稳定符号。
- **不得包含任何内联逻辑或有状态行为** – 该文件纯粹是一个重新导出的枢纽。
- **不在范围内：** 实现清单逻辑或业务规则；编排服务、任务或引擎；提供对指定外观之外的私有实现模块的直接访问。

## 最重要的导出/外部可见行为
- `Manifest` – 核心清单类。
- `hasManifestBaseline` – 用于检查基线清单的查询函数。

该外观为清单子系统建立了稳定的公共 API 边界，减少了变动影响，并将使用者与内部重构隔离开来。最近的变更（基础设施层重构）可能会拆分清单外观状态，但该文件作为稳定导出点的角色支持了该重构。
