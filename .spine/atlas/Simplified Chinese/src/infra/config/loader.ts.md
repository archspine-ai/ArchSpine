# ArchSpine 配置加载与验证模块

该模块是 ArchSpine 系统的配置加载与验证管道，负责读取、解析、验证配置文件，并将其与系统默认配置合并。

## 角色

ArchSpine 系统的配置加载与验证模块，负责读取、解析、验证配置文件，并将其与默认配置合并。

## 主要职责

- 使用同步文件操作从文件系统读取配置文件。
- 解析 JSON 格式的配置内容，并依据模式验证其结构。
- 通过深度合并逻辑，将用户提供的配置覆盖与系统默认配置合并。
- 生成并记录人类可读的配置解析警告信息。

## 重要不变性

- 配置文件必须是有效的 JSON，并符合 SpineConfig 模式。
- 默认配置始终作为基础，覆盖配置在此基础上合并。
- 解析警告针对结构性问题生成，但不会阻止执行。

## 排除范围（不负责）

- 不处理异步文件操作或流式读取。
- 不管理运行时状态或会话生命周期。
- 不执行网络 I/O 或远程配置获取。
- 不实施访问控制或身份验证。

## 公开接口

- `mergeConfigWithDefaults` — 用于将用户配置与系统默认配置合并的主要导出函数。

## 架构意图

提供一个稳定且经过验证的配置加载管道，将文件 I/O、模式验证和默认合并的关注点分离。最近的变更加强了模式处理并增加了试用预览支持，从而提高了验证的健壮性和预览能力。
