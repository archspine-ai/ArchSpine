# ArchSpine – ConfigSupportedValueAccess 接口

## 角色
核心 TypeScript 接口，定义了在 ArchSpine 系统中访问受支持配置值的契约。

## 主要职责
- 定义类型安全的接口，用于获取所有用户可配置的系统参数。
- 集中配置访问签名，确保配置消费者之间的一致性。
- 通过方法名称（例如 LLM 设置、验证策略、钩子模式）记录可用的配置选项。

## 重要不变性与负面范围
- 必须保持为纯 TypeScript 接口，不包含实现。
- 只能导入类型定义，不能导入运行时模块。
- 必须定义返回配置值或 undefined 的 getter 方法。
- **不**实现配置检索逻辑或存储值。
- **不**编排服务、任务或引擎。
- **不**充当基础设施外观或模块。

## 最重要的导出行为
- 导出 `ConfigSupportedValueAccess` 接口，作为配置访问的稳定、类型安全契约，将消费者与实现细节解耦。
