# ArchSpine 发布说明摘要

ArchSpine 的更新日志旨在为所有公开发布版本提供简洁的记录，详细列出每个版本新增的功能、可靠性改进以及文档更新。

**目标读者：** 本文件面向需要跟踪版本变更并了解新功能的 ArchSpine 用户和开发者。无论您是采用 CLI、集成本地 .spine/ 控制平面，还是评估项目的成熟度，更新日志都能让您清晰了解每个版本带来的内容。

**锚定的决策和工作流程：** 更新日志定义了核心工作流命令：`try`、`init`、`build`、`sync`、`publish`、`check`、`fix` 和 `mcp start`。同时确立了项目承诺的可靠性保障：受保护的输出边界、原子化文件写入以及基于锁的互斥机制。在 1.0.0 版本（首个开源发布）中，这些内容包括仓库工件策略、架构规则模板、模式资产、Atlas 文档生成、实验性视图输出，以及支持全局和项目级 LLM 配置的凭据安全存储后端。

**关键要点：**
- ArchSpine CLI 及本地 .spine/ 控制平面的首个开源版本发布。
- 核心工作流包括 `try`、`init`、`build`、`sync`、`publish`、`check`、`fix` 和 `mcp start`。
- 可靠性特性：受保护的输出边界、原子化文件写入、基于锁的互斥机制。
- 文档提供英文和简体中文版本，涵盖协议、运行手册、MCP、本地 LLM、成本、God Mode 和当前能力等支持文档。

更新日志是了解 ArchSpine 每个版本引入内容及其演变过程的唯一权威来源。
