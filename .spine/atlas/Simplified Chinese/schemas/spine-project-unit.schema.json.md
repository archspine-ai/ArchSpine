# ArchSpine SpineProjectUnit 配置摘要

此配置单元定义了 `SpineProjectUnit`（一种自包含且经过验证的配置单元）在 ArchSpine 镜像系统中的结构模式。每个项目单元必须遵循此结构，以确保数据完整性、可审计性，并安全地融入更广泛的镜像基础设施。

## 配置的控制内容

该配置描述了一个项目单元的元数据、模块布局和溯源追踪。它强制校验必需的字段，并禁止任何额外的属性，从而防止格式错误的条目进入系统。该配置用于：

- 通过名称和功能角色标识项目单元。
- 声明该单元的责任或用途。
- 定义所有子模块的目录路径、角色以及子模块数量。
- 记录溯源元数据，用于追踪和流水线审计。

## 关键参数

| 参数 | 类型 | 必需 | 描述 |
|------|------|------|------|
| `schemaVersion` | 字符串（引用共享模式） | 是 | 此配置单元使用的模式版本，确保向前/向后兼容。 |
| `projectName` | 字符串（非空） | 是 | 在镜像系统中唯一标识项目的名称。 |
| `role` | 字符串（非空） | 是 | 此项目单元的功能角色（例如 `gateway`、`connector`）。 |
| `responsibility` | 字符串（非空） | 是 | 描述此项目单元所承担的责任。 |
| `modules` | 模块对象数组 | 是 | 子模块配置的列表。每个对象包含：<br>- `directory`（路径，必需）<br>- `role`（非空字符串，必需）<br>- `childCount`（整数 ≥ 0，必需） |
| `provenance` | 对象 | 是 | 跟踪来源和处理流程。包含：<br>- `indexedAt`（ISO 时间戳）<br>- `generatorVersion`（非空字符串）<br>- `pipelineStages`（流水线阶段数组） |

## 稳定性与操作风险

- **严格校验**：所有必需字段必须存在，且顶层和模块对象内部不允许添加额外属性。缺失字段或存在额外属性将导致配置被拒绝，这保护了系统免受格式错误的数据影响。
- **非负的 `childCount`**：`childCount` 必须为整数且 ≥ 0。不允许出现负值，因为负值可能导致镜像基础设施中图遍历或依赖解析出错。
- **溯源可审计性**：`provenance` 块需要精确的时间戳、生成器版本和流水线阶段。不正确或缺失的流水线阶段会阻碍调试和审计。操作人员必须确保溯源数据准确且完整。
- **整体数据完整性**：严格遵循此模式可维护 ArchSpine 镜像系统中数据的一致性。任何偏差应在 CI/CD 验证阶段在部署前被发现。

**操作人员提示**：在引入新项目单元之前，务必使用自动化工具对照此模式验证配置文件，检查模式合规性，并确保模块计数和路径与实际的目录结构相符。
