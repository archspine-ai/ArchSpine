# ArchSpine 配置摘要

本配置为 **ArchSpine** 定义了项目元数据、模块结构以及溯源管道。ArchSpine 是一个对仓库知识图谱进行语义索引并生成文档的系统。该配置同时被运行时管道（索引、文档生成）以及验证仓库一致性的工具所消费。

## 关键参数及其含义

| 参数 | 取值 / 约束 | 作用 |
|---|---|---|
| `schemaVersion` | 必须为 `"1.0.0"` | 确保向后兼容性；下游消费者根据此版本号解释配置格式。 |
| `projectName` | 必须为 `"archspine"` | 标识项目以进行路由和验证；不可更改，否则会破坏集成。 |
| `modules` | 必须包含两项：`src`（14 个子项）和 `docs`（4 个子项） | 声明项目顶层目录。`src` 存放索引管道实现，`docs` 存放规范与策略资产。`childCount` 用于结构完整性检查。 |
| `provenance.indexedAt` | ISO 8601 时间戳（例如 `2026-04-02T10:00:00Z`） | 记录上次索引生成的时间，对缓存失效、增量构建和审计追踪至关重要。 |
| `provenance.generatorVersion` | 必须为 `"archspine/1.0.0"` | 锁定生成此配置的生成器版本，确保可复现性。 |
| `provenance.pipelineStages` | 有序列表 `["ast", "llm"]` | 定义处理阶段（先进行抽象语法树分析，再进行大语言模型处理）。顺序决定了依赖链，不可随意更改。 |

## 运维风险与稳定性注意事项

- **严格的版本强制要求**：`schemaVersion` 和 `generatorVersion` 被锁定为固定值。更改或省略它们会导致下游工具拒绝该配置或产生不可预测的结果。请确保工具版本与配置模式同步更新。

- **模块结构要求**：`modules` 数组必须包含所需的目录及其指定的子项数量。缺失 `src` 或 `docs` 会导致索引生成不完整——且这种失败可能是静默的（例如生成了一个空索引文件），难以察觉。建议在持续集成管道中加入结构验证。

- **溯源时间戳的准确性**：错误或过时的 `indexedAt` 会导致缓存系统误用旧索引或触发不必要的全量重建。对于增量构建，请确保每次重新生成索引时都更新该时间戳。

- **管道阶段的顺序**：`pipelineStages` 列表是有序的，必须按顺序包含 `"ast"` 和 `"llm"`。更改顺序或移除某个阶段会破坏索引管道的依赖链，可能导致生成的文档数据不一致。

总之，该配置充当了仓库结构与 ArchSpine 工具之间的契约。任何对强制不变量的偏离都可能损害语义知识图谱及其生成的文档的可靠性。
