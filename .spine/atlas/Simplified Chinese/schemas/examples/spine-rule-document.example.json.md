# ArchSpine 配置概要：`no-direct-db-import` 规则

本配置定义了一条**架构治理规则**，用于强制维持分层架构的边界，防止应用服务层直接导入底层数据库适配器。该规则通过代码分析在构建或 CI 阶段自动执行，对违规行为生成错误报告。

## 规则说明

- **规则 ID**：`no-direct-db-import`，用于在系统中唯一引用和报告。
- **标题**：禁止直接导入数据库层。
- **摘要**：服务层和中间层不得直接依赖 `src/db/` 下的底层数据库适配器。
- **适用范围**：
  - 包含模式：`src/**/*.ts`（所有 TypeScript 文件）
  - 排除模式：`!src/db/**`（`src/db/` 目录下的文件被豁免）
- **严重级别**：`error`，违反该规则将导致构建失败或报告错误，必须强制修复。
- **可执行性**：`true`，表示该规则在检查过程中会被主动执行。
- **理由**：保持领域边界清晰，避免业务层与存储实现耦合。
- **详细说明（Markdown）**：包含规则描述和允许的做法示例，供工具解析生成提示。

## 关键参数说明

| 参数 | 含义与注意事项 |
|------|----------------|
| `schemaVersion` | 规则定义使用的架构版本，当前为 `1.0.0`。不同版本可能对应不同的解析逻辑或兼容性要求，升级时需留意。 |
| `ruleId` | 唯一标识该规则的 ID，用于在系统中引用和报告。修改 ID 会导致历史违规记录无法关联。 |
| `title` | 规则的中文标题，面向开发者展示。保持简洁清晰。 |
| `summary` | 简短描述规则的行为：服务层和中间层不得直接依赖底层数据库适配器。 |
| `appliesTo` | 指定该规则适用的文件路径模式（支持 glob 通配符）。当前包含一个排除规则 `!src/db/**`，即 `src/db/` 下的文件不受此规则约束。**注意**：路径模式的准确性直接影响规则的有效性，若文件迁移或重命名，可能导致误报或漏报。 |
| `severity` | 违规级别。`error` 表示违反该规则将导致构建失败或报告错误，需强制修复。其他可能取值如 `warning` 仅作为提示。 |
| `enforceable` | 布尔值，指示该规则是否在检查过程中执行。设为 `true` 表示必须遵守。若设为 `false`，规则仅记录但不阻止流程。 |
| `rationale` | 解释为何制定此规则：保持领域边界清晰，避免业务层与存储实现耦合。有助于新成员理解架构意图。 |
| `bodyMarkdown` | Markdown 格式的规则详细说明，包括允许的做法示例（如使用服务抽象或仓库接口）。可被工具解析并用于生成开发者提示。 |

## 稳定性与风险提示

- **正面影响**：此规则通过阻止违反分层架构的导入，有效降低系统耦合度，提高可维护性，保持领域边界清晰。
- **潜在风险**：
  - 排除模式 `!src/db/**` 可能不完整或过于严格，导致合法的间接依赖也被禁止。需要持续维护并补充允许的导入模式（例如通过仓库接口层）。
  - 规则依赖文件路径模式匹配，若项目目录结构发生变更（如移动文件夹），可能产生误报或漏报，需同步更新 `appliesTo` 配置。
  - 若出现新的数据库适配器实现但位于不同路径，可能被规则遗漏，需审查排除模式是否全面。
- **建议**：定期审查该规则的适用范围和排除模式，结合代码库实际变更维护；在新模块引入时，注意其导入路径是否合规；将此规则作为架构评审的一部分，确保它与实际分层策略一致。
