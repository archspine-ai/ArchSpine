# ArchSpine：`no-direct-db-import` 架构规则

## 角色
此规则通过禁止服务层和中间层直接导入数据库层（`src/db/`）来强制架构层隔离，确保业务逻辑与存储实现之间保持清晰的领域边界。

## 主要职责
- 防止服务层和中间层代码直接导入数据库适配器。
- 强制业务逻辑与数据访问之间的关注点分离。
- 将任何违反规则的行为视为编译或部署错误。

## 不变约束
- 任何匹配 `src/**/*.ts` 的文件（`src/db/` 目录下的文件除外）不得从 `src/db/*` 导入。
- 违反规则被视为错误（严重级别：`error`）。
- 该规则可由工具自动强制执行，且不可绕过。

## 负面范围（不涉及范围）
- 此规则不限制数据库层内部的导入。
- 它不定义数据库适配器的结构或暴露方式。
- 它不强制任何特定的抽象模式（例如仓库模式），仅禁止直接导入。

## 导出的/外部可见行为
- 规则标识符为 `ruleId: "no-direct-db-import"`。
- 适用于 `src/db/` 之外的所有 TypeScript 文件，由 `appliesTo` 通配模式定义。
- 严重级别为 `error`，违反规则将阻止编译或部署。
- `enforceable` 标志为 `true`，允许在 CI/CD 流水线中自动执行。
- `rationale`（中文）解释了规则目的：保持领域边界清晰，避免业务层耦合底层存储实现。

## 稳定性与风险
此规则通过强制严格的分层架构来增强系统稳定性，防止业务逻辑与数据库细节之间的意外耦合。它降低了存储实现变更时引发级联修改的风险。然而，过于严格的执行可能需要额外的抽象层，增加了初始开发的复杂性。违反规则被视为错误，确保在 CI/CD 流水线中及早发现。
