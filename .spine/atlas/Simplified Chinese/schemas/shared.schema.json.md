# ArchSpine 共享类型与验证模式

本文件定义了 ArchSpine 系统中所有组件共享的核心类型定义和验证模式，确保数据一致性、安全性和互操作性。

## 角色

定义 ArchSpine 系统中共享的可复用类型定义和验证模式。

## 主要职责

- **模式版本与时间戳**：强制使用语义化版本格式（`X.Y.Z`）确保模式兼容性，并验证 ISO 8601 时间戳用于事件记录和同步。
- **路径验证**：验证仓库相对路径（不得以 `/`、`./`、`../` 开头，不得包含反斜杠或双斜杠）和作用域路径（可为空或满足相同约束）。
- **内容完整性**：验证 SHA-256 内容哈希（64 字符小写十六进制字符串）用于文件完整性校验。
- **区域设置与语言**：验证语言标签（例如 `en`、`zh-CN`），确保语言选择键为非空字符串用于本地化映射。
- **分类**：枚举源代码语言、文件类型（源代码、配置、文档、其他）、符号类型和依赖关系，供分析管道使用。
- **来源追踪**：跟踪依赖边是来自 AST 还是推断得出，并记录数据由哪个处理阶段产生（AST、LLM 等）。
- **通用类型**：定义 `nonEmptyString` 和 `stringArray` 用于灵活使用。
- **本地化内容**：将本地化内容组织为键值对结构，使用经过验证的语言键。

## 重要不变约束

- `schemaVersion` 必须遵循语义化版本格式（`X.Y.Z`）。
- `repoRelativePath` 不得以 `/`、`./` 或 `../` 开头，不得包含反斜杠或双斜杠。
- `scopePath` 必须为空或满足与 `repoRelativePath` 相同的路径约束。
- `contentHash` 必须为 64 字符小写十六进制字符串（SHA-256）。
- `locale` 必须匹配有效的语言标签模式。
- `sourceLanguage`、`fileKind`、`symbolKind`、`dependencyRelation`、`edgeProvenance` 和 `pipelineStage` 必须为其各自枚举值之一。
- `nonEmptyString` 必须至少包含一个字符。
- `localizedContentMap` 的属性名称必须符合 `languageSelectionKey` 模式。

## 负面范围（不在范围内）

本文件不处理任何运行时逻辑、数据处理或外部系统交互。它纯粹是一个模式定义和验证层。

## 最重要的导出行为

主要的导出行为是一组验证函数和类型定义，用于强制执行上述不变约束。这些被 ArchSpine 的所有其他组件使用，以确保数据完整性和一致性。

## 稳定性与风险

此模式文件是 ArchSpine 数据完整性的基础。验证不当可能导致路径遍历漏洞、哈希不匹配造成数据损坏，或区域设置解析错误破坏国际化功能。严格的模式和枚举减少了歧义，但在添加新语言或管道阶段时需要谨慎维护。总体而言，该文件强制执行一致的数据形状，可防止许多常见的配置和安全错误。
