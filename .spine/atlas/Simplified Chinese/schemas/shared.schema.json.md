# ArchSpine 共享定义 — 配置摘要

此模式定义了 ArchSpine 所有数据组件的基础验证类型和结构约束，确保整个流水线中格式一致与类型安全——从源码分析到图形构建均受此约束。

## 该配置控制的内容

该模式规定了核心数据字段的塑形与验证方式。操作人员必须确保进入系统的任何数据符合这些模式；违反将导致拒绝或流水线失败。

### 关键参数

| 参数 | 用途与影响 |
|------|------------|
| **schemaVersion** | 强制版本字符串符合语义化版本格式（例如 `1.0.0`）。影响兼容性检查和模式迁移决策。 |
| **isoTimestamp** | 验证 ISO 8601 日期时间字符串。影响事件排序、审计追踪以及多工作节点的同步。 |
| **repoRelativePath** | 清理相对于仓库根目录的文件路径。阻止路径遍历（`../`）、绝对路径（`/`）或格式错误的引用（反斜杠、双斜杠）。 |
| **scopePath** | 规范化作用域标识符；允许空字符串表示根作用域。影响模块解析和命名空间隔离。 |
| **contentHash** | 强制 SHA‑256 十六进制摘要（恰好 64 个小写十六进制字符）。对缓存失效和内容完整性验证至关重要。 |
| **locale** | 验证 BCP 47 区域代码（例如 `zh-CN`、`en-US`）。控制元数据和用户界面中的国际化匹配。 |
| **languageSelectionKey** | 用作多语言内容映射键的非空字符串。确保每个条目都可寻址。 |
| **sourceLanguage** | 限制源代码语言为已知类型（`typescript`、`javascript`、`python`、`unsupported`）。防止不支持的语言进入分析。 |
| **fileKind** | 将文件分类为 `source`（源码）、`config`（配置）、`document`（文档）或 `other`（其他）。按类型强制执行处理规则。 |
| **symbolKind** | 对符号进行分类（`class` 类、`function` 函数、`variable` 变量、`interface` 接口、`type` 类型、`enum` 枚举、`unknown` 未知）。驱动依赖图语义和图谱生成。 |
| **dependencyRelation** | 定义模块间关系类型（`import` 导入、`reexport` 重导出、`type-import` 类型导入）。影响构建顺序、摇树优化和模块边界决策。 |
| **edgeProvenance** | 标记依赖边的发现方式（`ast` 基于 AST、`inferred` 推断）。区分确定性关系与推测性关系，支持基于可信度的过滤。 |
| **pipelineStage** | 指示数据由哪个分析阶段生成（`ast`、`llm`、`lite-ast` 轻量 AST、`lite-llm` 轻量 LLM、`fallback` 回退）。支持按阶段处理、质量评分和回退逻辑。 |
| **nonEmptyString** | 确保字符串值至少有一个字符。防止关键数据中出现空字段（例如标识符）。 |
| **stringArray** | 定义一个同质的字符串数组。适用于标签、路径或 ID 列表。 |
| **localizedContentMap** | 以区域代码（必须为有效的 `languageSelectionKey`）为键存储多语言内容。支持本地化配置、文档和面向用户的字符串。 |

## 稳定性与风险

此模式建立了**强不变量**，防止格式错误或不明确的数据在 ArchSpine 中传播。但操作人员需注意以下风险：

- **模式严格性**：过于严格的正则可能拒绝有效的边界情况——例如非标准的语义版本格式（如 `1.0.0-beta`）在未扩展模式时不被允许。未来的区域模式（如 `zh-Hans-CN`）若符合 BCP 47 则受支持。
- **枚举过时**：语言、文件种类、符号种类等枚举值可能随生态系统发展而过时。增加新值需要更新模式并与下游消费者协调。
- **迁移开销**：对这些基础定义的任何更改都可能破坏现有数据。建议对模式本身采用语义化版本管理；操作人员应锁定数据生产者和消费者的版本。
- **建议**：定期审查模式以满足新需求（例如新增源码语言、新的依赖关系类型），同时不破坏现有记录。使用 `pipelineStage` 字段追踪数据由哪个分析版本产生，促进渐进式采用。
