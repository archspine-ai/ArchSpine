# ArchSpine SpineRule 配置摘要

本配置定义了 **SpineRule 文档** 的标准结构和验证规则——这些文档编码了 ArchSpine 镜像编排系统中的可执行策略。所有控制镜像行为的策略都必须以有效的 SpineRule 形式表达。

## 此配置控制的内容

- **规则文档的有效性** – 只有通过 JSON Schema 验证的文档才会被接受。
- **必需的元数据** – 每条规则必须提供 `schemaVersion`、`ruleId`、`title`、`summary`、`appliesTo`、`severity`、`enforceable` 和 `bodyMarkdown`。
- **命名一致性** – `ruleId` 必须为小写字母加连字符（例如 `my-rule`）。
- **严重级别分类** – 规则被归类为 `advisory`（建议）、`warning`（警告）或 `error`（错误），这决定了下游执行行为。
- **目标选择** – `appliesTo` 数组列出了规则所影响的子系统或组件，至少需要一个条目。
- **可执行性** – `enforceable` 布尔值指示规则是否可以自动执行，还是仅提供信息。

## 关键参数

| 参数            | 描述                                                                 |
|----------------|----------------------------------------------------------------------|
| `schemaVersion` | 架构版本标识符，确保与解析器的兼容性。                                     |
| `ruleId`       | 唯一标识符，使用小写字母和连字符，用于引用和更新操作。                         |
| `title`        | 简短的人类可读名称，用于快速提供上下文。                                   |
| `summary`      | 规则意图和作用范围的一行描述。                                           |
| `appliesTo`    | 目标子系统/组件名称的数组；必须非空。                                     |
| `severity`     | 严重级别：`advisory`（建议）、`warning`（潜在问题）、`error`（严格违规）。 |
| `enforceable`  | 布尔标志：`true` 表示规则可以自动执行。                                   |
| `rationale`    | 可选的规则理由文本（字符串或 null）。                                     |
| `bodyMarkdown` | 规则的详细内容（Markdown 格式），必须非空以确保可操作性。                    |

## 稳定性与风险

- **严格验证** – 任何缺少必填字段、模式不匹配或多余属性都会导致 **立即解析失败**。这防止了不明确或格式错误的规则进入系统。
- **过度约束的风险** – 严格的模式可能限制规则作者的灵活表达，但这种权衡保证了 **分布式镜像节点间的确定性执行**。操作员应确保规则作者了解必填字段和模式要求，以避免拒绝。
- **版本兼容性** – `schemaVersion` 字段必须与解析器预期的版本匹配；使用过时的版本将导致失败。
