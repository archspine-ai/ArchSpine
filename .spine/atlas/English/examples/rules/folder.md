This directory houses the core architectural and naming conventions that enforce codebase consistency and layering rules for the ArchSpine mirror system. It contains two primary submodules:

- **layered-architecture.yml**: Defines the layered architecture model with strict constraints to ensure domain layer purity, prevent reverse dependencies, and mandate that the API layer directly calls the domain layer. This file is critical for maintaining a clean separation of concerns and preventing architectural drift.
- **naming-conventions.yml**: Establishes project-wide naming standards to ensure consistency across a large monorepo. Key rules include prefixing internal interfaces with `I` (e.g., `IUserRepository`), requiring test files to use `.test.ts` or `.spec.ts` suffixes, and assigning severity levels (Warning, Error) to each rule with documented rationale to guide enforcement.

The most important implementation areas are enforcing domain isolation, preventing upward dependencies from the domain to outer layers, and ensuring all developers follow a uniform naming pattern. Together, these files form the governance backbone for codebase structure and readability.
