This directory defines the architectural foundation and naming rules for the ArchSpine mirror system. It contains two principal configuration files that together enforce clean dependency direction and codebase consistency. The files are grouped by concern: one for layered architecture constraints (`layered-architecture.yml`) and one for naming conventions (`naming-conventions.yml`). The most critical implementation areas are (1) domain isolation, interface dependency, and API layer separation as specified in the layered architecture file, and (2) interface naming with the 'I' prefix in `src/types/**/*.ts` and test file suffix rules (`*.test.ts` or `*.spec.ts`) in the `tests/` directory. Both files provide severity levels and rationales for each rule, serving as a reference for developers and automated validation tools.
