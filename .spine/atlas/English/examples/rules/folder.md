The `examples/rules` directory serves as the governance backbone of the ArchSpine project. It holds the architectural constraint definitions that enforce structural integrity and naming consistency across the entire monorepo. Two concrete rule files define the core constraints: `layered-architecture.yml` prescribes strict dependency isolation between architectural layers (e.g., domain, application, infrastructure), while `naming-conventions.yml` standardizes file, module, and symbol naming to prevent drift in a large codebase. Together, these files implement the project’s policy of “rules-as-code”, enabling automated validation during development and CI. This directory is the authoritative source for how ArchSpine modules must be structured and named, directly supporting maintainability and scalable collaboration.
