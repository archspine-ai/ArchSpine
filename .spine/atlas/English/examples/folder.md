The `examples` directory serves as the primary repository of sample configurations and rule definitions for the ArchSpine mirror system. It is organized into two distinct submodules that together demonstrate how to maintain system integrity and consistency through structured state management and architectural enforcement.

- **`examples/demo-project`**: This subdirectory bundles root-level configuration for system scanning, hooks, LLM providers, file extension language support, synchronization health monitoring, and state‑sync snapshots. It also contains runtime execution records for pipeline stages, file processing, and hook behavior. Collectively, these components enable the mirroring process while preserving architectural integrity.

- **`examples/rules`**: This submodule focuses on enforcing architectural and naming rules. It defines layered architecture constraints to ensure clean dependency direction and establishes consistent naming conventions across the codebase, thereby preventing code drift and maintaining a predictable structure.

The most important implementation areas highlighted here are **system configuration and state synchronization** (via `demo-project`) and **rule‑based architectural governance** (via `rules`). Together they illustrate how the ArchSpine system achieves reliable mirroring and long‑term codebase consistency.
