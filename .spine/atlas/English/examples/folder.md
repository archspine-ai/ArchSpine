The `examples/` directory is the practical backbone of the ArchSpine mirror system, serving both as a concrete demonstration and an enforcement guide for architectural rules. It is organized into two notable submodules:

- **demo-project**: A full demonstration project that shows how ArchSpine works in practice. It contains the configuration file (`.spine`), a visual walkthrough (`demo.gif`), and source code under `src/`. This submodule is the primary reference for understanding ArchSpine’s architecture and usage.

- **rules**: Defines the architectural and naming conventions that enforce codebase consistency and proper layering. It maintains strict rules such as: ensuring domain layer purity, prohibiting reverse dependencies, requiring the API layer to directly call the domain layer, and applying project‑wide naming conventions (e.g., prefix internal interfaces with `I`, use `.test.ts` or `.spec.ts` suffixes for test files). Each rule has a defined severity level and a documented rationale.

The most important implementation areas are the layered architecture constraints and the naming convention definitions, which together make the system’s architecture both exemplifiable and enforceable.
