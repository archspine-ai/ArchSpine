The `src/api` directory within the ArchSpine index holds the core metadata schema definitions for source code analysis in the mirror system. It defines how directories and files are structurally described, including indexing provenance, dependency graphs, and architectural rule enforcement. Two concrete submodules make up this directory:

- **folder.json**: Captures the structural metadata and indexing provenance for a source code directory, serving as the schema for describing folders in the mirrored project.
- **handler.ts.json**: Defines the full semantic metadata, dependency graph, and structural skeleton for an individual source file (`handler.ts` in this demo), enabling fine-grained code analysis.

This directory is essential for the indexing pipeline (AST + LLM stages) and provides the contract that other parts of ArchSpine rely on to understand the project's structure and intent.
