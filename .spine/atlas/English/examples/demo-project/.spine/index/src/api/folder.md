This directory, `/api`, serves as the metadata schema hub for the ArchSpine mirror system’s indexing layer. It defines how both directories and individual source files are structurally and semantically recorded, including provenance, dependency graphs, and architectural rule enforcement.

Two configuration files stand out:
- `folder.json` — captures the structural metadata and indexing provenance for an entire directory.
- `handler.ts.json` — captures the semantic metadata, dependency graph, and structural skeleton for a single source file.

Together, these files form the core of ArchSpine’s index schema. The most critical implementation areas are the design of the metadata schemas, integration with the AST and LLM pipeline stages (noted in the provenance), and the enforcement of architectural rules via the defined relationships. Concrete submodules include the metadata schema validator, the dependency graph builder that consumes these definitions, and the provenance logger that records the indexing pipeline stages (ast, llm).
