# ArchSpine Mirror System – `.spine` Directory Summary

The `.spine` directory is the central configuration and rule hub for the ArchSpine mirror system. It defines system-level parameters, structural guidelines, architectural rules, synchronization policies, indexing, validation, routing, and access control for mirrored data across distributed nodes. The directory is organized into several key submodules:

- **`atlas/`** – Aggregates core configuration and rule definitions that govern the mirror system's behavior and structure.
- **`config.json`** – Project-level configuration for localization, LLM provider, MCP context mode, hooks, artifact storage, and file scanning policy.
- **`index/`** – Root configuration and structural index for the mirror system.
- **`languages.json`** – Language and extension registry for the mirror system.
- **`manifest.json`** – Health-check and sync-state snapshot for the mirror system.
- **`rules/`** – Architectural governance specification for the ArchSpine project.
- **`runtime/`** – Execution checkpoint records for synchronization runs.

The most critical implementation areas are configuration management (via `config.json` and `languages.json`), architectural governance (via `rules/`), and synchronization state tracking (via `manifest.json` and `runtime/`). The `atlas/` and `index/` directories provide foundational structure and indexing, while `rules/` enforces consistency and extensibility across the system.
