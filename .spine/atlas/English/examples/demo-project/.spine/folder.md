The `.spine` directory is the central configuration and state store for the ArchSpine analysis system within this demo project. It groups project-level settings (LLM provider, scan policy, ignore chains) in `config.json`, language detection mappings in `languages.json`, and a lightweight health-check snapshot (`manifest.json`) that exposes sync status, violation counts, and generator compatibility. Two subdirectories house distinct enforcement areas: `rules` defines project-specific architectural constraints (e.g., layer isolation, documentation requirements), while `runtime` tracks the lifecycle of sync pipeline runs via checkpoint records, enabling reliable resumption and audit trails. Together, these artifacts manage the entire ArchSpine analysis pipeline from configuration and language support to rule enforcement and runtime monitoring.
