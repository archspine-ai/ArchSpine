The `checkpoints` directory under `.spine/runtime` stores execution records for synchronization runs in the ArchSpine mirror system. It currently contains a single configuration file `sync.json`, which tracks the full lifecycle and status of a sync operation. This record includes metadata such as whether the run was full or incremental, the hook mode used, and any resume state. It logs per-stage progress with timestamps, captures filtered file paths and affected directories during the scan-cleanup phase, and records AST extraction results per source file alongside summarization results for each configuration or documentation file. The `sync.json` file is the concrete submodule that embodies all these tracking responsibilities.
