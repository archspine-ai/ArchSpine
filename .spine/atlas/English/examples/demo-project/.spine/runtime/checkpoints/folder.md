The `checkpoints` directory, located at `examples/demo-project/.spine/runtime/checkpoints`, holds execution records for each synchronization run performed by the ArchSpine mirror system. It plays a critical role in tracking the full lifecycle of sync operations—recording whether a run was full or incremental, which hook mode was active, and whether the run can be resumed from a saved state. The directory also captures per-stage progress and completion timestamps, logs filtered file paths and affected directories during scan-cleanup, and stores AST extraction results for source files along with summarization results for configuration or documentation files.

The sole notable child is `sync.json`, a configuration‑type file that embodies the entire checkpoint record for a single sync execution. All the metadata described above—status, stage timestamps, file paths, and extracted data—is serialized into this file. Because there is only one checkpoint at a time (the most recent run), no sub‑grouping of children is needed. The key implementation areas are the lifecycle management of sync runs, the stage‑based progress logging, and the AST‑to‑summary pipeline that feeds higher‑level documentation generation.
