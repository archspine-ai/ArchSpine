The `sync_logs/` directory stores execution records of synchronization runs in the ArchSpine mirror system. The primary child is `sync.json`, which captures the full lifecycle and outcome of each sync command. It reports the status and timing of every pipeline stage, lists all processed files and directories for traceability, and contains metadata used for resumption logic and hook behavior. This directory is essential for monitoring sync health, debugging failures, and enabling deterministic replay or continuation of interrupted operations.
