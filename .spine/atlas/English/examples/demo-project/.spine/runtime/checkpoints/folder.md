This directory holds execution checkpoint records for the ArchSpine sync pipeline. Its primary content is the `sync.json` file, which is not a configuration file but a detailed trace of a specific sync command run. Each instance of `sync.json` logs the complete lifecycle of a sync operation: start time, timestamps for each pipeline stage (reconciliation, scan-cleanup, AST extraction, summarization), per-item completion status within the AST extraction and summarization stages, and metadata indicating whether the run was full, in hook mode, or a resumed run (via `resumedFromRunId`). This checkpoint enables reliable resumption of interrupted runs and provides an authoritative audit trail for debugging and post-mortem analysis. The implementation area that matters most is the checkpoint logic that writes and reads this file, ensuring crash-resilient state management across pipeline stages.
