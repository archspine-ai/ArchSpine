The `api` directory is the public-facing interface layer of the ArchSpine mirror system. It acts as the entry point for all external interactions, translating incoming data into domain-friendly structures and handling request validation along with error propagation back to callers. This layer orchestrates user creation requests through the `CreateUserHandler` module.

The directory contains two primary children: `folder.md` — a documentation file describing the Public API and Interface Layer responsibilities, and `handler.ts.md` — the core API request handler that serves as the system's entry point. The most critical implementation areas here are request validation pipelines, data format translation, and the `CreateUserHandler` orchestration logic.
