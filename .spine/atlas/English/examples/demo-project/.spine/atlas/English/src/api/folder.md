The `api` directory serves as the public-facing interface layer of the ArchSpine mirror system. It is the external entry point where incoming requests are received, translated into domain-friendly structures, and validated before being passed to internal handlers. The directory contains two documents: `folder.md` provides an overview of the API layer, while `handler.ts.md` defines the request handler that orchestrates user creation via the `CreateUserHandler`. Error propagation and request validation are managed here, ensuring that the internal domain logic remains decoupled from external interaction patterns. This layer is critical for system integration and security, as it governs all inbound communication.
