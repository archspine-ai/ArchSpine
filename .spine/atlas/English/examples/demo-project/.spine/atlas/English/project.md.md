# ArchSpine Demonstration Project

## Purpose
This document serves as a minimal demonstration project for the ArchSpine protocol, designed to showcase how architectural rules and semantic context are managed within a local repository.

## Context and Audience
Intended for developers and architects evaluating ArchSpine's capabilities, particularly those interested in domain-driven design enforcement and architectural boundary validation.

## Key Takeaways
- Demonstrates separation of domain logic from infrastructure using DDD principles
- Includes intentional violations to test ArchSpine's rule checking
- Shows how .spine/ directory stores semantic context locally

## Scope
This project is a demonstration and validation of the ArchSpine protocol's architectural governance capabilities through a minimal example project. It is **not** intended for production use, does not represent a complete system architecture, and does not include deployment or operational procedures.

## Responsibilities
- Illustrating Domain-Driven Design separation between domain logic and infrastructure
- Demonstrating ArchSpine's boundary enforcement via intentional architectural violations
- Showcasing the local control plane and semantic context storage in .spine/ directory

## Out of Scope

- Complete system architecture documentation
- Deployment or operational procedures
