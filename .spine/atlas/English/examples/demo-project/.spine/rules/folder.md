This directory contains the ArchSpine architecture rule definition for the project. Its sole child is the `arch.yml` file, which enforces critical architectural constraints. The rules defined within this file are grouped into two main areas: **Layer Isolation** and **Documentation Requirements**. The Layer Isolation rule prevents the API layer from directly importing infrastructure modules, while the Documentation rule mandates TSDoc comments for all public domain classes and methods. Each rule specifies a severity level (`error` or `warning`), a rationale, and scoped target paths (e.g., `src/api/**`, `src/domain/**`) for precise application. Implementing these rules ensures that the project’s architecture remains clean, maintainable, and well-documented.
