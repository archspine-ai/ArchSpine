This directory serves as the architectural policy hub for the ArchSpine project. It holds the core rules and configuration files that define how the project's layers interact and how documentation must be formatted. The primary artifact here is the `arch.yml` file, which encodes two critical constraints:

- **Layer isolation**: API handlers are permitted to depend on the domain layer but must _not_ directly import from the infrastructure layer. This enforces a clean hexagonal architecture.
- **Documentation standards**: All public classes and methods within the domain layer must carry TSDoc comments, ensuring that the core business logic remains self-documenting and discoverable.

These rules apply project-wide, making this directory essential for maintaining architectural integrity. Developers should consult `arch.yml` before adding new modules or dependencies, as it defines the boundaries that keep the codebase modular and testable. No other notable subdirectories or files exist outside of this configuration.
