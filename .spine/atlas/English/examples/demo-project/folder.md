The `.spine` directory serves as the root configuration and state management hub for the ArchSpine mirror system. It collectively defines all settings that govern the mirroring process: scanning policies, hook definitions, and LLM provider configurations. Beyond runtime parameters, it maintains a language-to-extension mapping for file recognition, synchronization health indicators, and point-in-time sync-state snapshots that preserve mirror integrity across sessions. Architectural rule enforcement is handled through dedicated submodules that impose layer isolation and documentation standards on the project structure. The directory also archives runtime execution records, including pipeline stage logs, file-level processing traces, and hook behavior audit trails. Concrete submodules within this directory typically include `scanning.yaml` for scan configuration, `hooks/` for custom hook scripts, `llm.yml` for AI provider settings, `extensions/` for file-type language mappings, `sync-state/` for snapshot storage, `rules/` for architectural constraints, and `pipeline/` for execution logs.
