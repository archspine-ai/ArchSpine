# `src/engines` — Core Engine & CLI Command Modules

This directory contains the operational heart of the ArchSpine mirror system. It houses all scanning, aggregation, analysis, rule enforcement, context resolution, fix generation, and CLI reporting logic. The modules here are responsible for discovering, validating, and documenting the codebase architecture.

## Notable Children

- **Scanner Subsystem** (`scanner.ts`, `scanner-utils.ts`, `scanner-git.ts`): The core file system scanner that discovers and filters repository files using layered ignore rules, git diff integration, and configurable scan policies. The `scanner-git.ts` module provides a reusable git command execution client, while `scanner-utils.ts` offers path and pattern normalization utilities.

- **Aggregator** (`aggregator.ts`): The core engine class that traverses the spine filesystem and aggregates index and atlas data into structured `SpineUnit` collections for downstream sync and view operations.

- **Rule Engine** (`rules.ts`): Loads, stores, and matches architectural rules against file paths. This is the enforcement backbone for architectural compliance.

- **Context Resolution** (`context.ts`, `context-path-resolver.ts`, `context-relevance.ts`): Provides architectural context resolution for dependency analysis and relevance scoring. The `context-path-resolver.ts` specializes in resolving relative import targets within a source tree, while `context-relevance.ts` extracts and filters keywords from architectural rule text.

- **Fix Generation** (`fix.ts`, `fix-prompt.ts`): The public API facade for the fix service module, with `fix-prompt.ts` providing an LLM prompt template generator for architectural violation fixes.

- **CLI Commands** (`god.ts`, `info.ts`, `usage.ts`): CLI orchestrators for generating comprehensive architectural dossiers (God Mode reports), checking project sync status and system health, and producing formatted usage/audit reports from the manifest database.

## Key Implementation Areas

- **Scanning & Discovery**: The scanner subsystem is the entry point for all file discovery, using layered ignore rules and git integration.
- **Aggregation & Structuring**: The aggregator transforms raw index/atlas data into structured collections.
- **Rule Enforcement**: The rules engine matches architectural rules against file paths.
- **Context & Relevance**: Context resolution engines provide dependency analysis and relevance scoring.
- **Fix Generation**: LLM-powered fix prompt generation for architectural violations.
- **CLI Reporting**: God Mode reports, system health checks, and usage audits.
