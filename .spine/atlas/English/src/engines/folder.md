# `src/engines` — Core Engine Modules

The `src/engines` directory is the operational heart of the ArchSpine mirror system. It contains all engine components that implement the file scanning, rule enforcement, dependency analysis, and report generation pipeline. These modules are grouped into several distinct areas:

- **Scanner Infrastructure** (`scanner.ts`, `scanner-git.ts`, `scanner-utils.ts`): The `scanner` engine discovers, filters, and reports on repository files using layered ignore rules and git diff integration. Utility modules provide git execution and path/pattern normalization.
- **Context & Rule Resolution** (`context.ts`, `context-path-resolver.ts`, `context-relevance.ts`, `rules.ts`): The `context` engine resolves architectural context for dependency analysis. `context-path-resolver` specializes in resolving relative imports. `context-relevance` extracts keywords from rule text. `rules` loads, stores, and matches architectural rules against file paths.
- **Violation Fix System** (`fix-prompt.ts`, `fix.ts`): `fix-prompt` generates LLM prompts to propose architectural violation fixes; `fix` is the public facade for the fix service module.
- **Reporting & Health Check** (`god.ts`, `info.ts`, `usage.ts`): `god` produces the comprehensive God Mode architectural dossier. `info` checks sync status and system health. `usage` generates formatted usage and audit reports from the manifest database.
- **Aggregation & Facade** (`aggregator.ts`, `check.ts`): `aggregator` traverses the spine filesystem to build structured `SpineUnit` collections. `check` re-exports the check subsystem for CLI and service consumption.

Key concrete submodules to note are the scanner engine (`scanner.ts`), the context resolution engine (`context.ts`), the rule engine (`rules.ts`), the aggregator (`aggregator.ts`), and the god mode reporter (`god.ts`). These collectively provide the end-to-end pipeline for mirroring and analyzing repository architecture.
