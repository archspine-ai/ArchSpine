This directory houses language-specific YAML configuration files that define pattern-based extraction rules for code structure analysis in the ArchSpine mirror system. Each file corresponds to a supported programming language—C, C++, Go, Java, Python, Rust, and TypeScript—and specifies how the system identifies imports (e.g., `#include`, `import`, `use`), exports (functions, classes, structs, interfaces, traits, variables, constants), and usages (function calls, field access, method calls, property access) within source code. These configurations collectively enable ArchSpine to parse multiple languages automatically, extracting semantic relationships and driving downstream tasks such as automatic documentation generation and dependency mapping. The key implementation areas are the reflection engine’s pattern matching grammar and the extensible configuration mechanism that allows adding new languages.
