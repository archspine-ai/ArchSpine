The `src/tasks` directory contains the core pipeline tasks and stages that drive the ArchSpine mirror system's automated workflows. Each file is a discrete `SpineTask` or stage orchestrator, forming a comprehensive processing pipeline that transforms source code into a structured mirror with documentation, insights, and compliance checks.

The children are naturally grouped by pipeline stage:

- **Scanning & Reconciliation:** `scan-cleanup.ts` (cleans orphaned files), `reconcile.ts` (synchronizes manifest with actual repository state).
- **AST Extraction:** `ast-extra.ts` (extracts symbols and registers them).
- **Summarization:** `lite-summarize.ts` (lightweight per-file summaries), `summarize.ts` (LLM-based semantic summary generation).
- **Validation & Fixing:** `validate.ts` (LLM-powered architectural rule checking), `fix.ts` (automated LLM-driven violation fixing).
- **Aggregation & Backfill:** `aggregate.ts` (hierarchical content aggregation), `document-backfill.ts` (generates project markdown from JSON via LLM, with idempotency).
- **State Commit:** `state-commit.ts` (writes file hashes and metadata to SQLite).
- **Derivation (post-commit):** `post-commit-derivation.ts` (orchestrates views, aggregations, reverse indices), `view-derivation.ts` (public surface, risk hotspots, architecture diagram), `reverse-index.ts` (reverse dependency edges from forward indices).

Key implementation areas include the LLM integration in `summarize.ts`, `validate.ts`, `fix.ts`, and `document-backfill.ts`; the AST-driven symbol extraction; the reconciliation logic for syncing with git/disk state; and the view derivation pipeline for generating architectural insights.
