The `src/assets/templates/rules` directory serves as the central repository for architectural and coding standards in the ArchSpine project. It contains two primary rule files that together define the project's structural and stylistic guidelines.

- **`layered-architecture.yml`**: Specifies the layered architecture model, including the boundaries and dependency constraints between the CLI, services, core, tasks, engines, and infrastructure layers. This file ensures that the codebase maintains clear separation of concerns and adheres to the intended hierarchy.
- **`naming-conventions.yml`**: Establishes consistent naming conventions across the entire monorepo. It covers interface prefixes, test file suffixes, and other naming practices, attaching severity levels and rationales to each rule to promote codebase uniformity.

These files are the principal implementation artifacts for enforcing the project's architectural and stylistic consistency, and they are referenced by both human developers and automated validation tools.
