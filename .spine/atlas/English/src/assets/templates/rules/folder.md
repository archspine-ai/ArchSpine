The `archspine` configuration directory serves as the central repository for architectural standards and conventions in the ArchSpine mirror system. It defines and enforces layered architecture constraints and naming conventions to maintain code consistency and clean separation of concerns across the monorepo. Two key configuration files reside here: **layered-architecture.yml**, which specifies import and dependency boundaries for all layers (CLI, services, core, tasks, engines, infra) along with severity levels and rationales, and **naming-conventions.yml**, which enforces interface naming rules (prefix `I`) for TypeScript types and test file suffixes (`.test.ts` or `.spec.ts`) with appropriate severity levels. Together, they guide contributors in respecting architectural boundaries and following consistent naming practices, both essential for the long-term health of the project.
