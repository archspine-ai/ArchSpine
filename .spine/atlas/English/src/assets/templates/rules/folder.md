This directory establishes the architectural governance and coding standards for the ArchSpine codebase. It contains two key rule definition files: `layered-architecture.yml` and `naming-conventions.yml`. The layered architecture rules enforce dependency flow and separation of concerns across the CLI, services, core, tasks, engines, and infrastructure layers, assigning severity levels (Error/Warning) to each constraint. These rules prevent layer violations and guide contributors on correct module responsibilities. The naming conventions mandate that TypeScript interfaces must begin with `I` (e.g., `ISomeInterface`) and that test files must end with `.test.ts` or `.spec.ts`. Together, these files define the concrete submodules that ensure a consistent, maintainable architecture and a clear naming standard for all developers and automated agents.
