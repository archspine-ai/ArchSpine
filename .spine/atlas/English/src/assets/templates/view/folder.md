# `analysis/` — Public Interface Inventory & Risk Assessment Hub

The `analysis/` directory aggregates auto-generated inventories and risk assessments that capture the project's public API surface and codebase health. It acts as the single source of truth for everything the `ArchSpine` system exposes externally — CLI commands, MCP tools, and exported modules — while also pinpointing high‑risk files that require refactoring or focused testing.

## Notable Children

- **`public-surface.md`** — An auto‑generated catalog of all public entry points. Lists CLI commands with their signatures and descriptions, all MCP (Model Context Protocol) tools and resources, and every exported module together with its public symbols. This file is refreshed via the `spine sync` command to stay synchronized with the actual implementation, serving both human readers and automated tooling.

- **`risk-hotspots.md`** — A risk assessment report that ranks files by complexity, change frequency, dependency issues, and architectural violations. Each high‑risk file is accompanied by a detailed analysis, making it easier to prioritize refactoring or testing efforts. Like `public-surface.md`, this snapshot is regenerated by `spine sync`.

## Key Implementation Areas

- **Public Surface Integrity** — Ensuring that all CLI commands, MCP tools, and module exports are accurately documented and automatically kept in sync with the codebase.  
- **Risk Prioritization** — Using quantitative and qualitative factors to surface the most problematic files, enabling targeted refactoring.  
- **Automated Refresh** — Both files rely on the `spine sync` command to maintain their accuracy, so the generation pipeline is a critical piece of the analysis workflow.

These two files collectively provide a concise yet comprehensive view of what the system exposes and where maintainability efforts should be directed.
