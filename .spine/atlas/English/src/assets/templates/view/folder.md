This directory holds the public API surface documentation and risk hotspot analysis for the ArchSpine mirror system. It brings together two key files that collectively define the system’s external interface and internal maintenance priorities.

- **public-surface.md** — Automatically generated document that lists all public CLI commands and flags, MCP tools/resources/prompts, and exported module symbols. It serves as the single source of truth for the system’s public surface.
- **risk-hotspots.md** — Dynamically generated report highlighting the highest-risk files in the codebase, ranked by composite scores. It includes detailed analysis of risk factors and impact, helping teams focus maintenance efforts where they matter most.

Together, these files give developers a clear view of what ArchSpine exposes externally and where internal code quality needs attention. The CLI, MCP, and module entry points are the most critical implementation areas to monitor, as any change there directly affects users. The risk hotspot data is equally important for prioritizing refactoring and testing.
