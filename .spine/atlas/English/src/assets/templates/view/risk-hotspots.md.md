# ArchSpine Hotspot Report

This document is a generated report that surfaces the riskiest files in a project. It ranks files by a composite risk score and explains why each file is flagged, helping teams prioritize refactoring or review efforts.

## Purpose

The report provides a data-driven overview of code health by identifying high-risk files based on factors such as complexity, change frequency, and dependency density.

## Key Responsibilities

- Displaying a ranked table of files with the highest risk scores
- Listing risk factors and impact for each identified file
- Providing detailed analysis for each hotspot file
- Serving as an experimental view that can be refreshed via the `spine sync` command

## Out of Scope

- Prescribing specific remediation steps or fixes
- Tracking risk over time or providing historical trend analysis
- Replacing a full code review or security audit

## Context and Audience

Intended for developers, tech leads, and code reviewers who need a quick, data-driven overview of code health. It is most useful during maintenance sprints, pre-release audits, or when onboarding onto a new codebase.

## Key Takeaways

- Files are ranked by risk score, combining factors like size, churn, and coupling.
- Each hotspot includes a breakdown of contributing risk factors and estimated impact.
- The report is experimental and can be regenerated by running `spine sync`.
