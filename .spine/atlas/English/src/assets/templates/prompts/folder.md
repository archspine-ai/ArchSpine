The `prompts` directory under `src/assets/templates` serves as the prompt template engine and schema definition hub for the ArchSpine summarization pipeline. It houses all components required to construct structured architecture summaries and diagrams from project and folder insights. The directory is organized into five TypeScript modules, each with a distinct responsibility: `schemas.ts` provides the JSON schema templates that define the semantic summary unit format; `arch-diagram.ts` contains the LLM prompt template for generating structured JSON architecture diagrams; `blocks.ts` offers pure utility functions for rendering structured text blocks used within those prompts; `examples.ts` supplies few-shot examples for semantic role description generation; and `index.ts` acts as the public API facade, aggregating and re-exporting all these modules for easy consumption. The most critical implementation areas are the arch-diagram template and the schema definitions, as they directly govern the output format of the pipeline. Developers integrating with ArchSpine’s prompt engine should pay special attention to `schemas.ts` and `arch-diagram.ts` for understanding the structural constraints and generation logic.
