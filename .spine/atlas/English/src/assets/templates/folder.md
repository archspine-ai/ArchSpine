The `src/assets/templates` directory serves as the core template repository for the ArchSpine mirror system. It provides a standardized and extensible framework for documenting components, generating structured architecture summaries, enforcing architectural and coding standards, and analyzing the public surface and risk profile of the project. These templates ensure consistency and completeness across all architectural artifacts for both human readers and AI agents.

The directory is organized into four main submodules:

- **`atlas/`** – Contains the fundamental template definitions for documenting components, documents, folders, projects, and sources. This is the backbone for all structural documentation within ArchSpine.
- **`prompts/`** – Houses the prompt template engine and schema definitions used to generate structured architecture summaries. This submodule is critical for automated documentation synthesis.
- **`rules/`** – Defines the architectural and coding standards for the ArchSpine project. These rules enforce consistency across the codebase and documentation.
- **`view/`** – Contains analysis and documentation files that describe the public surface and risk profile of the ArchSpine project. This submodule is essential for security and API governance.

The most important implementation areas are the prompt engine (for AI-driven summary generation), the atlas templates (for consistent component documentation), and the rules definitions (for enforcing standards). The view submodule is key for assessing the project's external exposure and risk.
