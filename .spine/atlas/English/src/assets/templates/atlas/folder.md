This directory houses the foundational documentation and templates that define the ArchSpine mirror system—a mechanism for creating and maintaining a parallel `.spine` directory of architecture documentation aligned with source code.  The children are grouped into three categories: (1) primary project overview files (`config.md` and `document.md`) that state the system’s purpose, configuration parameters, and vision; (2) template definitions (`folder.md` and `source.md`) that establish structured architectural contracts for declaring a component’s role, responsibilities, invariants, and out‑of‑scope boundaries; and (3) a system‑vision document (`project.md`) that explains how modules orchestrate to keep multi‑language documentation in sync via a central spine and language‑specific atlases.  The most important implementation areas are mirroring configuration, drift detection, annotation templates, multi‑language atlas management, and architectural contract governance.
