# `src/assets` — Template Definitions & Documentation Framework

The `src/assets` directory serves as the core knowledge foundation for the ArchSpine mirror system. It contains the authoritative template definitions and documentation standards that govern how architectural artifacts are created, validated, and consumed.

## Key Child Directory

- **`templates/`** — This is the primary submodule. It houses reusable template files that define the structure for:
  - Component documentation
  - Structured architecture summaries (for both humans and AI agents)
  - Architectural and coding standards enforcement
  - Public surface and risk analysis reports

Together, these templates ensure that every artifact generated by ArchSpine is consistent, complete, and machine-readable. The directory is the single source of truth for ArchSpine’s documentation conventions and is referenced by all later pipeline stages.
