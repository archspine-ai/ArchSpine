The `src/types/protocol` directory houses the core data contracts and configuration schema for the ArchSpine mirror system. It defines the foundational type definitions, configuration interfaces, and versioning constants that form the data model and public API contract for the entire infrastructure layer. This directory enables consistent representation of mirrored code units, dependency graphs, language metadata, synchronization manifests, and rule documents.

The children are grouped into several key areas:
- **Core data model**: `index-documents.ts` provides all interfaces for the complete mirror data structure, including unit, identity, semantic, skeleton, graph, provenance, folder, and project representations.
- **Configuration schema**: `config.ts` defines the central configuration interface for the system.
- **Public API facade**: `index.ts` aggregates and re-exports all infra-layer modules as a barrel export.
- **Language support**: `languages.ts` defines contracts for language metadata.
- **Synchronization and manifest**: `manifest.ts` defines shared data transfer object (DTO) interfaces for the sync and manifest system.
- **Rule engine**: `rules.ts` defines the canonical data structure for rule documents.
- **Versioning**: `versions.ts` centralizes schema and package version constants.

Concrete submodules to note are `index-documents.ts` (the most extensive type definitions), `config.ts` (the configuration hub), and `manifest.ts` (critical for synchronization). Collectively, these modules establish the contract between different parts of the ArchSpine infrastructure, ensuring consistency across mirrored code representations.
