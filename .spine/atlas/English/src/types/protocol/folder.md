This directory defines the core data structures and type contracts for the ArchSpine mirror system. Its responsibilities are grouped into several focused submodules:

- **Configuration schema** (`config.ts`): declares the `SpineConfig` interface, which specifies schema version, project metadata (name, locales), and optional LLM integration settings (provider, model, baseURL, mode). This module also exports the `MCPContextMode` type.
- **Complete mirror data model** (`index-documents.ts`): the backbone of the system, defining all major interfaces for representing mirrored code units: `SpineUnit`, `SpineIdentity`, `SpineSemantic`, `SpineSkeleton`, `SkeletonImport`, `SkeletonExport`, `DeclaredSymbol`, `StructuralHints`, `ArchSpine`, `FileDependencyEdge`, `SymbolDependencyEdge`, `EdgeProvenance`, `SpineProvenance`, `SpineFolderUnit`, `FolderChild`, `SpineProjectUnit`, `ProjectModule`, as well as rule checking results (`RuleViolation`, `Invariant`), change tracking (`ChangeIntent`), and public surface documentation (`PublicSurfaceEntry`). It also enumerates the `SourceLanguage`, `FileKind`, `SymbolKind`, `DependencyRelation`, `SymbolRelation`, and `PipelineStage` union types.
- **Language support metadata** (`languages.ts`): provides contracts for `LanguageSnapshot` (full detected‑language state), `LanguageSupport` (per‑language availability and metadata), and `LanguageDelta` (changes between states).
- **Synchronization and manifest DTOs** (`manifest.ts`): defines interfaces for synchronization state (`SyncBlock`), localized documentation references (`DocRef`), file status (`FileStatus`), and the root project manifest (`SpineManifest`).
- **Rule document schema** (`rules.ts`): specifies the canonical structure for rule definitions, including `schemaVersion`, `ruleId`, `title`, `summary`, `appliesTo`, `severity`, `enforceable`, `rationale`, and `bodyMarkdown`.
- **Version definitions** (`versions.ts`): centralizes schema and package versioning with the `SchemaVersion` type alias, constants for current versions, a generator version string, and a type guard for config schema version compatibility.
- **Public API facade** (`index.ts`): barrel export that re‑exports all of the above, providing a single consumption point for other subsystems.

The most critical implementation areas are the core data model (`index-documents.ts`), which defines the structural foundation for all mirroring operations; the configuration schema (`config.ts`), which ensures system‑wide consistency; and the versioning module (`versions.ts`), which enforces schema evolution across the entire ArchSpine pipeline.
