This directory defines the core data contracts and configuration schema for the ArchSpine mirror system. Its children are grouped by responsibility: `config.ts` provides the central configuration interface; `index-documents.ts` defines the complete data model for mirrored units, identity, semantic, skeleton, graph, provenance, folder, and project representations; `index.ts` serves as the public barrel export; `languages.ts` establishes language metadata contracts; `manifest.ts` provides DTOs for synchronization manifests; `rules.ts` defines the canonical structure for rule documents; and `versions.ts` centralizes schema and package version constants. The most critical implementation areas are the core data model interfaces (unit, dependency graph, provenance), the configuration schema, versioning constants, language support metadata, and the manifest and rule document contracts—all of which form the foundational API contract for the entire infrastructure layer.
