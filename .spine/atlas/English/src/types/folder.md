# `src/types` — Core Data Contracts & Public API Types

This directory defines the foundational type system for the ArchSpine mirror infrastructure. It establishes the data model, configuration schema, versioning constants, and view artifact contracts that govern how mirrored code units, dependency graphs, language metadata, synchronization manifests, rule documents, and architecture diagrams are represented across the entire system.

## Notable Children

- **`src/types/protocol/`** — A subfolder containing the core data contracts and configuration schema for the mirror protocol. This is where the primary type definitions for code units, dependency graphs, language metadata, sync manifests, and rule documents reside.
- **`src/types/protocol.ts`** — A public facade module that re-exports all protocol types from the `protocol/` subfolder, providing a single stable entry point for consumers.
- **`src/types/view.ts`** — Defines the contract for view artifacts generated by the ArchSpine view generation system, including architecture diagram specifications and other visual representations.

## Key Implementation Areas

- **Data Contracts** — The `protocol/` subfolder contains the most critical type definitions, including interfaces for code units, dependency graphs, language metadata, synchronization manifests, and rule documents.
- **Configuration Schema** — Type definitions for system configuration, ensuring consistent validation and serialization across the infrastructure layer.
- **Versioning Constants** — Shared version identifiers that coordinate compatibility between different ArchSpine components.
- **View Artifact Schema** — The `view.ts` module defines how generated diagrams and visual outputs are structured, enabling downstream rendering tools to consume them reliably.
