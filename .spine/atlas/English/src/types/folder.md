The `src/types` directory forms the foundational data model and public API contract for the ArchSpine mirror system's protocol layer. It contains three primary components: a `protocol` subfolder that defines core data contracts and configuration schemas; a `protocol.ts` facade module that provides a stable, single entry point for importing all protocol types; and a `view.ts` module that establishes type definitions for generated view artifacts. Together, these elements define configuration interfaces, versioning constants, language metadata, dependency graph representations, synchronization manifests, rule documents, and view schemas that ensure consistent representation of mirrored code units across the entire system. The implementation areas of greatest importance are the protocol contracts (in the `protocol` subfolder) and the view artifact schema (in `view.ts`), as they govern interoperability and output generation.
