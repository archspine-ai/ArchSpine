This directory serves as the public API surface for LLM provider mocking infrastructure within the ArchSpine system. Its primary responsibility is to expose the `MockClient` class as a stable, importable export from the internal LLM provider mocking implementation. This allows consumers—such as test suites and development environments—to mock LLM providers without depending on internal implementation details.

The implementation areas that matter most here are the stability of the `MockClient` interface and the simplicity of the re-export pattern. By keeping the public surface minimal, the directory reduces coupling and makes it easy for consumers to adopt mocking without needing to understand the internal provider hierarchy.
