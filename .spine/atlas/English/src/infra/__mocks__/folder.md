This directory serves as the public facade for the LLM provider mocking infrastructure within the ArchSpine mirror system. It provides a single, stable export point—the `MockClient`—which is re-exported from the internal `../llm/providers/mock.js` module. This design decouples test and development consumers from the internal provider hierarchy.

The notable file `llm.ts` is the sole public entry point, grouping re-export logic and acting as the module’s export surface. The underlying implementation lives in the `llm/providers/` subdirectory, which contains concrete mock providers. Key implementation concerns include maintaining API contract compatibility with real LLM providers (e.g., OpenAI, Anthropic) and ensuring the mock supports deterministic responses for test scenarios. Consumers should rely on this facade rather than importing directly from internal paths to guarantee forward compatibility.
