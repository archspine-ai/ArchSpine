This directory serves as the public facade for the LLM mocking infrastructure. Its primary purpose is to expose the `MockClient` class as a stable, decoupled export so that consumers can use it without depending on internal modules. The only file, `llm.ts`, is a re-export module that pulls `MockClient` from the internal `../llm/providers/mock.js` path. This design ensures that internal mock implementations can evolve without breaking external test or development code. The key implementation area is the decoupling layer – this facade manages the boundary between internal provider mocks and external usage.
