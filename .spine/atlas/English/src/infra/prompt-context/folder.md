The `src/infra/prompt-context` directory holds the central orchestration and configuration modules of the ArchSpine prompt policy system. It is responsible for the entire lifecycle of prompt generation: resolving the appropriate prompt policies based on task mode, calculating token budgets, trimming content to fit constraints, generating diagnostic information, and enforcing type-safe configuration parsing. This directory is organized around a few key clusters: the core orchestration layer (`artifacts.ts`), budget and policy utilities (`budgets.ts`, `policy.ts`), configuration and type definitions (`constants.ts`, `types.ts`), and supporting utilities (`diagnostics.ts`, `parsers.ts`, `trim.ts`). The most important implementation areas are the prompt assembly pipeline managed by `artifacts.ts`, which coordinates all other modules; the budget calculations in `budgets.ts` that dynamically allocate token limits based on validation policies and task modes; the diagnostic analysis in `diagnostics.ts` that tracks rule block retention; and the policy resolution in `policy.ts` that provides default behaviors. Concrete submodules like `constants.ts` define typed budget profiles and configuration constants, while `types.ts` centralizes all TypeScript definitions for prompt generation, validation, and diagnostic configuration. `parsers.ts` handles the safe parsing of `PromptPolicyTier` values from raw strings, and `trim.ts` provides text trimming functions essential for fitting prompts within token budgets.
