The `src/infra/prompt-context` directory serves as the central nervous system of ArchSpine's prompt policy orchestration. It is responsible for the end-to-end lifecycle of prompt generation: resolving policies based on task mode, calculating token budgets, trimming excess content, generating diagnostics, and parsing configuration in a type-safe manner. Notable submodules include the orchestration service in `artifacts.ts` which coordinates all these steps, the token budget calculator in `budgets.ts`, the typed constant definitions in `constants.ts`, the diagnostic analyzer in `diagnostics.ts`, the policy resolver in `policy.ts`, the text trimming utilities in `trim.ts`, the configuration parser in `parsers.ts`, and the comprehensive type definitions in `types.ts`. The most impactful implementation areas are policy resolution (how default and validation policies are selected by mode), budget calculation (ensuring prompt size stays within limits), and content trimming (optimizing relevance through rule block filtering). Together these modules ensure consistent, efficient, and rule-aware prompt assembly for the LLM orchestration system.
