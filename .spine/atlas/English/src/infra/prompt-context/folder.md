# ArchSpine Prompt Context Directory

The `src/infra/prompt-context` directory is the central orchestration hub for the ArchSpine prompt policy system. It manages the complete lifecycle of prompt generation, from policy resolution and budget calculation to content trimming and diagnostic analysis.

## Key Components

- **`artifacts.ts`** – The core orchestration service that assembles source prompt artifacts by coordinating policy resolution, budget calculation, content trimming, and diagnostic generation.
- **`budgets.ts`** – Calculates prompt token budgets based on task mode, validation policy, and source code artifacts.
- **`constants.ts`** – Defines typed constants and budget profiles for the entire prompt policy system.
- **`diagnostics.ts`** – Analyzes rule block retention and filtering during prompt relevance processing.
- **`parsers.ts`** – Parses and validates the `PromptPolicyTier` configuration value from raw string input.
- **`policy.ts`** – Resolves default prompt and validation policies based on the current task mode.
- **`trim.ts`** – Provides text trimming and formatting utilities for prompt assembly.
- **`types.ts`** – Central TypeScript type definitions for prompt generation, validation, diagnostic configuration, and generation strategy.

## Implementation Focus

The most critical areas are the orchestration service (`artifacts.ts`), which ties together all other modules, and the type definitions (`types.ts`), which ensure type-safe configuration throughout the system. The budget calculation (`budgets.ts`) and policy resolution (`policy.ts`) modules are also essential for maintaining consistent and efficient prompt assembly.
