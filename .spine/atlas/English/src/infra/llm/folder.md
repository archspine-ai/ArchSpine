The `src/infra/llm` directory is the foundational LLM infrastructure layer for the ArchSpine mirror system. It abstracts all provider interactions through clean interfaces, centralizes configuration management, and ensures resilience with exponential backoff retry logic.  

The notable children are grouped into several areas:  
- **Interface definitions** (`base.ts`) establish contracts for clients, responses, provider configs, and context tracking.  
- **Factory and runtime** (`factory.ts`, `runtime.ts`) handle provider selection and client instantiation based on configuration.  
- **Global configuration** (`global.ts`) manages file I/O and credential integration.  
- **Retry utility** (`retry.ts`) provides configurable exponential backoff for async operations.  
- **Provider adapters** (`providers/` folder) contain concrete client implementations for generating structured semantic summaries.  

Key implementation areas that matter most include the abstraction layer that decouples the rest of the system from specific LLM providers, the robust retry mechanism that enhances reliability, and the centralized configuration flow that keeps secrets and settings manageable.
