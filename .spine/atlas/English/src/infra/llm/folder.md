# `src/infra/llm` — Core LLM Infrastructure

This directory is the backbone of ArchSpine's semantic generation layer. It provides provider-agnostic client abstractions, configuration management, and resilient execution patterns for all LLM interactions.

## Key Components

- **`base.ts`** — Defines TypeScript interfaces for LLM client abstractions, response structures, provider configuration, and semantic context tracking. This is the contract that all providers must implement.
- **`factory.ts`** — A facade factory that instantiates LLM provider clients based on configuration, decoupling provider selection from client usage.
- **`global.ts`** — Manages global LLM configuration file I/O and credential store integration, handling persistent settings and secrets.
- **`runtime.ts`** — Resolves runtime configuration and acts as a provider client factory within the LLM subsystem, bridging configuration to instantiation.
- **`retry.ts`** — Provides configurable retry logic with exponential backoff for resilient asynchronous operations, ensuring robustness against transient failures.
- **`providers/`** — Contains provider-specific implementations and utilities for generating semantic summaries. This subfolder is where concrete provider adapters live (e.g., OpenAI, Anthropic, etc.).

## Implementation Focus

The most critical areas are the interface contracts in `base.ts` (which define the system's LLM abstraction boundary) and the retry utility in `retry.ts` (which ensures operational resilience). The `providers/` subfolder is where new provider integrations are added, making it the primary extension point for the LLM subsystem.
