The `credential` directory implements a pluggable, secure secret storage system with platform-specific backends and scoped store factories. It is organized into two core modules: `backend.ts` provides low-level secure persistence via an extensible `CredentialBackend` interface, with concrete implementations for macOS Keychain, Linux secret-tool, Windows DPAPI, and an in-memory fallback; `store.ts` offers high-level factory functions (`createProjectLLMCredentialStore`, `createGlobalLLMCredentialStore`) that use these backends to manage LLM API keys and other credentials. Key implementation areas include platform-aware backend selection, deterministic secret naming derived from file paths, credential validation and trimming, and scoped access control for project-specific vs. global stores.
