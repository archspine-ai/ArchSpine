The `src/infra/config` directory serves as the centralized configuration management layer for the ArchSpine mirror system. It is responsible for loading, validating, merging, and providing runtime access to all system settings, including LLM configuration, scan policies, pre-commit hooks, and artifact strategies.

The directory contains several tightly related submodules that work together:

- **Type definitions and defaults** – `types.ts` defines configuration TypeScript interfaces and enums; `defaults.ts` provides a factory that returns a default `SpineConfig` object.
- **Loading and validation** – `loader.ts` reads, parses, validates, and merges configuration files with defaults; `validation.ts` acts as a facade for validating configuration payloads.
- **Environment and runtime access** – `env.ts` offers environment variable parsing utilities; `facade.ts` is the central configuration manager that exposes and mutates the runtime settings.
- **Pre‑commit resolution** – `precommit.ts` resolves the boolean pre‑commit setting from environment variables or explicit parameters.
- **Supported values contract** – `supported-values.ts` defines the interface for accessing supported configuration values.

Key implementation areas include the configuration lifecycle (load ➜ validate ➜ merge ➜ access), integration with environment variables, and the separation of concerns between type contracts, defaults, and active configuration state.
