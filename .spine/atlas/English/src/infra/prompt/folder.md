The `src/infra/prompt` directory is the nerve center of prompt construction and assembly for the ArchSpine system. It provides a comprehensive toolkit for building structured, localized prompts that AI agents can consume. The directory groups its children into several logical areas: core builder and factory modules (`aggregate.ts`, `builder.ts`), localization and output formatting utilities (`markdown.ts`, `shared.ts`), a source‑code analysis specialist (`source.ts`), and the foundational type definitions (`types.ts`).

Concrete submodules include:
- **aggregate.ts**: A factory that orchestrates the `PromptBuilder` pipeline to produce prompts for documents, configuration files, directories, and entire project units.
- **builder.ts**: The central assembly class that coordinates the rendering of structured prompt blocks for agent interactions.
- **markdown.ts**: A utility that converts semantic JSON input into localized, markdown‑only prompt strings, essential for human‑readable outputs.
- **shared.ts**: Generates formatted output contract strings that define the expected response structure.
- **source.ts**: A facade that produces tailored LLM prompts for source‑code analysis tasks.
- **types.ts**: Defines the TypeScript types underpinning the entire prompt generation and response formatting protocol.

The most critical implementation areas are prompt construction (how different input sources are transformed into structured instructions), localization support (enabling multilingual agent communication), and response contract generation (ensuring AI outputs adhere to a predictable schema). Together, these modules form a robust infrastructure for all ArchSpine prompt workflows.
