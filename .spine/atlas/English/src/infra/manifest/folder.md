# ArchSpine `src/infra/manifest` — Infrastructure Layer for Manifest & Integrity

This directory provides the core persistence, verification, and tracking mechanisms for the ArchSpine mirror system. It is responsible for file hashing, manifest state management, language snapshot I/O, and file status drift tracking.

The module is composed of five tightly coupled submodules:

- **`facade.ts`** – The top-level entry point that exposes unified methods for manifest persistence, file status queries, and drift history management.
- **`integrity.ts`** – Implements SHA-256 file hashing with metadata validation and database integration, ensuring every mirrored file’s integrity can be verified.
- **`io.ts`** – Provides low-level file path resolution, JSON reading/writing, and snapshot creation for both the manifest and language‑specific snapshots.
- **`state.ts`** – Acts as an internal facade for manifest file persistence, reverse‑index state tracking, and language snapshot I/O operations.
- **`types.ts`** – Defines the TypeScript interface that structures file metadata snapshots, used across all other modules.

Key implementation areas include hashing logic, state persistence, and the integration between manifest and language snapshots. The `facade.ts` and `state.ts` modules form the primary API for higher layers, while `integrity.ts` and `io.ts` handle the heavy lifting of actual file operations and verification.
