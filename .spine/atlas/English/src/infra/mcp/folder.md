The `mcp` directory serves as the infrastructure for Model Context Protocol (MCP) integration, enabling external AI agents to interact with ArchSpine's internal resources and tools. It contains four main submodules: `context.ts` (MCPContextGate for context-aware access control), `resources.ts` (resource templates and handlers for project metadata, files, and index), `server.ts` (MCP server setup with stdio transport and request handling), and `tools.ts` (exposure of core ArchSpine tools like queries, scanning, and drift history). Key implementation areas include context gating based on priming state, resource URI normalization and filesystem resolution, server initialization with error wrapping, and tool execution with audit tracing.
