**`src/infra/mcp` – Model Context Protocol Infrastructure**

This directory implements the Model Context Protocol (MCP) infrastructure that allows external AI agents to interact with ArchSpine’s internal project data and capabilities. All communication happens over **stdio transport**, and access is gated by a context‑aware control layer.

**Key files and their grouping:**

- **`context.ts`** – Defines the `MCPContextGate` facade, which controls MCP context flow based on the system’s priming state and operational mode. It acts as the first gate for all incoming requests.
- **`resources.ts`** – Provides MCP resource templates and handlers for exposing project metadata and files inside the `.spine` directory. Uses `MCPContextGate` to enforce access rules before serving resources.
- **`server.ts`** – Implements the MCP server itself, wiring together resources and tools, and exposing them to external agents via stdio.
- **`tools.ts`** – A tool facade that wraps ArchSpine system capabilities as queryable MCP tools, allowing agents to perform operations like reading or mutating project state.

Together, these files form a cohesive layer that (1) secures access based on system context, (2) exposes resources and tools in a standardised MCP format, and (3) communicates over stdio for lightweight agent integration. The most critical implementation areas are **context gating** and **resource/tool definition**.
