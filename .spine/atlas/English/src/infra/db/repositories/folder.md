The `src/infra/db/repositories` directory serves as the persistence backbone of the ArchSpine indexing system. It houses five SQLite-based Data Access Objects (DAOs), each dedicated to a core entity of the architecture: file metadata (`files.ts`), symbol tables (`symbols.ts`), semantic drift audit events (`drift.ts`), token usage metrics (`usage.ts`), and architectural violation records (`violations.ts`). This unified data access layer ensures reliable CRUD operations for all system state, enabling efficient indexing and retrieval while maintaining referential integrity. The most critical implementation areas are the symbol table operations (foundational for cross‑reference resolution) and the drift event repository (which drives the semantic change tracking pipeline). The repo grouping is flat by entity type, with each module encapsulating all SQLite queries and schema interactions for its domain.
