The `src/infra/db` directory provides the complete SQLite persistence layer for the ArchSpine indexing system. It is responsible for database lifecycle management—filesystem preparation, WAL recovery, schema evolution, and error handling—while exposing a unified data access API for core entities.

Notable children are grouped by concern:

- **Lifecycle and recovery**: `runtime.ts` orchestrates low-level database open/close and stale WAL detection; `wal-recovery.ts` handles detection and cleanup of orphaned WAL files.
- **Schema management**: `schema.ts` provides safe migration utilities (e.g., column addition) and error handling for DDL operations.
- **Data access**: The `repositories/` folder contains SQLite-based DAOs for file metadata, symbol tables, drift events, usage metrics, and violation records.
- **Atomic operations**: `batch.ts` implements a commit wrapper for synchronizing file metadata in a single transaction.
- **Error mapping**: `errors.ts` translates SQLite errors into domain-level exceptions.
- **Type contracts**: `types.ts` defines stable TypeScript interfaces used across indexing, audit, and status reporting domains.

The most implementation-critical areas are the runtime lifecycle management (`runtime.ts` and `wal-recovery.ts`) and the repository layer inside `repositories/`, which directly supports all CRUD flows of the system.
