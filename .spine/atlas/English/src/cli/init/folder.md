The `src/cli/init` directory forms the initialization and bootstrapping subsystem of ArchSpine. It provides the core logic for setting up a new ArchSpine-managed repository, including configuration bootstrapping, runtime initialization, LLM credential configuration, file scanning, language discovery, and git hook installation. The subsystem is coordinated through shared type contracts defined in `types.ts`. The two main operational modules are `repository-bootstrap.ts`, which acts as the CLI command adapter for bootstrapping configuration and artifacts into a target repository, and `runtime-bootstrap.ts`, an orchestrator that initializes the ArchSpine runtime and triggers the build pipeline. Together, these components ensure a seamless initial setup experience.
