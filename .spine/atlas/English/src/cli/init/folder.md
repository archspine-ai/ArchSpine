The `init` directory houses the bootstrapping subsystem for ArchSpine, responsible for initializing both the repository and the runtime. It is organized into three key modules:

- **`repository-bootstrap.ts`** – A CLI command adapter that orchestrates repository-level setup, including artifact strategy selection (index, atlas, hybrid), rule template installation, and Git hook configuration for pre-commit integrity checks. It handles user prompts and fallback logic for critical steps.
- **`runtime-bootstrap.ts`** – A service module that drives the runtime bootstrap sequence: LLM credential configuration and scope confirmation, file system scanning, language discovery, manifest update, and handoff to the build workflow. It also supports an optional initial sync prompt.
- **`types.ts`** – Provides shared type contracts (`InitSharedOptions`, `RepositoryBootstrapResult`) and type aliases (`LLMScope`, `HookSetupStatus`, `ArtifactStrategy`) that standardize data structures across the CLI and service layers.

The most significant implementation concerns are artifact strategy configuration, secure LLM credential setup, and the coordination of scanning, language detection, and manifest updates in a reliable, interactive sequence.
