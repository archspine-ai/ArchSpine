The `init/` directory is the central bootstrap subsystem for ArchSpine, responsible for initializing both the repository configuration and the runtime environment. It orchestrates the complete setup sequence—from artifact strategy selection and rule template installation to git hook configuration, LLM credential setup, file system scanning, language discovery, manifest updates, and final handoff to the build workflow.

The directory groups its logic into three concrete submodules:

- **repository-bootstrap.ts** — A CLI command adapter that guides the user through repository initialisation. It prompts for an artifact strategy (index, atlas, or hybrid), installs recommended rule templates via the repository-admin-service, configures git hooks for pre-commit integrity checks, and manages cancellation/fallback logic.
- **runtime-bootstrap.ts** — A service module that runs the runtime bootstrap sequence after repository setup. It coordinates LLM credential setup and scope confirmation, triggers the Scanner engine for file system scans, performs language discovery to detect programming languages, updates the system manifest with the discovered language snapshot, and calls the build workflow command. It also provides a user-driven initial sync prompt.
- **types.ts** — A shared type contract module that defines interfaces (`InitSharedOptions`, `RepositoryBootstrapResult`) and type aliases (`LLMScope`, `HookSetupStatus`, `ArtifactStrategy`) to ensure consistent data structures across CLI commands and service layers during the entire setup process.

The most important implementation areas are the separation of repository configuration from runtime bootstrapping, the use of shared type contracts for consistency, and the orchestration of async steps that involve user prompts, file system operations, and service coordination.
