# `src/cli/commands` — CLI Command Adapters

This directory contains the user-facing command-line interface layer of ArchSpine. Each file is a thin adapter that parses arguments, delegates to core services (RuntimeService, Config, Manifest, Scanner, etc.), handles errors consistently, and formats terminal output.

## Notable Children

- **`build.ts`** — Orchestrates the build workflow by delegating to core services.
- **`check.ts`** — Entry point for the check operation, using CheckService and structured error signaling.
- **`config.ts`** — Manages configuration settings.
- **`fix.ts`** — Thin interface to the runtime service's fix subsystem.
- **`god.ts`** — Handles God Mode with user warnings and confirmation before delegation.
- **`history.ts`** — Parses arguments, retrieves drift history and file documentation from Manifest, and formats output.
- **`hook.ts`** — Manages git hook integration and synchronization.
- **`info.ts`** — Orchestrates the info report engine.
- **`init.ts`** — Initializes the ArchSpine environment, configuration, and repository structure.
- **`languages.ts`** — Interactive configuration of documentation languages.
- **`llm.ts`** — Interactive configuration of LLM providers, models, and runtime.
- **`mcp.ts`** — Starts the Model Context Protocol (MCP) server.
- **`publish.ts`** — Orchestrates the publish workflow including preflight checks, sync, document backfill, and atlas state management.
- **`remove.ts`** — Cleans up ArchSpine-managed artifacts from a repository.
- **`repo.ts`** — Router for repository-level operations.
- **`scan.ts`** — Handles source code scanning and dry-run reporting.
- **`status.ts`** — Displays synchronization status of the mirror system.
- **`sync.ts`** — Orchestrates the synchronization workflow.
- **`try.ts`** — Preview command with filesystem validation and configuration parsing.
- **`usage.ts`** — Entry point for executing usage reports.
- **`view.ts`** — Handles view selection, validation, and protected output writes.

## Key Implementation Areas

- **Argument Parsing & Delegation** — Each adapter parses CLI arguments and delegates to the appropriate core service.
- **Error Handling** — Consistent error handling and structured error signaling across all commands.
- **Output Formatting** — Terminal-friendly output formatting for all subcommands.
- **Workflow Orchestration** — Commands like `build`, `publish`, and `sync` orchestrate multi-step workflows involving several core services.
