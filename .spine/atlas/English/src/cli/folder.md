The `src/cli` directory serves as the entry point and command-line interface layer for the ArchSpine system. It is responsible for handling user commands, parsing arguments, rendering help information, and performing runtime bootstrapping. The contents are logically grouped into utility modules and three subfolders: `commands`, `init`, and `repo`. Key source files include `index.ts` (primary router and entry point), `cli-utils.ts` (UI formatting utilities for language discovery results and banners), `document-languages.ts` (configuration types for multilingual documentation tiers), and `help.ts` (help text rendering). The `commands` subfolder contains specific command adapters that handle argument parsing and delegation to core services. The `init` subfolder houses the bootstrapping subsystem for initializing an ArchSpine project. The `repo` subfolder provides a CLI adapter for managing repository artifact strategies. Together, these components form the complete command-line interface through which users interact with ArchSpine's semantic mirror workflows.
