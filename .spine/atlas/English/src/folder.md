The `src` directory is the root source code location for the ArchSpine mirror system. It houses the entire implementation needed to generate and manage semantic mirrors of codebases. The directory is organized into ten submodules, each with a distinct responsibility:  

- **assets** – Core template definitions and documentation standards for mirror generation.  
- **ast** – AST parsing and language discovery, enabling structural analysis of source files.  
- **cli** – Command-line interface entry point, handling argument parsing, help display, and runtime bootstrapping.  
- **core** – L2 aggregation logic, the central engine for combining lower-level data into coherent mirror structures.  
- **engines** – File scanning, aggregation, architectural rule enforcement, dependency analysis, and system diagnostics.  
- **infra** – Infrastructure services: configuration, persistence, LLM abstraction, prompt engineering, and security.  
- **services** – Orchestration layer for runtime command pipelines (check, fix, sync, view), with session management and LLM integration.  
- **tasks** – Pipeline tasks and stages that drive automated workflows.  
- **types** – Protocol-layer type definitions and configuration contracts.  
- **utils** – Foundational utilities for file synchronization, path normalization, locking, and CLI presentation.  

Key implementation areas include the AST-based parsing pipeline, the core aggregation engine, the CLI tooling, and the infrastructure layer supporting LLM interactions. Together, these modules form a complete system for creating and maintaining reflective mirrors of software projects.
