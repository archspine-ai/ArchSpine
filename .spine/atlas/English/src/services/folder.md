The `src/services/` directory is the central orchestration layer of ArchSpine, where all runtime command pipelines—check, fix, sync, and view generation—are coordinated. It combines scanning, AST extraction, validation, LLM correction, reconciliation, and summarization into cohesive workflows, all managed within resumable sessions protected by checkpoint and lock mechanisms.

Notable children include:

- **check-service.ts**, **fix-service.ts**, and **sync-service.ts** – the primary orchestration facades for their respective pipelines, each responsible for chaining the necessary domain tasks (scanning, AST, validation, usage recording, retry logic, reconciliation, commit) with consistent error handling and metrics.

- **view-service.ts** and the **view/** subfolder – view generation and rendering for architecture diagrams, risk hotspots, and public surface reports, powered by the same session and orchestration infrastructure.

- **runtime-service.ts** and **runtime-service.types.ts** – resolve execution profiles, LLM settings, and view configurations; they construct and wire domain service instances (Check, Fix, Sync) on demand.

- **runtime-session.ts** – manages resumable command execution sessions with checkpoint validation and output mutation detection.

- **task-runtime.ts** – a factory that prepares a fully configured task context for analysis pipelines, integrating scanning and policy enforcement.

- **repository-admin-service.ts** and **publish-preflight.ts** – administrative services for repository initialization and publish pre‑flight checks (baseline, snapshot readiness, local strategy warnings).

- **llm-admin-service.ts** – provides LLM configuration type definitions for the command and view layers.

Implementation areas of greatest importance are the check/fix/sync orchestration flows (each a distinct multi‑stage pipeline), the session‑based resumability and lock protection, and the view generation that turns indexed data into actionable architectural visualisations.
