The `src/services` directory is the central orchestration layer of the ArchSpine mirror system. It coordinates multi-stage pipelines for analysis, correction, synchronization, and visualization, while managing runtime sessions and execution profiles to ensure resumability and consistency.

Key submodules include:

- **check-service.ts** – Orchestrates the 'check' command pipeline (scanning, AST extraction, validation, usage recording).
- **fix-service.ts** – Drives the architectural fix pipeline with LLM correction, retry support, and session integration.
- **sync-service.ts** – Coordinates the full synchronization pipeline across multiple stages.
- **runtime-service.ts** – Resolves execution profiles, LLM settings, and view configurations, then constructs domain service instances for Check, Fix, and Sync.
- **runtime-session.ts** – Manages resumable command sessions with checkpoint validation and output mutation detection.
- **task-runtime.ts** – Factory that prepares a fully configured task context for analysis pipelines.
- **view-service.ts** – Generates architectural views from indexed codebase data.
- **view/** – Subfolder dedicated to view layer components and rendering logic.
- **publish-preflight.ts** – Validates runtime conditions before publish operations.
- **repository-admin-service.ts** – Handles repository artifact management and initialization strategies.
- **llm-admin-service.ts**, **runtime-execution-profile.ts**, **runtime-service.types.ts** – Supporting type definitions and configuration structures.

The most critical implementation areas are pipeline orchestration (check, fix, sync), session management for resumability, LLM integration, and architectural view generation.
