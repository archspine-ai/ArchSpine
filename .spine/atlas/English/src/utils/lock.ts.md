# ArchSpine Lock Utility

## Role
Infrastructure utility providing file-based mutual exclusion for runtime operations across the ArchSpine system.

## Key Responsibilities
- Creates and manages lock files within the `.spine` directory to enforce exclusive access and prevent race conditions.
- Generates unique lock tokens using `crypto.randomUUID` for ownership identification and validation.
- Detects and handles stale locks by checking process liveness and cleaning up dead processes to ensure system robustness.
- Registers cleanup handlers for process termination signals to guarantee lock release.
- Provides serialization and deserialization utilities for lock payload data via `parseLockPayload` and `serializeLockPayload` functions.

## Notable Invariants
- Lock files are always stored under the `.spine` directory relative to the `rootDir`.
- Each lock token is a unique UUID generated by `crypto.randomUUID`.
- Lock release is guaranteed via process signal handlers (`SIGINT`, `SIGTERM`, `exit`).

## Negative Scope (Out of Scope)
- Inter-process communication beyond file-based locking.
- Distributed lock management across multiple machines.
- Database-level or network-based locking mechanisms.

## Public Surface
- `LockManager` class (constructor, `acquire`, `release`, `isHeld` methods)
- `parseLockPayload(content: string): RuntimeLockPayload`
- `serializeLockPayload(payload: RuntimeLockPayload): string`
- `RuntimeLockPayload` interface

## Architectural Intent
Provide a robust, file-based mutual exclusion mechanism to prevent concurrent access conflicts in the ArchSpine runtime environment.
