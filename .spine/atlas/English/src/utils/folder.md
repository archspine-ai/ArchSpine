# `src/utils` — ArchSpine Core Utilities

This directory provides the foundational infrastructure and utility modules that power the ArchSpine mirror system. It is organized into three main groups:

1. **Agent Instruction Management** — Three files (`agent-instructions.sync.ts`, `agent-instructions.templates.ts`, `agent-instructions.ts`) handle the synchronization, templating, and public API for managing agent instruction blocks across repository configuration files. The `.sync.ts` file performs file-level sync and removal operations, `.templates.ts` stores static instruction blocks and ignore patterns, and `.ts` serves as the barrel export.

2. **Infrastructure & File Operations** — `fs.ts` provides atomic file writes and safe copies, `lock.ts` implements file-based mutual exclusion for runtime safety, and `repo-path.ts` normalizes repository paths for consistent database key representation across platforms.

3. **CLI & Presentation** — `banner.ts` renders branded banners and version info, `confirm.ts` handles interactive user prompts, and `footprint.ts` generates deterministic structural fingerprints of source files. `git-hook.ts` manages the pre-commit Git hook shell script block.

The most critical implementation areas are the atomic file operations in `fs.ts`, the locking mechanism in `lock.ts`, and the agent instruction synchronization logic in `agent-instructions.sync.ts`, as these directly affect data integrity and system reliability.
