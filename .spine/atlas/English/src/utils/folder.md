The `src/utils` directory supplies core infrastructure and utility modules that drive the ArchSpine system's file synchronization, CLI interactivity, and runtime coordination. These modules are organized into three key groups:

- **Agent instruction utilities** – Three files (`agent-instructions.sync.ts`, `agent-instructions.templates.ts`, `agent-instructions.ts`) handle lifecycle operations for agent instruction blocks: synchronizing instruction blocks into repository config files, supplying static templates and ignore patterns, and exposing a barrel-export API for unified access.
- **CLI presentation and interaction** – `banner.ts` renders branded startup banners and version info; `confirm.ts` implements interactive terminal prompts for user confirmation.
- **Core infrastructure services** – `fs.ts` provides atomic file writes and safe copies; `lock.ts` enforces file-based mutual exclusion for critical sections; `git-hook.ts` generates and manages the pre-commit Git hook shell script; `footprint.ts` computes deterministic structural fingerprints of source files; `repo-path.ts` normalizes repository paths cross-platform for consistent database key representation.

The most impactful implementation areas are atomic file safety (`fs.ts`), exclusive locking (`lock.ts`), Git hook automation (`git-hook.ts`), and structural change detection via `footprint.ts`. The agent instruction sync module (`agent-instructions.sync.ts`) is also central as it directly manages how ArchSpine configuration is injected and removed across multiple repository targets.
