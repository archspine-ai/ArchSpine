The `schemas` directory serves as the formal backbone of the ArchSpine mirror system: it contains all JSON schema definitions that validate metadata units, manifests, and governance rule documents. Together, these schemas enforce structural integrity, data validation rules, and type constraints for the entire semantic indexing and architecture enforcement system.  

The directory’s children are logically grouped into two categories:  
- **A dedicated `examples` subfolder** that holds core configuration and rule definitions used for system initialization and reference.  
- **Individual schema files** that define the structure and validation logic for each key component.  

Key implementation areas include:  
- **Metadata unit validation** – via `spine-unit.schema.json` (core file metadata) and `spine-project-unit.schema.json` (project descriptor).  
- **Structural node validation** – via `spine-folder-unit.schema.json` (directory-level role and responsibility).  
- **Manifest synchronization** – via `spine-manifest.schema.json` (tracks index state and inventory).  
- **Governance enforcement** – via `spine-rules.schema.json` (rule document constraints).  
- **Shared type definitions** – via `shared.schema.json` (reusable validation pieces across all schemas).  

These concrete submodules together enable the ArchSpine system to maintain a consistent, self‑validating mirror of a project’s architecture and metadata.
