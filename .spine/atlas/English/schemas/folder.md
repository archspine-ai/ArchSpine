This directory contains the canonical JSON Schema definitions for all data components in the ArchSpine mirror system. These schemas serve as the single source of truth for structural constraints, validation rules, and type safety across repository indexing, manifest tracking, project configuration, and rule orchestration. The schemas are organized by component: `shared.schema.json` defines reusable base types (version, timestamp, path, hash, locale, etc.); `spine-folder-unit.schema.json` validates SpineFolderUnit directory metadata; `spine-manifest.schema.json` enforces sync state and file index consistency; `spine-project-unit.schema.json` governs self-contained project configurations; `spine-rules.schema.json` ensures rule documents are syntactically and structurally correct; and `spine-unit.schema.json` manages code unit identity and metadata. The `examples/` subfolder provides sample configurations to illustrate usage patterns. Key implementation areas include data validation pipelines, schema-based code generation, and cross-component consistency enforcement.
