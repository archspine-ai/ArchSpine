The `scripts` directory houses all automation and demonstration tooling for the ArchSpine project. Its contents span the entire development lifecycle: from building and watching during active development (`build.mjs`, `dev-build.mjs`), to end‑to‑end verification of multi‑language AST extraction (`final-verify.mjs`), live benchmarking of the semantic extraction pipeline (`phase5-live-bench.mjs`), and release‑gating checks (`release-gate.mjs`). Demonstration scripts (`demo.tape`, `project-demo.tape`) produce CLI walkthroughs and visual previews. A database schema migration script (`update-db-schema.cjs`) manages the cache system’s schema. The `publish-placeholder.mjs` script covers operational placeholder publishing. Together these scripts provide a complete build, test, release, and demo automation layer, with particular implementation focus on the AST‑based extraction verification and the semantic benchmarking pipeline.
