The root directory of ArchSpine serves as the project's operational hub, housing all automation, demonstration, verification, and release utilities that drive the development lifecycle. These scripts collectively implement build compilation via `build.mjs`, continuous development watching with `dev-build.mjs`, multi-language AST extraction validation through `final-verify.mjs`, and schema and protocol consistency checks via `validate-protocol-assets.mjs`. Two demonstration scripts (`demo.tape` and `project-demo.tape`) provide terminal-based walkthroughs of typical CLI workflows, including spine initialization, sync, check, and scan commands. Release management is handled by `release-gate.mjs` (pre-publish validation) and `publish-placeholder.mjs` (npm package name reservation), while `update-db-schema.cjs` manages SQLite cache migrations. The most critical implementation areas are build reliability (ensuring TypeScript compilation, asset copying, and mock exclusion), multi-language verification (covering Java, C++, Rust, C, Python, and Go), and release integrity gates that enforce tests, schema validation, and pack checks before publication.
