The `scripts/` directory houses all automation and tooling scripts for the ArchSpine project, covering the full development lifecycle from building and testing to benchmarking and release management. Its contents fall into several key groups:

- **Build automation** – `build.mjs` orchestrates compilation, asset copying, and cleanup for distribution; `dev-build.mjs` provides a watch-mode variant for rapid iteration during development.
- **Verification and benchmarking** – `final-verify.mjs` runs an end-to-end check on the ASTExtractor’s multi-language export pipeline, while `phase5-live-bench.mjs` profiles the semantic extraction core under live conditions.
- **Release gating** – `release-gate.mjs` enforces quality checks before publication, and `publish-placeholder.mjs` handles placeholder package publishing for CI/CD workflows.
- **Database maintenance** – `update-db-schema.cjs` manages schema migrations for the ArchSpine cache system, using CommonJS for legacy compatibility.
- **Demonstration** – `demo.tape` and `project-demo.tape` are terminal recording scripts (`tape` format) that produce visual walkthroughs of the CLI workflow.

These scripts are the operational backbone of the project, providing reproducible automation for every stage of the pipeline.
