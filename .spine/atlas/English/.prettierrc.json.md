# ArchSpine Code Formatting Configuration

This configuration enforces a consistent and readable code style across the entire ArchSpine project. By standardizing formatting, it reduces cognitive overhead during code review and ensures that contributions follow the same conventions.

## Controlled Parameters

- **`printWidth`**: Sets the maximum line length to 100 characters. This balances readability with compact code, preventing overly long lines that are hard to scan.
- **`singleQuote`** (enabled): Forces the use of single quotes for strings instead of double quotes, reducing visual clutter and aligning with common JavaScript style preferences.
- **`semi`** (enabled): Requires semicolons at the end of all statements, preventing subtle errors that can occur when JavaScript's automatic semicolon insertion behaves unexpectedly.
- **`trailingComma`** (set to `"all"`): Enables trailing commas in multi-line arrays, objects, function parameters, etc. This simplifies version control diffs and makes reordering or adding items less error‑prone.

## Stability and Risks

This configuration is purely a code‑formatting standard; it does not affect runtime behavior. However, inconsistency in formatting across the codebase can lead to unnecessary merge conflicts and reduce code quality during collaborative development. Strict enforcement (e.g., via automated formatting tools) is recommended to avoid style drift and maintain readability for all contributors.
