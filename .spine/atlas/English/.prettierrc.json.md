# ArchSpine Code Formatting Configuration

## Role
Defines code formatting rules for the ArchSpine project's source files, ensuring consistent style across contributions.

## Key Responsibilities
- Enforce source code formatting rules
- Configure developer tooling (Prettier)

## Invariants
- **Print width**: Must not exceed 100 characters per line
- **Semicolons**: Required at end of every statement
- **Trailing commas**: Mandatory in all multi-line constructs

## Parameter Definitions
- **printWidth**: Sets the maximum line length to 100 characters. Lines exceeding this will be wrapped by the formatter, improving readability and preventing horizontal scrolling.
- **singleQuote**: Enforces the use of single quotes (`'`) instead of double quotes (`"`) for all string literals. This reduces visual noise and aligns with common JavaScript conventions.
- **semi**: Requires semicolons at the end of every statement. This prevents potential issues from automatic semicolon insertion (ASI) and clarifies statement boundaries.
- **trailingComma**: Mandates trailing commas wherever valid (e.g., in arrays, objects, function parameters). This reduces diff noise in version control when adding new items.

## Stability and Risks
This configuration enforces a strict, predictable code style. The primary risk is developer friction if the rules conflict with existing code or personal preferences, but this is mitigated by automated formatting. The invariants (printWidth, semi, trailingComma) are safe and widely adopted; they improve code consistency and reduce merge conflicts. No operational or security risks are introduced.

## Negative Scope
This configuration does not define any linting rules, type-checking behavior, or build processes. It is solely concerned with code formatting.

## Exported Behavior
This file is consumed by Prettier as its configuration. It is not imported or used programmatically by other modules.
