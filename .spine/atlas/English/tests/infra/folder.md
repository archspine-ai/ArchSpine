The `tests/infra` folder is the verification backbone for ArchSpine’s index-based recovery and LLM retry logic. It contains two concrete test areas: a Vitest suite (`index-recovery.test.ts`) that validates resume recovery from corrupted index states, and a nested `llm` subfolder dedicated to testing the error-classification function used by the retry utility. Together, these tests ensure data integrity and robust error handling when the system faces malformed indices or needs to decide which LLM errors should trigger a retry.
