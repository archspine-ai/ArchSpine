The `tests/infra` directory validates two critical fault-tolerance subsystems inside ArchSpine: index-based recovery and LLM retry handling. It contains two main test suites:

- **index-recovery.test.ts** – A Vitest suite that exercises recovery from corrupted or incomplete index states, ensuring data integrity is restored.  
- **llm/** – A folder containing tests for the LLM retry utility's error classification function, which determines which LLM errors should trigger a retry and which should be surfaced immediately.

Together these tests guarantee that ArchSpine can gracefully survive partial failures in its indexing pipeline and correctly decide when to retry LLM calls, forming the backbone of the system's resilience layer.
