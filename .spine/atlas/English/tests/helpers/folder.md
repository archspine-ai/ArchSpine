This directory provides utilities for testing concurrency control via lock management in the ArchSpine project. Its primary component is the lock-worker script (`lock-worker.mjs`), which runs as a child process in test scenarios. The worker acquires, holds, and releases locks managed by `LockManager`, supporting critical operation modes: acquire-release, acquire-hold, and acquire-release-rewrite. It emits JSON status messages to stdout for test assertion, handles process signals (SIGINT, SIGTERM) for clean termination, and allows a configurable hold duration via command-line argument. This setup is essential for verifying lock-based concurrency mechanisms and ensuring robust signal handling in the mirror system.
