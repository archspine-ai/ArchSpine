该文件是 ArchSpine CLI 流水线的端到端 Vitest 集成测试，用于验证核心工作流程的正确性。它通过创建隔离的临时目录、初始化带有固定分支名称与用户配置的 Git 仓库，并模拟 prompts 模块的行为，来控制 CLI 执行过程。测试运行构建后的 CLI 二进制文件 (`dist/cli/index.js`)，并使用各种参数检查其输出是否符合预期。

### 主要职责
- **环境隔离**：使用 `fs.mkdtempSync` 为每次测试运行创建全新的临时目录。
- **Git 初始化**：通过 `execFileSync` 设置 Git 仓库，包含预定义的分支名称和用户配置。
- **交互模拟**：编写包装脚本，在 CLI 执行期间拦截或模拟 `prompts` 模块的行为。
- **流水线验证**：运行 CLI 二进制文件并附带不同参数，断言其输出与预期一致。

### 范围外 / 不变量
- **不涵盖**：针对单个模块或函数的单元测试、核心流水线之外的 CLI 命令测试、以及超出临时目录和 git init 的环境搭建或清理。
- **不变量**：测试文件须遵守命名约定（`.test.ts` 或 `.spec.ts`）；每次测试必须使用新的临时目录；系统中必须存在 Git 命令（位于 PATH 环境变量中）。

### 公开接口
主要导出的实用函数为 `makeTempDir` 和 `initGitRepo`，它们封装了隔离环境的设置过程。

### 变更意图
从架构上看，该文件作为端到端测试工具，确保 CLI 流水线的核心工作流正确无误。最近的变更（提交 CHK-02、FIX-02）增加了对 Windows 平台的支持并修复了测试问题，与项目的 Git 历史记录保持一致。
