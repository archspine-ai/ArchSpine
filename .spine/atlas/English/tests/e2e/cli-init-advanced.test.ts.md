This file is a Vitest end-to-end test suite specifically for the `init` command of the ArchSpine CLI. It validates that the interactive prompt and filesystem behavior work correctly. Key responsibilities include: setting up isolated temporary directories with Git initialization; spawning the built CLI binary using `child_process.spawnSync` or `execFileSync`; simulating user input by wrapping the `prompts` module; asserting that generated files, directory structures, and configurations match expectations; and cleaning up temporary directories after each test via `afterEach` hooks.

Important invariants: the file name must end with `.test.ts` or `.spec.ts`; all tests must be idempotent and not depend on global system state; Vitest is the sole test runner used. This suite does not include unit tests, integration tests that bypass the full CLI, or any production code. The architectural intent is to provide end-to-end validation of CLI command behavior by simulating real user input and checking filesystem outcomes. No externally visible exports are provided; this file is purely a test runner. (Note: The change intent from commit 171f7b4 added this E2E test suite with 37 tests across 7 files.)
