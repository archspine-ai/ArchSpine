The `tests/e2e` directory contains end-to-end integration tests for the ArchSpine CLI binary. These tests validate the full command-line interface—covering commands like `init`, `generate`, validation, and pipeline execution—by running the built binary inside isolated temporary Git repositories. They simulate user interactions, check exit codes, file system side effects, and rule violation detection, ensuring that every major CLI command works correctly and regressions are caught before release.

The test suite is implemented as eight Vitest files. Key groupings include:

- **CLI initialization and configuration**: `cli-init-advanced.test.ts` focuses on the interactive `init` command and its filesystem behavior; `cli-config.test.ts` covers global configuration options.
- **Pipeline and LLM workflows**: `cli-pipeline-mock.test.ts` and `cli-real-llm.test.ts` test the pipeline end-to-end, with the latter using a real LLM integration and the former using a mock.
- **Read-only and removal commands**: `cli-readonly.test.ts` validates read-only execution (e.g., `--dry-run` or inspect modes), while `cli-remove.test.ts` tests the removal of rules or templates.
- **Repository and violation detection**: `cli-repo.test.ts` exercises repository-level commands, and `cli-real-violation.test.ts` verifies that rule violations are accurately reported in a temporary Git environment.

Each test spawns the built CLI binary with `child_process` and asserts both stdout/stderr output and actual changes to the file system, providing a high level of confidence in the CLI’s correctness.
