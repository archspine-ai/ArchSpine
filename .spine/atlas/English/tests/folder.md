# ArchSpine Test Suite Directory

The `tests/` directory is the quality assurance backbone of the ArchSpine system, containing over 60 test files that provide comprehensive coverage across all core components. This suite is organized into three main testing layers: **unit tests** for isolated module validation, **integration tests** for verifying component interactions, and **end-to-end tests** that exercise complete workflows through the CLI.

## Key Testing Areas

### CLI & Command Workflow
- **`cli.test.ts`** – End-to-end integration tests validating real command-line behavior and service orchestration.
- **`sync-command.test.ts`** and **`sync-cli-built.test.ts`** – Tests for the sync command's repair mode and built executable behavior.
- **`publish-command.test.ts`** and **`publish-preflight.test.ts`** – Validates the publish workflow and preflight service with lock serialization.
- **`view-command.test.ts`** and **`view-service.test.ts`** – Tests for view rendering and service layer behavior.
- **`init-cli.test.ts`** – CLI initialization integration test utilities.

### Configuration & Credentials
- **`config.test.ts`** and **`config-validation.test.ts`** – Validate configuration persistence, language settings, and hook synchronization.
- **`credentials-store.test.ts`** and **`credentials-backend.test.ts`** – Test secure secret storage with Windows DPAPI backend and fallback safety.
- **`pre-commit-config.test.ts`** – Validates pre-commit hook configuration resolution.

### Database & Storage
- **`db-modules.test.ts`** – Integration tests for runtime database schema initialization and batch commits.
- **`integrity.test.ts`** – Tests SpineDB batch commit operations and transaction integrity.
- **`index-reader.test.ts`** – Validates index reader schema enforcement and compatibility.

### LLM Integration
- **`llm.test.ts`** – Validates the LLM Facade infrastructure module's public API surface.
- **`llm-config.test.ts`** – Tests credential backend availability and LLM configuration resolution.
- **`llm-provider-utils.test.ts`** – Tests provider utility functions for parsing and context building.
- **`llm-command-ui.test.ts`** – Validates the LLM command's user interface and configuration surface.
- **`prompt-engine.test.ts`** – Benchmark tests for prompt engine performance under large inputs.
- **`prompt-context-modules.test.ts`** – Tests budget calculation and dependency diagnostics.

### Synchronization & Scanning
- **`scanner.test.ts`** and **`engines/`** – Tests for the Scanner engine with isolated temporary directories.
- **`scan-policy.test.ts`** – Validates ScanPolicy behavior with temporary file structures.
- **`semantic-drift.test.ts`** – Integration tests for semantic drift detection in the sync system.
- **`schema-compliance.test.ts`** – Tests JSON schema validation and sync service orchestration.

### Repair & Recovery
- **`repair-policy.test.ts`** and **`repair-forced-processing.test.ts`** – Validate repair policy evaluation and forced processing behavior.
- **`execution-checkpoint.test.ts`** – Tests checkpoint resume candidate derivation during sync and check operations.
- **`resume-services.test.ts`** – Validates resume-aware aggregation service behavior.
- **`aggregation-resume.test.ts`** – Tests the Aggregator engine's resume functionality.

### View & Rendering
- **`view-renderer.test.ts`** – Tests deterministic rendering of architectural view artifacts for risk-hotspots and public-surface.
- **`view-module-boundary.test.ts`** – Enforces architectural module boundaries for the view service layer.
- **`view-runtime.test.ts`** – Integration tests for experimental view layer configuration resolution.
- **`arch-diagram-renderer.test.ts`** and **`renderer.test.ts`** – Tests for architecture diagram and documentation renderers.

### Infrastructure & Boundaries
- **`infra-facade-boundary.test.ts`** – Validates architectural boundary rules through import analysis.
- **`writer-boundary.test.ts`** – Tests write protection and file system interaction for SpineWriterBoundary.
- **`spine-gate.test.ts`** – Tests protected output mutation detection.
- **`error-system.test.ts`** – Validates error handling consistency across service and infrastructure facade modules.

### Task Management & Runtime
- **`task-runtime.test.ts`** – Tests core task state management, runtime cache, and telemetry statistics.
- **`task-state-guard.test.ts`** – Validates task-state module and TypeScript file collection utilities.
- **`validate-task.test.ts`** – Tests the ValidationTask class for file selection and summary generation.
- **`runtime-service.test.ts`** and **`runtime-session.test.ts`** – Tests LLM configuration resolution, secret injection, and checkpoint management.

### Specialized Tests
- **`mcp-context-mode.test.ts`** – Integration tests for MCP context modes with resource and tool access control.
- **`god-mode.test.ts`** – Tests the God mode dossier feature.
- **`demo-governance.test.ts`** – Acceptance tests for end-to-end demo governance workflow.
- **`robustness.test.ts`** – Tests cross-process lock acquisition and release using isolated worker scripts.
- **`footprint.test.ts`** – Validates footprint stability invariants.
- **`syntax.test.ts`** – Tests the FixTask's TypeScript syntax validation guardrail.
- **`document-languages.test.ts`** – Validates document language selection and ordering logic.
- **`repo-strategy.test.ts`** – Tests repository strategy configuration and drift detection.
- **`rules-loader.test.ts`** – Validates the rule loading subsystem.
- **`runtime-compat-matrix.test.ts`** – Tests runtime compatibility matrix and configuration schema version resolution.
- **`agent-host-init.test.ts`** – Tests repository bootstrap and managed file synchronization.
- **`build-script.test.ts`** – Tests dist packaging filter function.
- **`output-manager.test.ts`** – Tests the pruneAtlasLocales method.
- **`manifest.test.ts`** – Tests Manifest infrastructure with mocked SpineDB.
- **`help-boundary-contract.test.ts`** – Tests CLI help command boundary contracts.
- **`post-commit-derivation.test.ts`** – Tests the PostCommitDerivationTask orchestration pipeline.
- **`context-engine.test.ts`** – Validates ContextEngine's relevance sorting and dependency resolution.

## Supporting Folders

- **`helpers/`** – Contains the lock worker script for testing distributed lock mechanisms.
- **`infra/`** – Contains test suites for index-based recovery and LLM retry mechanisms.
- **`engines/`** – Contains the Scanner engine test suite.
- **`validate-protocol-assets.mjs`** – A validation script for protocol schema conformance.

## Testing Infrastructure

All tests are written using **Vitest** as the test runner and framework. The suite leverages isolated temporary directories, mocked dependencies (particularly for LLM clients and database interactions), and worker scripts for cross-process testing scenarios. The tests are organized to mirror the project's module structure, ensuring that each component has corresponding test coverage at the appropriate level of integration.
