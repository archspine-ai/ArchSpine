This directory defines the architectural and naming conventions for the ArchSpine mirror system. It enforces layer-specific constraints following the CLI → services → core/tasks/engines → infra model, ensuring each module adheres to its responsibilities and separation of concerns. Two key files drive these rules:

- **layered-architecture.yml** – Documents boundary rules for the layered model, with severity levels (Error/Warning) and rationales for each constraint. This file covers CLI entrypoints, service orchestration, core pipelines, task stages, engines, and infrastructure facades.
- **naming-conventions.yml** – Establishes consistent naming patterns, including interfaces prefixed with `I` (e.g., `IUserRepository`) and test files ending with `.test.ts` or `.spec.ts`. Severity levels are Warning for prefix rules and Error for test suffix rules.

Together, these files form the structural backbone that governs how components across all layers are named and organized, making the codebase predictable and easier to navigate.
