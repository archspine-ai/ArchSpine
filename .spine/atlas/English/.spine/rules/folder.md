This directory serves as the **architectural spine** of the ArchSpine codebase, defining the constraints and conventions that preserve structural integrity across the entire monorepo. It houses two essential configuration files that together enforce coherent modularity and consistent naming practices.

- **`layered-architecture.yml`** – Establishes rigid boundaries between system layers (CLI entrypoints, service orchestration, core pipeline, task stages, analysis engines, and infrastructure facades). It specifies allowed and forbidden dependency directions, with severity levels (Error/Warning) and rationales for each rule. This file is the primary defense against architectural erosion.
- **`naming-conventions.yml`** – Defines naming rules for interfaces, test files, and other codebase elements, with scoped constraints, severity levels, and underlying justifications. This ensures that every contributor and AI agent follows consistent naming patterns across all packages.

These files are grouped by their function: one for **structural layering rules** and one for **naming discipline**. The most critical implementation areas are enforcing dependency direction (preventing upward or sideways violations) and standardizing identifier patterns to reduce cognitive load and codebase friction. Together, they form the “constitutional” layer of the ArchSpine project.
