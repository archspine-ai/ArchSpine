# `.spine/rules` — Architectural and Coding Standard Rulebooks

This directory serves as the central rulebook repository for the ArchSpine project. It enforces the layered module architecture, import boundaries, naming conventions, and coding standards that govern the entire monorepo. The two rulebooks here are the backbone of consistency and maintainability across all contributions.

- **`layered-architecture.yml`** — Defines the strict layered module structure (e.g., domain, application, infrastructure) and associated constraints such as dependency direction and package visibility.
- **`naming-conventions.yml`** — Specifies file, directory, and symbol naming patterns plus coding style guidelines, ensuring uniformity across the codebase.

Together, these files are the primary enforcement mechanism for the project's architecture. Any new module, refactor, or contribution must comply with these rules. The pipeline stages (`ast`, `llm`) indicate that the rules are applied both statically and via AI-assisted validation.
