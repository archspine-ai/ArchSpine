The `.spine/rules` directory holds the architectural and coding standards rulebooks that define the core conventions of the ArchSpine project. These rulebooks govern the layered module structure, import boundaries, naming conventions, and general coding practices across the monorepo, acting as the single source of truth for consistency and maintainability.

The directory contains two key documents:
- **layered-architecture.yml** — Enforces the project’s layered module architecture, specifying how modules are grouped, how cross-layer dependencies are permitted, and which tiers exist (e.g., domain, application, infrastructure).
- **naming-conventions.yml** — Defines naming patterns for files, classes, functions, variables, and other code elements; also sets coding standards such as casing, abbreviations, and terminology choices.

Together, these files ensure that all contributions adhere to a uniform structure, making the codebase predictable and easier to navigate. The most impactful implementation areas are the strict layer isolation rules and the consistent naming guidelines, which together reduce accidental coupling and improve code readability.
