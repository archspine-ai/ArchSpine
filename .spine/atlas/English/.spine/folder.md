The root directory of the ArchSpine project mirror system, serving as the central hub for both operational configuration and architectural governance. It holds `config.json`, which defines core operational parameters such as scanning policy (file source, ignore chains, protocol inclusions/exclusions), artifact generation strategy, pre-commit hooks, sync mode, LLM provider details, and initialization defaults for agent instructions, ignore files, and git attributes. The `config.json` also manages project identity and locale support. The subdirectory `.spine/rules` enforces architectural constraints and naming conventions, establishing layered modularity boundaries and consistent naming rules to prevent architectural erosion. Key implementation areas include scanning configuration, artifact generation pipelines, hook integration, and the architectural rule engine that maintains codebase coherence.
