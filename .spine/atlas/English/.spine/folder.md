The `.spine` directory serves as the root configuration and rule directory for the ArchSpine project mirror system. It centralizes project-wide settings for LLM, MCP, hooks, scanning, and artifacts, along with architectural rulebooks that enforce layered module structure, import boundaries, naming conventions, and coding standards across the monorepo.

Notable children include:
- `config.json`: The central configuration file defining project metadata, LLM/MCP settings, hook behavior, artifact strategy, scan policy, and initialization state.
- `rules/`: A directory containing architectural and coding standard rulebooks that govern the project's structure and conventions.

Key implementation areas include the management of LLM and MCP configurations, hook behavior, artifact strategy, and the enforcement of architectural rules through the rulebooks in the `rules/` submodule.
