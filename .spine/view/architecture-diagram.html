ArchSpine

AI-augmented architectural governance mirror system with automated validation and documentation

Core Modules

  • CLI and MCP Server expose commands and AI agent integration
  • Engine layer provides scanning, rule enforcement, and fixing
  • Service and Task layers coordinate resumable pipeline execution

Key Dependencies

  • SQLite for persistent metadata and audit events
  • LLM providers (Gemini, OpenAI) for semantic summaries and specs
  • Secure credential store for API keys and secrets

System Boundaries

  • External LLM providers are accessed via abstracted client
  • User interaction limited to CLI commands and MCP protocol
  • All configuration and rules reside in .spine directory within project
